5. 初等数论

本章说明如何形式化数论中的一些初等结果。随着我们处理更有实质性的数学内容，证明会变得更长、更复杂，并建立在你已经掌握的技能之上。

5.1. 无理根

先从古希腊人已知的一个事实开始：2 的平方根是无理数。若反设不然，则可以把 \(\sqrt{2} = a / b\) 写成既约分数。两边平方得到 \(a^2 = 2 b^2\)，这推出 \(a\) 是偶数。若写 \(a = 2c\)，则得到 \(4c^2 = 2 b^2\)，从而 \(b^2 = 2 c^2\)。这又推出 \(b\) 也是偶数，与我们假定 \(a / b\) 已经约成既约分数相矛盾。

说 \(a / b\) 是既约分数，意味着 \(a\) 与 \(b\) 没有公共因子，也就是说它们互素。Mathlib 把谓词 Nat.Coprime m n 定义为 Nat.gcd m n = 1。使用 Lean 的匿名投影记法，若 s 和 t 是类型 Nat 的表达式，就可以写 s.Coprime t 代替 Nat.Coprime s t；Nat.gcd 也类似。照常，在必要时 Lean 常会自动展开 Nat.Coprime 的定义，但我们也可以用标识符 Nat.Coprime 手动重写或化简。norm_num 策略足够聪明，可以计算具体数值。

#print Nat.Coprime

example (m n : Nat) (h : m.Coprime n) : m.gcd n = 1 :=
  h

example (m n : Nat) (h : m.Coprime n) : m.gcd n = 1 := by
  rw [Nat.Coprime] at h
  exact h

example : Nat.Coprime 12 7 := by norm_num

example : Nat.gcd 12 8 = 4 := by norm_num

我们已经在 第 2.4 节 中遇到过 gcd 函数。整数上也有一个 gcd 版本；稍后我们会回到不同数系之间关系的讨论。甚至还有一般的 gcd 函数，以及在一般代数结构类中有意义的一般 Prime 和 Coprime 概念。下一章我们将理解 Lean 如何管理这种一般性。在此之前，本节只关注自然数。

我们还需要素数概念 Nat.Prime。定理 Nat.prime_def_lt 给出一个熟悉的刻画，而 Nat.Prime.eq_one_or_self_of_dvd 给出另一个刻画。

#check Nat.prime_def_lt

example (p : ℕ) (prime_p : Nat.Prime p) : 2 ≤ p ∧ ∀ m : ℕ, m < p → m ∣ p → m = 1 := by
  rwa [Nat.prime_def_lt] at prime_p

#check Nat.Prime.eq_one_or_self_of_dvd

example (p : ℕ) (prime_p : Nat.Prime p) : ∀ m : ℕ, m ∣ p → m = 1 ∨ m = p :=
  prime_p.eq_one_or_self_of_dvd

example : Nat.Prime 17 := by norm_num

-- commonly used
example : Nat.Prime 2 :=
  Nat.prime_two

example : Nat.Prime 3 :=
  Nat.prime_three

在自然数中，素数具有不能写成非平凡因子乘积的性质。在更广泛的数学语境中，环中具有这一性质的元素称为不可约元素。若环中的一个元素满足：只要它整除一个乘积，就整除其中某个因子，则称它为素元素。自然数的一个重要性质是，在该语境中这两个概念重合，从而得到定理 Nat.Prime.dvd_mul。

可以用这个事实建立上述论证中的关键性质：若一个数的平方是偶数，则该数本身也是偶数。Mathlib 在 Algebra.Group.Even 中定义了谓词 Even，但出于稍后会清楚的原因，我们将简单地使用 2 ∣ m 来表示 m 是偶数。

#check Nat.Prime.dvd_mul
#check Nat.Prime.dvd_mul Nat.prime_two
#check Nat.prime_two.dvd_mul

theorem even_of_even_sqr {m : ℕ} (h : 2 ∣ m ^ 2) : 2 ∣ m := by
  rw [pow_two, Nat.prime_two.dvd_mul] at h
  cases h <;> assumption

example {m : ℕ} (h : 2 ∣ m ^ 2) : 2 ∣ m :=
  Nat.Prime.dvd_of_dvd_pow Nat.prime_two h

随着学习推进，你需要熟练找到所需事实。记住，如果能猜出名称前缀并已导入相关库，就可以使用 Tab 补全（有时是 Ctrl-Tab）寻找目标。可以在任意标识符上 Ctrl-点击，跳转到定义它的文件，从而浏览附近的定义和定理。也可以使用 Lean 社区网页上的搜索引擎；如果都失败了，不要犹豫，到 Zulip 上提问。

example (a b c : Nat) (h : a * b = a * c) (h' : a ≠ 0) : b = c :=
  -- apply? suggests the following:
  (mul_right_inj' h').mp h

我们证明 2 的平方根无理性的核心包含在下面的定理中。请尝试使用 even_of_even_sqr 和定理 Nat.dvd_gcd 补全证明草图。

example {m n : ℕ} (coprime_mn : m.Coprime n) : m ^ 2 ≠ 2 * n ^ 2 := by
  intro sqr_eq
  have : 2 ∣ m := by
    sorry
  obtain ⟨k, meq⟩ := dvd_iff_exists_eq_mul_left.mp this
  have : 2 * (2 * k ^ 2) = 2 * n ^ 2 := by
    rw [← sqr_eq, meq]
    ring
  have : 2 * k ^ 2 = n ^ 2 :=
    sorry
  have : 2 ∣ n := by
    sorry
  have : 2 ∣ m.gcd n := by
    sorry
  have : 2 ∣ 1 := by
    sorry
  norm_num at this

事实上，只需很少修改，就可以把 2 替换为任意素数。请在下一个例子中试一试。在证明末尾，需要从 p ∣ 1 导出矛盾。可以使用 Nat.Prime.two_le，它说明任意素数都大于等于 2；以及 Nat.le_of_dvd。

example {m n p : ℕ} (coprime_mn : m.Coprime n) (prime_p : p.Prime) : m ^ 2 ≠ p * n ^ 2 := by
  sorry

让我们考虑另一种方法。下面快速证明：若 \(p\) 是素数，则 \(m^2 \ne p n^2\)。如果假设 \(m^2 = p n^2\)，并考虑 \(m\) 和 \(n\) 的素因子分解，则 \(p\) 在等式左侧出现偶数次，而在右侧出现奇数次，矛盾。注意，这个论证要求 \(n\)，从而 \(m\)，不等于零。下面的形式化确认这个假设足够。

唯一分解定理说，每个非零自然数都能以唯一方式写成素数的乘积。Mathlib 包含该事实的形式版本，用函数 Nat.primeFactorsList 表示；它返回一个数按非递减顺序排列的素因子列表。库证明了 Nat.primeFactorsList n 的所有元素都是素数，任意大于零的 n 等于其因子的乘积，并且如果 n 等于另一个素数列表的乘积，那么该列表是 Nat.primeFactorsList n 的一个排列。

#check Nat.primeFactorsList
#check Nat.prime_of_mem_primeFactorsList
#check Nat.prod_primeFactorsList
#check Nat.primeFactorsList_unique

即使我们还没有讨论列表成员关系、乘积或排列，你也可以浏览这些定理以及附近的其他定理。当前任务不需要其中任何内容。相反，我们将使用 Mathlib 有一个函数 Nat.factorization 这一事实，它把同样的数据表示为函数。具体地说，Nat.factorization n p，也可写作 n.factorization p，返回 p 在 n 的素因子分解中的重数。我们将使用以下三个事实。

theorem factorization_mul' {m n : ℕ} (mnez : m ≠ 0) (nnez : n ≠ 0) (p : ℕ) :
    (m * n).factorization p = m.factorization p + n.factorization p := by
  rw [Nat.factorization_mul mnez nnez]
  rfl

theorem factorization_pow' (n k p : ℕ) :
    (n ^ k).factorization p = k * n.factorization p := by
  rw [Nat.factorization_pow]
  rfl

theorem Nat.Prime.factorization' {p : ℕ} (prime_p : p.Prime) :
    p.factorization p = 1 := by
  rw [prime_p.factorization]
  simp

事实上，n.factorization 在 Lean 中定义为一个有限支撑函数；这解释了你逐步查看上面证明时会看到的奇怪记号。现在不必担心这些。就这里的目的而言，可以把上面三个定理当作黑箱使用。

下一个例子说明，化简器足够聪明，可以把 n^2 ≠ 0 替换为 n ≠ 0。策略 simpa 只是调用 simp 后接 assumption。

看看能否使用上述恒等式补全证明中缺失的部分。

example {m n p : ℕ} (nnz : n ≠ 0) (prime_p : p.Prime) : m ^ 2 ≠ p * n ^ 2 := by
  intro sqr_eq
  have nsqr_nez : n ^ 2 ≠ 0 := by simpa
  have eq1 : Nat.factorization (m ^ 2) p = 2 * m.factorization p := by
    sorry
  have eq2 : (p * n ^ 2).factorization p = 2 * n.factorization p + 1 := by
    sorry
  have : 2 * m.factorization p % 2 = (2 * n.factorization p + 1) % 2 := by
    rw [← eq1, sqr_eq, eq2]
  rw [add_comm, Nat.add_mul_mod_self_left, Nat.mul_mod_right] at this
  norm_num at this

这个证明的一个优点是它也能推广。2 并没有特殊之处；稍作修改后，该证明表明：只要写出 m^k = r * n^k，任意素数 p 在 r 中的重数必须是 k 的倍数。

为了把 Nat.count_factors_mul_of_pos 用于 r * n^k，需要知道 r 为正。但当 r 为零时，下面的定理是平凡的，并且容易由化简器证明。因此证明按情形进行。行 rcases r with _ | r 把目标替换为两个版本：一个把 r 替换为 0，另一个把 r 替换为 r + 1。在第二种情形中，可以使用定理 r.succ_ne_zero，它建立 r + 1 ≠ 0（succ 表示后继）。

还要注意，以 have : npow_nz 开头的那一行给出了 n^k ≠ 0 的一个短证明项证明。为了理解它如何工作，可以尝试把它替换为策略证明，然后思考这些策略如何描述该证明项。

看看能否补全下面证明中的缺失部分。在最后，可以使用 Nat.dvd_sub 和 Nat.dvd_mul_right 完成证明。

注意，这个例子并不假定 p 是素数；但当 p 不是素数时，结论是平凡的，因为此时 r.factorization p 按定义为零，而证明无论如何都适用于所有情形。

example {m n k r : ℕ} (nnz : n ≠ 0) (pow_eq : m ^ k = r * n ^ k) {p : ℕ} :
    k ∣ r.factorization p := by
  rcases r with _ | r
  · simp
  have npow_nz : n ^ k ≠ 0 := fun npowz ↦ nnz (eq_zero_of_pow_eq_zero npowz)
  have eq1 : (m ^ k).factorization p = k * m.factorization p := by
    sorry
  have eq2 : ((r + 1) * n ^ k).factorization p =
      k * n.factorization p + (r + 1).factorization p := by
    sorry
  have : r.succ.factorization p = k * m.factorization p - k * n.factorization p := by
    rw [← eq1, pow_eq, eq2, add_comm, Nat.add_sub_cancel]
  rw [this]
  sorry

我们可能希望以多种方式改进这些结果。首先，一个证明 2 的平方根无理性的证明，应当真正谈论 2 的平方根，也就是实数或复数中的一个元素。而说它无理，应当谈论有理数，即没有任何有理数等于它。此外，还应把本节定理扩展到整数。虽然数学上显然，如果能把 2 的平方根写成两个整数的商，也就能把它写成两个自然数的商，但形式化证明这一点需要一些工作。

在 Mathlib 中，自然数、整数、有理数、实数和复数分别由不同的数据类型表示。把注意力限制在单独的论域上往往很有帮助：我们会看到，对自然数做归纳很容易；而当实数不在图景中时，关于整数整除性的推理最容易。但必须在不同论域之间协调是一件麻烦事，我们将不得不处理它。本章稍后会回到这个问题。

我们还应期待能够加强最后一个定理的结论，说明数 r 是一个 k 次幂，因为它的 k 次根就是每个整除 r 的素数，按其在 r 中的重数除以 k 后所得指数相乘。为了做到这一点，需要更好的手段来推理有限集上的乘积和求和，这也是我们稍后会回到的主题。

事实上，本节中的结果都已在 Mathlib 的 Data.Real.Irrational 中以远为一般的形式建立。重数概念对任意交换幺半群都有定义，并且取值于扩展自然数 enat，它是在自然数中加入无穷大所得的类型。下一章我们将开始发展理解 Lean 如何支持这种一般性的手段。

5.2. 归纳与递归

自然数集 \(\mathbb{N} = \{0, 1, 2, \ldots\}\) 不仅本身极为重要，也在构造新的数学对象时起核心作用。Lean 的基础允许我们声明归纳类型，即由给定构造子列表归纳生成的类型。在 Lean 中，自然数声明如下。

inductive Nat where
  | zero : Nat
  | succ (n : Nat) : Nat

可以通过写 #check Nat，然后在标识符 Nat 上 Ctrl-点击，在库中找到这个定义。该命令说明 Nat 是由两个构造子 zero : Nat 和 succ : Nat → Nat 自由且归纳生成的数据类型。当然，库分别为 Nat 和 zero 引入记号 ℕ 和 0。（数字会被翻译为二进制表示，但现在不必关心这些细节。）

对工作数学家来说，“自由”意味着类型 Nat 有一个元素 zero，以及一个单射的后继函数 succ，且 succ 的像不包含 zero。

example (n : Nat) : n.succ ≠ Nat.zero :=
  Nat.succ_ne_zero n

example (m n : Nat) (h : m.succ = n.succ) : m = n :=
  Nat.succ.inj h

对工作数学家来说，“归纳”意味着自然数带有归纳证明原理和递归定义原理。本节将说明如何使用它们。

下面是阶乘函数递归定义的一个例子。

def fac : ℕ → ℕ
  | 0 => 1
  | n + 1 => (n + 1) * fac n

这种语法需要一些时间适应。注意第一行没有 :=。接下来的两行给出递归定义的基本情形和归纳步骤。这些方程在定义意义下成立，但也可以通过把名称 fac 提供给 simp 或 rw 来手动使用。

example : fac 0 = 1 :=
  rfl

example : fac 0 = 1 := by
  rw [fac]

example : fac 0 = 1 := by
  simp [fac]

example (n : ℕ) : fac (n + 1) = (n + 1) * fac n :=
  rfl

example (n : ℕ) : fac (n + 1) = (n + 1) * fac n := by
  rw [fac]

example (n : ℕ) : fac (n + 1) = (n + 1) * fac n := by
  simp [fac]

阶乘函数实际上已经在 Mathlib 中定义为 Nat.factorial。同样，可以输入 #check Nat.factorial 并使用 Ctrl-点击跳转到它。出于说明目的，我们将在例子中继续使用 fac。Nat.factorial 定义前的标注 @[simp] 指定把定义方程加入化简器默认使用的恒等式数据库。

归纳原理说，可以通过证明命题对 0 成立，并证明一旦它对自然数 \(n\) 成立也对 \(n + 1\) 成立，来证明关于自然数的一般命题。因此，下面证明中的行 induction' n with n ih 会产生两个目标：第一个需要证明 0 < fac 0，第二个带有附加假设 ih : 0 < fac n，并要求证明 0 < fac (n + 1)。短语 with n ih 用于给归纳变量和归纳假设命名，你可以为它们选择任意名称。

theorem fac_pos (n : ℕ) : 0 < fac n := by
  induction' n with n ih
  · rw [fac]
    exact zero_lt_one
  rw [fac]
  exact mul_pos n.succ_pos ih

induction' 策略足够聪明，会把依赖于归纳变量的假设纳入归纳假设。请逐步查看下一个例子，看看发生了什么。

theorem dvd_fac {i n : ℕ} (ipos : 0 < i) (ile : i ≤ n) : i ∣ fac n := by
  induction' n with n ih
  · exact absurd ipos (not_lt_of_ge ile)
  rw [fac]
  rcases Nat.of_le_succ ile with h | h
  · apply dvd_mul_of_dvd_right (ih h)
  rw [h]
  apply dvd_mul_right

下面的例子给出阶乘函数的一个粗略下界。事实证明，先按情形证明更容易，这样证明余下部分从 \(n = 1\) 的情形开始。看看能否使用 pow_succ 或 pow_succ' 通过归纳证明完成论证。

theorem pow_two_le_fac (n : ℕ) : 2 ^ (n - 1) ≤ fac n := by
  rcases n with _ | n
  · simp [fac]
  sorry

归纳常用于证明涉及有限和与有限乘积的恒等式。Mathlib 定义表达式 Finset.sum s f，其中 s : Finset α 是类型 α 的元素组成的有限集，而 f 是定义在 α 上的函数。f 的陪域可以是任意支持带零元的交换结合加法运算的类型。若导入 Algebra.BigOperators.Ring 并执行命令 open BigOperators，就可以使用更直观的记号 ∑ x ∈ s, f x。当然，有限乘积也有类似运算和记号。

我们将在下一节以及后续章节中讨论 Finset 类型及其支持的运算。现在只使用 Finset.range n，它是小于 n 的自然数组成的有限集。

variable {α : Type*} (s : Finset ℕ) (f : ℕ → ℕ) (n : ℕ)

#check Finset.sum s f
#check Finset.prod s f

open BigOperators
open Finset

example : s.sum f = ∑ x ∈ s, f x :=
  rfl

example : s.prod f = ∏ x ∈ s, f x :=
  rfl

example : (range n).sum f = ∑ x ∈ range n, f x :=
  rfl

example : (range n).prod f = ∏ x ∈ range n, f x :=
  rfl

事实 Finset.sum_range_zero 和 Finset.sum_range_succ 给出了到 \(n\) 为止的求和的递归描述；乘积也类似。

example (f : ℕ → ℕ) : ∑ x ∈ range 0, f x = 0 :=
  Finset.sum_range_zero f

example (f : ℕ → ℕ) (n : ℕ) : ∑ x ∈ range n.succ, f x = ∑ x ∈ range n, f x + f n :=
  Finset.sum_range_succ f n

example (f : ℕ → ℕ) : ∏ x ∈ range 0, f x = 1 :=
  Finset.prod_range_zero f

example (f : ℕ → ℕ) (n : ℕ) : ∏ x ∈ range n.succ, f x = (∏ x ∈ range n, f x) * f n :=
  Finset.prod_range_succ f n

每一对中的第一个恒等式按定义成立，也就是说，可以把证明替换为 rfl。

下面把我们定义的阶乘函数表示为一个乘积。

example (n : ℕ) : fac n = ∏ i ∈ range n, (i + 1) := by
  induction' n with n ih
  · simp [fac]
  simp [fac, ih, prod_range_succ, mul_comm]

把 mul_comm 作为化简规则纳入值得说明。用恒等式 x * y = y * x 化简看起来应当很危险，因为它通常会无限循环。Lean 的化简器足够聪明，能够识别这一点；它只在所得项相对于某个固定但任意的项排序更小时才应用该规则。下面的例子表明，使用 mul_assoc、mul_comm 和 mul_left_comm 三条规则化简，能够识别那些只在括号位置和变量顺序上不同的相同乘积。

example (a b c d e f : ℕ) : a * (b * c * f * (d * e)) = d * (a * f * e) * (c * b) := by
  simp [mul_comm, mul_left_comm]

粗略地说，这些规则通过把括号向右推，然后按同一个标准顺序重新排列两边表达式来工作。使用这些规则以及对应的加法规则化简，是一个方便技巧。

回到求和恒等式，我们建议逐步查看下面的证明：从 0 到 \(n\) 的自然数之和为 \(n(n+1)/2\)。证明的第一步清除分母。形式化恒等式时，这通常很有用，因为带除法的计算一般有旁条件。（类似地，在自然数上尽可能避免使用减法也很有用。）

theorem sum_id (n : ℕ) : ∑ i ∈ range (n + 1), i = n * (n + 1) / 2 := by
  symm; apply Nat.div_eq_of_eq_mul_right (by norm_num : 0 < 2)
  induction' n with n ih
  · simp
  rw [Finset.sum_range_succ, mul_add 2, ← ih]
  ring

我们鼓励你证明平方和的类似恒等式，以及你能在网上找到的其他恒等式。

theorem sum_sqr (n : ℕ) : ∑ i ∈ range (n + 1), i ^ 2 = n * (n + 1) * (2 * n + 1) / 6 := by
  sorry

在 Lean 的核心库中，加法和乘法本身也是用递归定义的，并且它们的基本性质由归纳证明建立。如果你喜欢思考这类基础性主题，也许会乐于完成乘法和加法的交换律、结合律以及乘法对加法分配律的证明。可以按照下面的提纲，在自然数的一个副本上完成这些工作。注意，可以对 MyNat 使用 induction 策略；Lean 足够聪明，知道应使用相关归纳原理（当然，它与 Nat 的归纳原理相同）。

我们先从加法交换律开始。一个好的经验法则是：由于加法和乘法是按第二个参数递归定义的，通常对出现在该位置的变量做归纳更有利。在结合律证明中决定使用哪个变量稍微有些微妙。

如果没有通常的零、一、加法和乘法记号，书写内容可能会让人困惑。稍后我们会学习如何定义这样的记号。在命名空间 MyNat 中工作，意味着可以写 zero 和 succ，而不必写 MyNat.zero 和 MyNat.succ，并且这些名称的解释优先于其他解释。在命名空间之外，例如下面定义的 add 的完整名称是 MyNat.add。

如果你发现自己确实喜欢这类事情，可以尝试定义截断减法和幂运算，并证明它们的一些性质。记住，截断减法会在零处截断。为了定义它，先定义前驱函数 pred 很有用；它从任意非零数中减去一，并保持零不变。函数 pred 可以由一个简单的递归实例定义。

inductive MyNat where
  | zero : MyNat
  | succ : MyNat → MyNat

namespace MyNat

def add : MyNat → MyNat → MyNat
  | x, zero => x
  | x, succ y => succ (add x y)

def mul : MyNat → MyNat → MyNat
  | _, zero => zero
  | x, succ y => add (mul x y) x

theorem zero_add (n : MyNat) : add zero n = n := by
  induction' n with n ih
  · rfl
  rw [add, ih]

theorem succ_add (m n : MyNat) : add (succ m) n = succ (add m n) := by
  induction' n with n ih
  · rfl
  rw [add, ih]
  rfl

theorem add_comm (m n : MyNat) : add m n = add n m := by
  induction' n with n ih
  · rw [zero_add]
    rfl
  rw [add, succ_add, ih]

  sorry

  sorry

  sorry

5.3. 无穷多个素数

让我们用另一个数学标准例子继续探索归纳与递归：证明有无穷多个素数。其一种表述是：对每个自然数 \(n\)，都存在一个大于 \(n\) 的素数。为了证明这一点，令 \(p\) 为 \(n! + 1\) 的任意素因子。若 \(p\) 小于等于 \(n\)，则它整除 \(n!\)。由于它也整除 \(n! + 1\)，于是它整除 1，矛盾。因此 \(p\) 大于 \(n\)。

要形式化该证明，需要说明任意大于等于 2 的数都有素因子。为此，需要说明任意不等于 0 或 1 的自然数都大于等于 2。这引出了形式化的一个怪异特征：常常正是这种平凡陈述最令人烦恼。这里我们考察几种处理方法。

首先，可以使用 cases 策略，以及后继函数尊重自然数序关系这一事实。

theorem two_le {m : ℕ} (h0 : m ≠ 0) (h1 : m ≠ 1) : 2 ≤ m := by
  cases m; contradiction
  case succ m =>
    cases m; contradiction
    repeat apply Nat.succ_le_succ
    apply zero_le

另一种策略是使用 interval_cases，当相关变量位于自然数或整数的某个区间内时，它会自动把目标拆成各个情形。记住，可以把鼠标悬停在它上面查看文档。

example {m : ℕ} (h0 : m ≠ 0) (h1 : m ≠ 1) : 2 ≤ m := by
  by_contra h
  push_neg at h
  interval_cases m <;> contradiction

回忆一下，interval_cases m 后面的分号表示下一个策略应用于它生成的每一个情形。还有一种选择是使用 decide 策略，它会尝试寻找判定过程来解决问题。Lean 知道，若一个命题以有界量词 ∀ x, x < n → ... 或 ∃ x, x < n ∧ ... 开始，就可以通过判定有限多个实例来判定其真值。

example {m : ℕ} (h0 : m ≠ 0) (h1 : m ≠ 1) : 2 ≤ m := by
  by_contra h
  push_neg at h
  revert h0 h1
  revert h m
  decide

有了定理 two_le，我们先证明每个大于 2 的自然数都有素因子。Mathlib 包含函数 Nat.minFac，它返回最小素因子；但为了学习库中的新内容，我们会避免使用它，直接证明该定理。

这里，普通归纳还不够。我们想使用强归纳，它允许我们通过证明对每个数 \(n\)，若性质 \(P\) 对所有小于 \(n\) 的值成立，则它在 \(n\) 处也成立，来证明每个自然数 \(n\) 都具有性质 \(P\)。在 Lean 中，这个原理称为 Nat.strong_induction_on，我们可以用关键字 using 告诉 induction 策略使用它。注意，这样做时没有基本情形；它被包含在一般归纳步骤中。

论证很简单。假设 \(n \ge 2\)，若 \(n\) 是素数，则完成。若不是，则由素数含义的某个刻画，它有一个非平凡因子 \(m\)，于是可以对该因子应用归纳假设。请逐步查看下一个证明，看看它如何展开。

theorem exists_prime_factor {n : Nat} (h : 2 ≤ n) : ∃ p : Nat, p.Prime ∧ p ∣ n := by
  by_cases np : n.Prime
  · use n, np
  induction' n using Nat.strong_induction_on with n ih
  rw [Nat.prime_def_lt] at np
  push_neg at np
  rcases np h with ⟨m, mltn, mdvdn, mne1⟩
  have : m ≠ 0 := by
    intro mz
    rw [mz, zero_dvd_iff] at mdvdn
    linarith
  have mgt2 : 2 ≤ m := two_le this mne1
  by_cases mp : m.Prime
  · use m, mp
  · rcases ih m mltn mgt2 mp with ⟨p, pp, pdvd⟩
    use p, pp
    apply pdvd.trans mdvdn

现在可以证明我们定理的如下表述。看看能否补全草图。可以使用 Nat.factorial_pos、Nat.dvd_factorial 和 Nat.dvd_sub'。

theorem primes_infinite : ∀ n, ∃ p > n, Nat.Prime p := by
  intro n
  have : 2 ≤ Nat.factorial n + 1 := by
    sorry
  rcases exists_prime_factor this with ⟨p, pp, pdvd⟩
  refine ⟨p, ?_, pp⟩
  show p > n
  by_contra ple
  push_neg at ple
  have : p ∣ Nat.factorial n := by
    sorry
  have : p ∣ 1 := by
    sorry
  show False
  sorry

我们来考虑上述证明的一个变体：不使用阶乘函数，而是假设给定有限集 \(\{p_1, \ldots, p_n\}\)，并考虑 \(\prod_{i=1}^n p_i + 1\) 的一个素因子。这个素因子必然不同于每个 \(p_i\)，从而说明不存在包含所有素数的有限集。

形式化这个论证要求我们推理有限集。在 Lean 中，对任意类型 α，类型 Finset α 表示类型 α 的元素组成的有限集。计算性地推理有限集需要有一个判定 α 上相等性的过程，这就是为什么下面代码片段包含假设 [DecidableEq α]。对 ℕ、ℤ 和 ℚ 这样的具体数据类型，该假设会自动满足。对实数推理时，可以通过使用经典逻辑并放弃计算解释来满足它。

我们使用命令 open Finset，以便使用相关定理的较短名称。不同于集合的情形，大多数涉及有限集的等价命题并不按定义成立，因此需要使用 Finset.subset_iff、Finset.mem_union、Finset.mem_inter 和 Finset.mem_sdiff 这类等价手动展开。ext 策略仍可用于通过证明一个有限集的每个元素是另一个有限集的元素，来说明两个有限集相等。

open Finset

section
variable {α : Type*} [DecidableEq α] (r s t : Finset α)

example : r ∩ (s ∪ t) ⊆ r ∩ s ∪ r ∩ t := by
  rw [subset_iff]
  intro x
  rw [mem_inter, mem_union, mem_union, mem_inter, mem_inter]
  tauto

example : r ∩ (s ∪ t) ⊆ r ∩ s ∪ r ∩ t := by
  simp [subset_iff]
  intro x
  tauto

example : r ∩ s ∪ r ∩ t ⊆ r ∩ (s ∪ t) := by
  simp [subset_iff]
  intro x
  tauto

example : r ∩ s ∪ r ∩ t = r ∩ (s ∪ t) := by
  ext x
  simp
  tauto

end

我们使用了一个新技巧：tauto 策略（以及使用经典逻辑的加强版本 tauto!）可以用来处理命题逻辑重言式。看看能否使用这些方法证明下面两个例子。

example : (r ∪ s) ∩ (r ∪ t) = r ∪ s ∩ t := by
  sorry
example : (r \ s) \ t = r \ (s ∪ t) := by
  sorry

定理 Finset.dvd_prod_of_mem 告诉我们，如果 n 是有限集 s 的元素，则 n 整除 ∏ i ∈ s, i。

example (s : Finset ℕ) (n : ℕ) (h : n ∈ s) : n ∣ ∏ i ∈ s, i :=
  Finset.dvd_prod_of_mem _ h

还需要知道，当 n 是素数且 s 是素数集合时，反向命题成立。为了说明这一点，需要下面的引理；你应当能用定理 Nat.Prime.eq_one_or_self_of_dvd 证明它。

theorem _root_.Nat.Prime.eq_of_dvd_of_prime {p q : ℕ}
      (prime_p : Nat.Prime p) (prime_q : Nat.Prime q) (h : p ∣ q) :
    p = q := by
  sorry

可以用这个引理证明：若素数 p 整除一组有限素数的乘积，则它等于其中某个素数。Mathlib 提供了关于有限集的有用归纳原理：要证明一个性质对任意有限集 s 成立，只需证明它对空集成立，并且证明当加入单个新元素 a ∉ s 时它保持成立。这个原理称为 Finset.induction_on。当我们告诉 induction 策略使用它时，也可以指定名称 a 和 s、归纳步骤中假设 a ∉ s 的名称，以及归纳假设的名称。表达式 Finset.insert a s 表示 s 与单元素集 a 的并。恒等式 Finset.prod_empty 和 Finset.prod_insert 则提供乘积的相关重写规则。在下面的证明中，第一个 simp 应用了 Finset.prod_empty。请逐步查看证明开头，观察归纳如何展开，然后完成它。

theorem mem_of_dvd_prod_primes {s : Finset ℕ} {p : ℕ} (prime_p : p.Prime) :
    (∀ n ∈ s, Nat.Prime n) → (p ∣ ∏ n ∈ s, n) → p ∈ s := by
  intro h₀ h₁
  induction' s using Finset.induction_on with a s ans ih
  · simp at h₁
    linarith [prime_p.two_le]
  simp [Finset.prod_insert ans, prime_p.dvd_mul] at h₀ h₁
  rw [mem_insert]
  sorry

我们还需要有限集的最后一个性质。给定元素 s : Set α 和 α 上的谓词 P，在 第 4 章 中我们写 { x ∈ s | P x } 表示 s 中满足 P 的元素集合。给定 s : Finset α，类似概念写作 s.filter P。

example (s : Finset ℕ) (x : ℕ) : x ∈ s.filter Nat.Prime ↔ x ∈ s ∧ x.Prime :=
  mem_filter

现在证明“有无穷多个素数”的另一个表述：给定任意 s : Finset ℕ，存在一个不属于 s 的素数 p。为了导出矛盾，我们假设所有素数都在 s 中，然后缩减到一个只包含且包含所有素数的集合 s'。取这个集合的乘积，加一，再找结果的一个素因子，就会得到所需矛盾。看看能否完成下面的草图。在证明第一个 have 时，可以使用 Finset.prod_pos。

theorem primes_infinite' : ∀ s : Finset Nat, ∃ p, Nat.Prime p ∧ p ∉ s := by
  intro s
  by_contra h
  push_neg at h
  set s' := s.filter Nat.Prime with s'_def
  have mem_s' : ∀ {n : ℕ}, n ∈ s' ↔ n.Prime := by
    intro n
    simp [s'_def]
    apply h
  have : 2 ≤ (∏ i ∈ s', i) + 1 := by
    sorry
  rcases exists_prime_factor this with ⟨p, pp, pdvd⟩
  have : p ∣ ∏ i ∈ s', i := by
    sorry
  have : p ∣ 1 := by
    convert Nat.dvd_sub pdvd this
    simp
  show False
  sorry

我们已经看到两种说“有无穷多个素数”的方式：说它们不被任何 n 所界定，以及说它们不包含于任何有限集 s。下面两个证明表明这些表述等价。在第二个证明中，为了构造 s.filter Q，必须假设有一个过程能判定 Q 是否成立。Lean 知道 Nat.Prime 有这样的判定过程。一般而言，如果通过写 open Classical 使用经典逻辑，就可以去掉这个假设。

在 Mathlib 中，Finset.sup s f 表示当 x 遍历 s 时 f x 的值的上确界；当 s 为空且 f 的陪域为 ℕ 时返回 0。在第一个证明中，我们使用 s.sup id，其中 id 是恒等函数，来指代 s 中的最大值。

theorem bounded_of_ex_finset (Q : ℕ → Prop) :
    (∃ s : Finset ℕ, ∀ k, Q k → k ∈ s) → ∃ n, ∀ k, Q k → k < n := by
  rintro ⟨s, hs⟩
  use s.sup id + 1
  intro k Qk
  apply Nat.lt_succ_of_le
  show id k ≤ s.sup id
  apply le_sup (hs k Qk)

theorem ex_finset_of_bounded (Q : ℕ → Prop) [DecidablePred Q] :
    (∃ n, ∀ k, Q k → k ≤ n) → ∃ s : Finset ℕ, ∀ k, Q k ↔ k ∈ s := by
  rintro ⟨n, hn⟩
  use (range (n + 1)).filter Q
  intro k
  simpa using hn k

对我们第二个“素数无穷多”证明稍作变化，可以证明有无穷多个模 4 余 3 的素数。论证如下。首先注意，若两个数 \(m\) 与 \(n\) 的乘积模 4 等于 3，则这两个数之一模 4 余 3。毕竟二者都必须为奇数；若它们都模 4 余 1，则乘积也模 4 余 1。可以用这个观察证明：若某个大于 2 的数模 4 余 3，则它有一个同样模 4 余 3 的素因子。

现在假设只有有限多个模 4 余 3 的素数，设为 \(p_1, \ldots, p_k\)。不失一般性，可以假设 \(p_1 = 3\)。考虑乘积 \(4 \prod_{i=2}^k p_i + 3\)。容易看出它模 4 余 3，因此有一个模 4 余 3 的素因子 \(p\)。不可能有 \(p = 3\)；因为 \(p\) 整除 \(4 \prod_{i=2}^k p_i + 3\)，若 \(p\) 等于 3，则它也整除 \(\prod_{i=2}^k p_i\)，从而推出 \(p\) 等于某个 \(p_i\)（\(i = 2, \ldots, k\)）；而我们已把 3 从这个列表中排除。因此 \(p\) 必须是其他某个元素 \(p_i\)。但在这种情况下，\(p\) 整除 \(4 \prod_{i=2}^k p_i\)，于是也整除 3，这与它不等于 3 相矛盾。

在 Lean 中，记号 n % m 读作“n 模 m”，表示 n 除以 m 的余数。

example : 27 % 4 = 3 := by norm_num

于是可以把“n 模 4 余 3”写成 n % 4 = 3。下面的例子和定理总结了稍后需要使用的关于该函数的事实。第一个命名定理再次展示了按少数情形推理。在第二个命名定理中，记住分号表示随后的策略块应用于前一个策略生成的所有目标。

example (n : ℕ) : (4 * n + 3) % 4 = 3 := by
  rw [add_comm, Nat.add_mul_mod_self_left]

theorem mod_4_eq_3_or_mod_4_eq_3 {m n : ℕ} (h : m * n % 4 = 3) : m % 4 = 3 ∨ n % 4 = 3 := by
  revert h
  rw [Nat.mul_mod]
  have : m % 4 < 4 := Nat.mod_lt m (by norm_num)
  interval_cases m % 4 <;> simp [-Nat.mul_mod_mod]
  have : n % 4 < 4 := Nat.mod_lt n (by norm_num)
  interval_cases n % 4 <;> simp

theorem two_le_of_mod_4_eq_3 {n : ℕ} (h : n % 4 = 3) : 2 ≤ n := by
  apply two_le <;>
    · intro neq
      rw [neq] at h
      norm_num at h

我们还需要下面这个事实：若 m 是 n 的非平凡因子，则 n / m 也是如此。看看能否使用 Nat.div_dvd_of_dvd 和 Nat.div_lt_self 完成证明。

现在把所有部分组合起来，证明任意模 4 余 3 的数都有一个具有同样性质的素因子。

theorem exists_prime_factor_mod_4_eq_3 {n : Nat} (h : n % 4 = 3) :
    ∃ p : Nat, p.Prime ∧ p ∣ n ∧ p % 4 = 3 := by
  by_cases np : n.Prime
  · use n
  induction' n using Nat.strong_induction_on with n ih
  rw [Nat.prime_def_lt] at np
  push_neg at np
  rcases np (two_le_of_mod_4_eq_3 h) with ⟨m, mltn, mdvdn, mne1⟩

    intro mz
    rw [mz, zero_dvd_iff] at mdvdn
    linarith

  have : m % 4 = 3 ∨ n / m % 4 = 3 := by
    apply mod_4_eq_3_or_mod_4_eq_3
    rw [neq, h]
  rcases this with h1 | h1
  . sorry
  . sorry

5.4. 更多归纳法

在 第 5.2 节 中，我们看到如何通过自然数上的递归来定义阶乘函数。

def fac : ℕ → ℕ
  | 0 => 1
  | n + 1 => (n + 1) * fac n

我们也看到如何使用 induction' 策略证明定理。

theorem fac_pos (n : ℕ) : 0 < fac n := by
  induction' n with n ih
  · rw [fac]
    exact zero_lt_one
  rw [fac]
  exact mul_pos n.succ_pos ih

induction 策略（不带撇号）允许使用更结构化的语法。

example (n : ℕ) : 0 < fac n := by
  induction n
  case zero =>
    rw [fac]
    exact zero_lt_one
  case succ n ih =>
    rw [fac]
    exact mul_pos n.succ_pos ih

example (n : ℕ) : 0 < fac n := by
  induction n with
  | zero =>
    rw [fac]
    exact zero_lt_one
  | succ n ih =>
    rw [fac]
    exact mul_pos n.succ_pos ih

照常，可以把鼠标悬停在 induction 关键字上阅读文档。情形名称 zero 和 succ 来自类型 ℕ 的定义。注意，succ 情形允许你为归纳变量和归纳假设选择任意名称，这里是 n 和 ih。甚至可以用与定义递归函数相同的记号来证明定理。

theorem fac_pos' : ∀ n, 0 < fac n
  | 0 => by
    rw [fac]
    exact zero_lt_one
  | n + 1 => by
    rw [fac]
    exact mul_pos n.succ_pos (fac_pos' n)

还要注意这里没有 :=，冒号之后没有 ∀ n，每个情形中有关键字 by，并且在归纳步骤中调用 fac_pos' n 作为归纳调用。它仿佛把该定理看作 n 的递归函数，并在归纳步骤中进行递归调用。

这种定义风格非常灵活。Lean 的设计者内置了复杂的递归函数定义机制，而这些机制也延伸到归纳证明。例如，可以用多个基本情形定义 Fibonacci 函数。

@[simp] def fib : ℕ → ℕ
  | 0 => 0
  | 1 => 1
  | n + 2 => fib n + fib (n + 1)

标注 @[simp] 表示化简器将使用这些定义方程。也可以写 rw [fib] 来应用它们。下面给 n + 2 情形命名会很有帮助。

theorem fib_add_two (n : ℕ) : fib (n + 2) = fib n + fib (n + 1) := rfl

example (n : ℕ) : fib (n + 2) = fib n + fib (n + 1) := by rw [fib]

使用 Lean 的递归函数记法，可以对自然数进行归纳证明，并使证明形式与 fib 的递归定义相呼应。下面的例子用黄金比例 \(\varphi\) 及其共轭 \(\varphi'\) 给出第 n 个 Fibonacci 数的显式公式。我们必须告诉 Lean 不期望这些定义生成代码，因为实数上的算术运算不可计算。

我们将使用 grind 策略完成计算，并告诉它使用 phi 和 phi’ 的定义，以及归纳假设 fib_eq n 和 fib_eq (n+1)。

noncomputable section

def phi  : ℝ := (1 + √5) / 2
def phi' : ℝ := (1 - √5) / 2

theorem fib_eq : ∀ n, fib n = (phi^n - phi'^n) / √5
  | 0   => by simp
  | 1   => by unfold fib; grind [phi, phi']
  | n+2 => by unfold fib; simp [fib_eq n, fib_eq (n+1), phi, phi']; grind

end

涉及 Fibonacci 函数的归纳证明并不一定都采用这种形式。下面复现 Mathlib 中证明相邻 Fibonacci 数互素的证明。

theorem fib_coprime_fib_succ (n : ℕ) : Nat.Coprime (fib n) (fib (n + 1)) := by
  induction n with
  | zero => simp
  | succ n ih =>
    simp only [fib, Nat.coprime_add_self_right]
    exact ih.symm

利用 Lean 的计算解释，可以求出 Fibonacci 数。

#eval fib 6
#eval List.range 20 |>.map fib

fib 的直接实现计算效率很低。事实上，它的运行时间关于参数呈指数增长。（你应当思考为什么。）在 Lean 中，可以实现如下尾递归版本，其运行时间关于 n 线性，并证明它计算同一个函数。

def fib' (n : Nat) : Nat :=
  aux n 0 1
where aux
  | 0,   x, _ => x
  | n+1, x, y => aux n y (x + y)

theorem fib'.aux_eq (m n : ℕ) : fib'.aux n (fib m) (fib (m + 1)) = fib (n + m) := by
  induction n generalizing m with
  | zero => simp [fib'.aux]
  | succ n ih => rw [fib'.aux, ←fib_add_two, ih, add_assoc, add_comm 1]

theorem fib'_eq_fib : fib' = fib := by
  ext n
  erw [fib', fib'.aux_eq 0 n]; rfl

#eval fib' 10000

注意 fib'.aux_eq 证明中的关键字 generalizing。它的作用是在归纳假设前插入 ∀ m，使得在归纳步骤中，m 可以取不同值。你可以逐步查看证明，并确认在这个例子中，归纳步骤需要把该量词实例化为 m + 1。

还要注意使用的是 erw（“extended rewrite”）而不是 rw。这是因为要重写目标 fib'.aux_eq，必须把 fib 0 和 fib 1 分别约简为 0 和 1。策略 erw 在展开定义以匹配参数方面比 rw 更积极。这并不总是好主意；在某些情况下它会浪费很多时间，因此应谨慎使用 erw。

下面是 generalizing 关键字在另一个恒等式证明中的例子，该恒等式也出现在 Mathlib 中。该恒等式的非形式证明可以在这里找到。我们提供形式证明的两个变体。

theorem fib_add (m n : ℕ) : fib (m + n + 1) = fib m * fib n + fib (m + 1) * fib (n + 1) := by
  induction n generalizing m with
  | zero => simp
  | succ n ih =>
    specialize ih (m + 1)
    rw [add_assoc m 1 n, add_comm 1 n] at ih
    simp only [fib_add_two, ih]
    ring

theorem fib_add' : ∀ m n, fib (m + n + 1) = fib m * fib n + fib (m + 1) * fib (n + 1)
  | _, 0     => by simp
  | m, n + 1 => by
    have := fib_add' (m + 1) n
    rw [add_assoc m 1 n, add_comm 1 n] at this
    simp only [fib_add_two, this]
    ring

作为练习，请使用 fib_add 证明如下命题。

example (n : ℕ): (fib n) ^ 2 + (fib (n + 1)) ^ 2 = fib (2 * n + 1) := by
  sorry

Lean 定义递归函数的机制足够灵活，可以允许任意递归调用，只要参数的复杂度按照某个良基度量下降。在下一个例子中，我们利用这样一个事实证明每个自然数 n ≠ 1 都有素因子：若 n 非零且不是素数，则它有一个更小的因子。（可以检查 Mathlib 在 Nat 命名空间中有同名定理，虽然证明不同于这里给出的证明。）

#check (@Nat.not_prime_iff_exists_dvd_lt :
  ∀ {n : ℕ}, 2 ≤ n → (¬Nat.Prime n ↔ ∃ m, m ∣ n ∧ 2 ≤ m ∧ m < n))

theorem ne_one_iff_exists_prime_dvd : ∀ {n}, n ≠ 1 ↔ ∃ p : ℕ, p.Prime ∧ p ∣ n
  | 0 => by simpa using Exists.intro 2 Nat.prime_two
  | 1 => by simp [Nat.not_prime_one]
  | n + 2 => by
    have hn : n + 2 ≠ 1 := by omega
    simp only [Ne, not_false_iff, true_iff, hn]
    by_cases h : Nat.Prime (n + 2)
    · use n + 2, h
    · have : 2 ≤ n + 2 := by omega
      rw [Nat.not_prime_iff_exists_dvd_lt this] at h
      rcases h with ⟨m, mdvdn, mge2, -⟩
      have : m ≠ 1 := by omega
      rw [ne_one_iff_exists_prime_dvd] at this
      rcases this with ⟨p, primep, pdvdm⟩
      use p, primep
      exact pdvdm.trans mdvdn

行 rw [ne_one_iff_exists_prime_dvd] at this 像魔术一样：我们在一个定理自己的证明中使用了正在证明的这个定理。它之所以有效，是因为递归调用实例化在 m 处，而当前情形是 n + 2，并且语境中有 m < n + 2。Lean 能找到这个假设，并用它说明归纳是良基的。Lean 很擅长判断什么在下降；在这个例子中，定理陈述中 n 的选择以及小于关系都很明显。在更复杂的情形中，Lean 提供机制让我们显式提供这些信息。参见 Lean Reference Manual 中关于良基递归的章节。

有时在证明中，需要根据自然数 n 是零还是后继数分情况讨论，但不需要在后继情形中获得归纳假设。为此，可以使用 cases 和 rcases 策略。

theorem zero_lt_of_mul_eq_one (m n : ℕ) : n * m = 1 → 0 < n ∧ 0 < m := by
  cases n <;> cases m <;> simp

example (m n : ℕ) : n*m = 1 → 0 < n ∧ 0 < m := by
  rcases m with (_ | m); simp
  rcases n with (_ | n) <;> simp

这是一个有用技巧。你常常会遇到关于自然数 n 的定理，其中零情形很容易。如果对 n 分情况并迅速处理零情形，剩下的就是把原目标中的 n 替换为 n + 1 后的目标。
