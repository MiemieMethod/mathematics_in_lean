4. 集合与函数

集合、关系和函数的词汇，为数学各分支中的构造提供了一种统一语言。由于函数和关系可以用集合来定义，公理化集合论可以作为数学的基础。

Lean 的基础则建立在类型这一原始概念之上，并包含定义类型之间函数的方法。Lean 中每个表达式都有类型：有自然数、实数、从实数到实数的函数、群、向量空间等等。有些表达式本身是类型，也就是说，它们的类型是 Type。Lean 和 Mathlib 提供了定义新类型的方法，也提供了定义这些类型中对象的方法。

从概念上说，可以把类型看作对象的集合。要求每个对象都有类型具有若干优点。例如，它使得像 + 这样的记号可以重载；并且有时可以减少输入的冗长程度，因为 Lean 能从对象的类型推断出大量信息。类型系统还使 Lean 能够在你把函数应用于错误数量的参数，或把函数应用于错误类型的参数时指出错误。

Lean 的库确实定义了初等集合论概念。与集合论不同，在 Lean 中，集合总是某个类型对象的集合，例如自然数集合，或从实数到实数的函数集合。类型与集合之间的区别需要一些时间适应，但本章会带你掌握其要点。

4.1. 集合

若 α 是任意类型，则类型 Set α 由 α 的元素组成的集合构成。这个类型支持通常的集合论运算和关系。例如，s ⊆ t 表示 s 是 t 的子集，s ∩ t 表示 s 与 t 的交，s ∪ t 表示它们的并。子集关系可输入为 \ss 或 \sub，交可输入为 \i 或 \cap，并可输入为 \un 或 \cup。库还定义了集合 univ，它包含类型 α 的所有元素；以及空集 ∅，可输入为 \empty。给定 x : α 和 s : Set α，表达式 x ∈ s 表示 x 是 s 的元素。提到集合成员关系的定理名中常包含 mem。表达式 x ∉ s 是 ¬ x ∈ s 的缩写。可以用 \in 或 \mem 输入 ∈，用 \notin 输入 ∉。

证明关于集合的命题的一种方法，是使用 rw 或化简器展开定义。在下面第二个例子中，我们使用 simp only，告诉化简器只使用我们给出的恒等式列表，而不使用其完整恒等式数据库。不同于 rw，simp 可以在全称或存在量词内部进行化简。逐步查看证明时，可以看到这些命令的效果。

variable {α : Type*}
variable (s t u : Set α)
open Set

example (h : s ⊆ t) : s ∩ u ⊆ t ∩ u := by
  rw [subset_def, inter_def, inter_def]
  rw [subset_def] at h
  simp only [mem_setOf]
  rintro x ⟨xs, xu⟩
  exact ⟨h _ xs, xu⟩

example (h : s ⊆ t) : s ∩ u ⊆ t ∩ u := by
  simp only [subset_def, mem_inter_iff] at *
  rintro x ⟨xs, xu⟩
  exact ⟨h _ xs, xu⟩

在这个例子中，我们打开 Set 命名空间，以便使用这些定理的较短名称。但事实上，可以完全删除对 rw 和 simp 的调用：

example (h : s ⊆ t) : s ∩ u ⊆ t ∩ u := by
  intro x xsu
  exact ⟨h xsu.1, xsu.2⟩

这里发生的事情称为定义约简：为了理解 intro 命令和匿名构造子，Lean 被迫展开定义。下面的例子也说明了这一现象：

example (h : s ⊆ t) : s ∩ u ⊆ t ∩ u :=
  fun _x ⟨xs, xu⟩ ↦ ⟨h xs, xu⟩

处理并集时，可以使用 Set.union_def 和 Set.mem_union。由于 x ∈ s ∪ t 展开为 x ∈ s ∨ x ∈ t，也可以使用 cases 策略迫使定义约简发生。

example : s ∩ (t ∪ u) ⊆ s ∩ t ∪ s ∩ u := by
  intro x hx
  have xs : x ∈ s := hx.1
  have xtu : x ∈ t ∪ u := hx.2
  rcases xtu with xt | xu
  · left
    show x ∈ s ∩ t
    exact ⟨xs, xt⟩
  · right
    show x ∈ s ∩ u
    exact ⟨xs, xu⟩

由于交的结合优先级高于并，表达式 (s ∩ t) ∪ (s ∩ u) 中的括号并非必要，但它们使表达式的含义更清楚。下面是同一事实的一个较短证明：

example : s ∩ (t ∪ u) ⊆ s ∩ t ∪ s ∩ u := by
  rintro x ⟨xs, xt | xu⟩
  · left; exact ⟨xs, xt⟩
  · right; exact ⟨xs, xu⟩

作为练习，请尝试证明另一个包含关系：

example : s ∩ t ∪ s ∩ u ⊆ s ∩ (t ∪ u) := by
  sorry

也许有帮助的是：使用 rintro 时，有时需要给析取模式 h1 | h2 加括号，才能让 Lean 正确解析它。

库还定义了集合差 s \ t，其中反斜杠是输入为 \\ 的特殊 Unicode 字符。表达式 x ∈ s \ t 展开为 x ∈ s ∧ x ∉ t。（∉ 可输入为 \notin。）可以用 Set.diff_eq 和 dsimp，或用 Set.mem_diff 手动重写它；但下面两个证明同一包含关系的证明展示了如何避免使用它们。

example : (s \ t) \ u ⊆ s \ (t ∪ u) := by
  intro x xstu
  have xs : x ∈ s := xstu.1.1
  have xnt : x ∉ t := xstu.1.2
  have xnu : x ∉ u := xstu.2
  constructor
  · exact xs
  intro xtu
  -- x ∈ t ∨ x ∈ u
  rcases xtu with xt | xu
  · show False; exact xnt xt
  · show False; exact xnu xu

example : (s \ t) \ u ⊆ s \ (t ∪ u) := by
  rintro x ⟨⟨xs, xnt⟩, xnu⟩
  use xs
  rintro (xt | xu) <;> contradiction

作为练习，请证明反向包含：

example : s \ (t ∪ u) ⊆ (s \ t) \ u := by
  sorry

要证明两个集合相等，只需证明一个集合的每个元素都是另一个集合的元素。这个原则称为“外延性”；不出意料，ext 策略正是为处理它而配备的。

example : s ∩ t = t ∩ s := by
  ext x
  simp only [mem_inter_iff]
  constructor
  · rintro ⟨xs, xt⟩; exact ⟨xt, xs⟩
  · rintro ⟨xt, xs⟩; exact ⟨xs, xt⟩

同样，删除 simp only [mem_inter_iff] 这一行并不会破坏证明。事实上，如果你喜欢难以读懂的证明项，下面这个单行证明就是为你准备的：

example : s ∩ t = t ∩ s :=
  Set.ext fun _x ↦ ⟨fun ⟨xs, xt⟩ ↦ ⟨xt, xs⟩, fun ⟨xt, xs⟩ ↦ ⟨xs, xt⟩⟩

下面是一个使用化简器的更短证明：

example : s ∩ t = t ∩ s := by ext x; simp [and_comm]

使用 ext 的另一种选择，是使用定理 Subset.antisymm；它允许我们通过证明 s ⊆ t 和 t ⊆ s，来证明集合等式 s = t。

example : s ∩ t = t ∩ s := by
  apply Subset.antisymm
  · rintro x ⟨xs, xt⟩; exact ⟨xt, xs⟩
  · rintro x ⟨xt, xs⟩; exact ⟨xs, xt⟩

请尝试完成这个证明项：

example : s ∩ t = t ∩ s :=
    Subset.antisymm sorry sorry

记住，可以把 sorry 替换为下划线；当你把鼠标悬停在它上面时，Lean 会显示此处期望什么。

下面是一些你或许会乐于证明的集合论恒等式：

example : s ∩ (s ∪ t) = s := by
  sorry

example : s ∪ s ∩ t = s := by
  sorry

example : s \ t ∪ t = s ∪ t := by
  sorry

example : s \ t ∪ t \ s = (s ∪ t) \ (s ∩ t) := by
  sorry

关于集合的表示，其底层机制如下。在类型论中，类型 α 上的性质或谓词只是函数 P : α → Prop。这很自然：给定 a : α，P a 正是断言 P 对 a 成立的命题。在库中，Set α 定义为 α → Prop，而 x ∈ s 定义为 s x。换句话说，集合其实就是性质，只是被当作对象来处理。

库还定义了集合构造式记号。表达式 { y | P y } 展开为 (fun y ↦ P y)，因此 x ∈ { y | P y } 约简为 P x。因此，我们可以把“为偶数”这一性质转换为偶数集合：

def evens : Set ℕ :=
  { n | Even n }

def odds : Set ℕ :=
  { n | ¬Even n }

example : evens ∪ odds = univ := by
  rw [evens, odds]
  ext n
  simp [-Nat.not_even_iff_odd]
  apply Classical.em

你应当逐步查看这个证明，确保理解发生了什么。注意，我们告诉化简器不要使用引理 Nat.not_even_iff，因为希望在目标中保留 ¬ Even n。请尝试删除 rw [evens, odds] 这一行，并确认证明仍然成立。

事实上，集合构造式记号用于定义

  • s ∩ t 定义为 {x | x ∈ s ∧ x ∈ t}，
  • s ∪ t 定义为 {x | x ∈ s ∨ x ∈ t}，
  • ∅ 定义为 {x | False}，并且
  • univ 定义为 {x | True}。

我们常常需要显式指出 ∅ 和 univ 的类型，因为 Lean 很难猜出我们指的是哪一个。下面的例子展示了必要时 Lean 如何展开最后两个定义。在第二个例子中，trivial 是库中 True 的标准证明。

example (x : ℕ) (h : x ∈ (∅ : Set ℕ)) : False :=
  h

example (x : ℕ) : x ∈ (univ : Set ℕ) :=
  trivial

作为练习，请证明下面的包含关系。使用 intro n 展开子集定义，并使用化简器把集合论构造化归为逻辑。我们还建议使用定理 Nat.Prime.eq_two_or_odd 和 Nat.odd_iff。

example : { n | Nat.Prime n } ∩ { n | n > 2 } ⊆ { n | ¬Even n } := by
  sorry

请小心：库中有多个版本的谓词 Prime，这有些容易混淆。最一般的版本在任意带零元的交换幺半群中都有意义。谓词 Nat.Prime 则专门用于自然数。幸运的是，有一个定理说明在这个特定情形下二者一致，因此总可以把一个重写为另一个。

#print Prime

#print Nat.Prime

example (n : ℕ) : Prime n ↔ Nat.Prime n :=
  Nat.prime_iff.symm

example (n : ℕ) (h : Prime n) : Nat.Prime n := by
  rw [Nat.prime_iff]
  exact h

rwa 策略先执行重写，然后执行 assumption 策略。

example (n : ℕ) (h : Prime n) : Nat.Prime n := by
  rwa [Nat.prime_iff]

Lean 引入记号 ∀ x ∈ s, ...，读作“对 s 中每个 x”，作为 ∀ x, x ∈ s → ... 的缩写。它也引入记号 ∃ x ∈ s, ...，读作“存在 s 中的 x 使得……”。这些有时称为有界量词，因为这种构造把量词的意义限制到集合 s 上。因此，库中使用它们的定理名常包含 ball 或 bex。定理 bex_def 断言 ∃ x ∈ s, ... 等价于 ∃ x, x ∈ s ∧ ...；但当它们与 rintro、use 和匿名构造子一起使用时，这两个表达式行为大致相同。因此，我们通常不需要显式用 bex_def 变换它们。下面是一些用法示例：

variable (s t : Set ℕ)

example (h₀ : ∀ x ∈ s, ¬Even x) (h₁ : ∀ x ∈ s, Prime x) : ∀ x ∈ s, ¬Even x ∧ Prime x := by
  intro x xs
  constructor
  · apply h₀ x xs
  apply h₁ x xs

example (h : ∃ x ∈ s, ¬Even x ∧ Prime x) : ∃ x ∈ s, Prime x := by
  rcases h with ⟨x, xs, _, prime_x⟩
  use x, xs

看看能否证明下面这些稍作变化的命题：

section
variable (ssubt : s ⊆ t)

example (h₀ : ∀ x ∈ t, ¬Even x) (h₁ : ∀ x ∈ t, Prime x) : ∀ x ∈ s, ¬Even x ∧ Prime x := by
  sorry

example (h : ∃ x ∈ s, ¬Even x ∧ Prime x) : ∃ x ∈ t, Prime x := by
  sorry

end

指标化并集和交集是另一类重要的集合论构造。可以把 α 的元素集合所成的序列 \(A_0, A_1, A_2, \ldots\) 表示为函数 A : ℕ → Set α；此时 ⋃ i, A i 表示它们的并，⋂ i, A i 表示它们的交。这里自然数没有特殊之处，ℕ 可以替换为任意用于给集合编号的类型 I。下面展示它们的用法。

variable {α I : Type*}
variable (A B : I → Set α)
variable (s : Set α)

open Set

example : (s ∩ ⋃ i, A i) = ⋃ i, A i ∩ s := by
  ext x
  simp only [mem_inter_iff, mem_iUnion]
  constructor
  · rintro ⟨xs, ⟨i, xAi⟩⟩
    exact ⟨i, xAi, xs⟩
  rintro ⟨i, xAi, xs⟩
  exact ⟨xs, ⟨i, xAi⟩⟩

example : (⋂ i, A i ∩ B i) = (⋂ i, A i) ∩ ⋂ i, B i := by
  ext x
  simp only [mem_inter_iff, mem_iInter]
  constructor
  · intro h
    constructor
    · intro i
      exact (h i).1
    intro i
    exact (h i).2
  rintro ⟨h1, h2⟩ i
  constructor
  · exact h1 i
  exact h2 i

指标化并集或交集经常需要括号，因为和量词一样，绑定变量的作用域会尽可能向右延伸。

请尝试证明下面的恒等式。其中一个方向需要经典逻辑！我们建议在证明的适当位置使用 by_cases xs : x ∈ s。

example : (s ∪ ⋂ i, A i) = ⋂ i, A i ∪ s := by
  sorry

Mathlib 也有有界并集和有界交集，它们类似于有界量词。可以用 mem_iUnion₂ 和 mem_iInter₂ 展开其含义。如下例所示，Lean 的化简器也会执行这些替换。

def primes : Set ℕ :=
  { x | Nat.Prime x }

example : (⋃ p ∈ primes, { x | p ^ 2 ∣ x }) = { x | ∃ p ∈ primes, p ^ 2 ∣ x } :=by
  ext
  rw [mem_iUnion₂]
  simp

example : (⋃ p ∈ primes, { x | p ^ 2 ∣ x }) = { x | ∃ p ∈ primes, p ^ 2 ∣ x } := by
  ext
  simp

example : (⋂ p ∈ primes, { x | ¬p ∣ x }) ⊆ { x | x = 1 } := by
  intro x
  contrapose!
  simp
  apply Nat.exists_prime_and_dvd

请尝试解决下面这个类似的例子。如果开始输入 eq_univ，Tab 补全会提示 apply eq_univ_of_forall 是一个好的证明开头。我们还建议使用定理 Nat.exists_infinite_primes。

example : (⋃ p ∈ primes, { x | x ≤ p }) = univ := by
  sorry

给定集合族 s : Set (Set α)，它们的并 ⋃₀ s 的类型是 Set α，并定义为 {x | ∃ t ∈ s, x ∈ t}。类似地，它们的交 ⋂₀ s 定义为 {x | ∀ t ∈ s, x ∈ t}。这些运算分别称为 sUnion 和 sInter。下面的例子展示它们与有界并集和有界交集的关系。

variable {α : Type*} (s : Set (Set α))

example : ⋃₀ s = ⋃ t ∈ s, t := by
  ext x
  rw [mem_iUnion₂]
  simp

example : ⋂₀ s = ⋂ t ∈ s, t := by
  ext x
  rw [mem_iInter₂]
  rfl

在库中，这些恒等式分别称为 sUnion_eq_biUnion 和 sInter_eq_biInter。

4.2. 函数

若 f : α → β 是函数，且 p 是类型 β 的元素组成的集合，则库定义 preimage f p，记作 f ⁻¹' p，为 {x | f x ∈ p}。表达式 x ∈ f ⁻¹' p 约简为 f x ∈ p。这常常很方便，如下面的例子所示：

variable {α β : Type*}
variable (f : α → β)
variable (s t : Set α)
variable (u v : Set β)

open Function
open Set

example : f ⁻¹' (u ∩ v) = f ⁻¹' u ∩ f ⁻¹' v := by
  ext
  rfl

若 s 是类型 α 的元素组成的集合，库还定义 image f s，记作 f '' s，为 {y | ∃ x, x ∈ s ∧ f x = y}。因此，假设 y ∈ f '' s 会分解为三元组 ⟨x, xs, xeq⟩，其中 x : α 满足假设 xs : x ∈ s 和 xeq : f x = y。rintro 策略中的 rfl 标记（见 第 3.2 节）正是为这类情形准备的。

example : f '' (s ∪ t) = f '' s ∪ f '' t := by
  ext y; constructor
  · rintro ⟨x, xs | xt, rfl⟩
    · left
      use x, xs
    right
    use x, xt
  rintro (⟨x, xs, rfl⟩ | ⟨x, xt, rfl⟩)
  · use x, Or.inl xs
  use x, Or.inr xt

还要注意，use 策略会在可能时应用 rfl 来关闭目标。

下面还有一个例子：

example : s ⊆ f ⁻¹' (f '' s) := by
  intro x xs
  show f x ∈ f '' s
  use x, xs

如果想使用专门为此目的设计的定理，可以把 use x, xs 这一行替换为 apply mem_image_of_mem f xs。不过，知道像集是以存在量词定义的，往往很方便。

下面这个等价命题是一个很好的练习：

example : f '' s ⊆ v ↔ s ⊆ f ⁻¹' v := by
  sorry

它表明 image f 和 preimage f 构成了 Set α 与 Set β 之间所谓 Galois 连接的一个实例，其中二者都按子集关系部分有序。在库中，这个等价命题名为 image_subset_iff。实践中，右侧通常是更有用的表示，因为 x ∈ f ⁻¹' t 会展开为 f x ∈ t，而处理 y ∈ f '' s 则需要拆开一个存在量词。

下面是一长串集合论恒等式，供你享用。不必一次全部完成；先做其中几个，其余可以留到有空时再做。

example (h : Injective f) : f ⁻¹' (f '' s) ⊆ s := by
  sorry

example : f '' (f ⁻¹' u) ⊆ u := by
  sorry

example (h : Surjective f) : u ⊆ f '' (f ⁻¹' u) := by
  sorry

example (h : s ⊆ t) : f '' s ⊆ f '' t := by
  sorry

example (h : u ⊆ v) : f ⁻¹' u ⊆ f ⁻¹' v := by
  sorry

example : f ⁻¹' (u ∪ v) = f ⁻¹' u ∪ f ⁻¹' v := by
  sorry

example : f '' (s ∩ t) ⊆ f '' s ∩ f '' t := by
  sorry

example (h : Injective f) : f '' s ∩ f '' t ⊆ f '' (s ∩ t) := by
  sorry

example : f '' s \ f '' t ⊆ f '' (s \ t) := by
  sorry

example : f ⁻¹' u \ f ⁻¹' v ⊆ f ⁻¹' (u \ v) := by
  sorry

example : f '' s ∩ v = f '' (s ∩ f ⁻¹' v) := by
  sorry

example : f '' (s ∩ f ⁻¹' u) ⊆ f '' s ∩ u := by
  sorry

example : s ∩ f ⁻¹' u ⊆ f ⁻¹' (f '' s ∩ u) := by
  sorry

example : s ∪ f ⁻¹' u ⊆ f ⁻¹' (f '' s ∪ u) := by
  sorry

也可以尝试下一组练习，它们刻画像和原像关于指标化并集与交集的行为。在第三个练习中，需要参数 i : I 来保证指标集非空。证明这些命题时，我们建议使用 ext 或 intro 展开集合之间等式或包含关系的含义，然后调用 simp 来拆解成员关系的条件。

variable {I : Type*} (A : I → Set α) (B : I → Set β)

example : (f '' ⋃ i, A i) = ⋃ i, f '' A i := by
  sorry

example : (f '' ⋂ i, A i) ⊆ ⋂ i, f '' A i := by
  sorry

example (i : I) (injf : Injective f) : (⋂ i, f '' A i) ⊆ f '' ⋂ i, A i := by
  sorry

example : (f ⁻¹' ⋃ i, B i) = ⋃ i, f ⁻¹' B i := by
  sorry

example : (f ⁻¹' ⋂ i, B i) = ⋂ i, f ⁻¹' B i := by
  sorry

库定义谓词 InjOn f s，表示 f 在 s 上是单射。它定义如下：

example : InjOn f s ↔ ∀ x₁ ∈ s, ∀ x₂ ∈ s, f x₁ = f x₂ → x₁ = x₂ :=
  Iff.refl _

命题 Injective f 可证明地等价于 InjOn f univ。类似地，库把 range f 定义为 {x | ∃y, f y = x}，因此 range f 可证明地等于 f '' univ。这是 Mathlib 中的一个常见主题：虽然函数的许多性质是相对于其完整定义域定义的，但常常也有把这些陈述限制到定义域类型的某个子集上的相对化版本。

下面是使用 InjOn 和 range 的一些例子：

open Set Real

example : InjOn log { x | x > 0 } := by
  intro x xpos y ypos e
  calc
    x = exp (log x) := by rw [exp_log xpos]
    _ = exp (log y) := by rw [e]
    _ = y := by rw [exp_log ypos]


example : range exp = { y | y > 0 } := by
  ext y; constructor
  · rintro ⟨x, rfl⟩
    apply exp_pos
  intro ypos
  use log y
  rw [exp_log ypos]

example : InjOn sqrt { x | x ≥ 0 } := by
  sorry

example : InjOn (fun x ↦ x ^ 2) { x : ℝ | x ≥ 0 } := by
  sorry

example : sqrt '' { x | x ≥ 0 } = { y | y ≥ 0 } := by
  sorry

example : (range fun x ↦ x ^ 2) = { y : ℝ | y ≥ 0 } := by
  sorry

要定义函数 f : α → β 的逆，我们将使用两个新成分。首先，需要处理 Lean 中任意类型可能为空这一事实。当不存在满足 f x = y 的 x 时，要定义 f 在 y 处的逆值，我们希望给 α 中指定一个默认值。把标注 [Inhabited α] 作为变量加入，就等同于假定 α 有一个首选元素，记作 default。其次，在有多个 x 满足 f x = y 的情况下，逆函数需要从中选择一个。这需要诉诸选择公理。Lean 允许以多种方式使用它；一种方便方法是使用经典的 choose 运算符，如下所示。

variable {α β : Type*} [Inhabited α]

#check (default : α)

variable (P : α → Prop) (h : ∃ x, P x)

#check Classical.choose h

example : P (Classical.choose h) :=
  Classical.choose_spec h

给定 h : ∃ x, P x，值 Classical.choose h 是某个满足 P x 的 x。定理 Classical.choose_spec h 说明 Classical.choose h 满足这个规范。

有了这些准备，就可以如下定义逆函数：

noncomputable section

open Classical

def inverse (f : α → β) : β → α := fun y : β ↦
  if h : ∃ x, f x = y then Classical.choose h else default

theorem inverse_spec {f : α → β} (y : β) (h : ∃ x, f x = y) : f (inverse f y) = y := by
  rw [inverse, dif_pos h]
  exact Classical.choose_spec h

这里需要 noncomputable section 和 open Classical 这两行，因为我们在本质上使用了经典逻辑。输入 y 时，函数 inverse f 在存在满足 f x = y 的 x 时返回某个这样的 x，否则返回 α 的默认元素。这是依赖式 if 构造的一个实例，因为在肯定情形下，返回值 Classical.choose h 依赖于假设 h。恒等式 dif_pos h 在给定 h : e 时，把 if h : e then a else b 重写为 a；类似地，dif_neg h 在给定 h : ¬ e 时把它重写为 b。还有适用于非依赖式 if 构造的版本 if_pos 和 if_neg，下一节会用到。定理 inverse_spec 说明 inverse f 满足该规范的第一部分。

如果没有完全理解这些机制，也不必担心。仅定理 inverse_spec 就应足以说明：inverse f 是左逆当且仅当 f 是单射，是右逆当且仅当 f 是满射。请在 VS Code 中双击或右击 LeftInverse 和 RightInverse 来查看它们的定义，或使用命令 #print LeftInverse 和 #print RightInverse。然后尝试证明两个定理。它们很有挑战！开始处理细节前，先在纸上完成证明会有帮助。每个证明大约可用半打短行完成。如果想额外挑战自己，可以尝试把每个证明压缩成一个单行证明项。

variable (f : α → β)

open Function

example : Injective f ↔ LeftInverse (inverse f) f :=
  sorry

example : Surjective f ↔ RightInverse (inverse f) f :=
  sorry

本节最后给出 Cantor 著名定理的类型论陈述：不存在从一个集合到其幂集的满射函数。看看你能否理解这个证明，然后补全缺失的两行。

theorem Cantor : ∀ f : α → Set α, ¬Surjective f := by
  intro f surjf
  let S := { i | i ∉ f i }
  rcases surjf S with ⟨j, h⟩
  have h₁ : j ∉ f j := by
    intro h'
    have : j ∉ f j := by rwa [h] at h'
    contradiction
  have h₂ : j ∈ S
  sorry
  have h₃ : j ∉ S
  sorry
  contradiction

4.3. 施罗德-伯恩斯坦定理

本章最后以集合论中的一个初等但非平凡的定理收尾。设 \(\alpha\) 和 \(\beta\) 为集合。（在我们的形式化中，它们实际上是类型。）假设 \(f : \alpha \to \beta\) 和 \(g : \beta \to \alpha\) 都是单射。直观地说，这意味着 \(\alpha\) 不大于 \(\beta\)，反之亦然。如果 \(\alpha\) 和 \(\beta\) 有限，这就蕴含它们有相同基数，等价地说，二者之间存在一个双射。十九世纪时，Cantor 陈述了即使在 \(\alpha\) 和 \(\beta\) 无限时同样成立的结果。后来 Dedekind、Schröder 和 Bernstein 分别独立证明了它。

我们的形式化将引入一些新方法，后续章节会更详细解释。如果这里讲得太快，也不必担心。我们的目标是让你看到，你已经具备为真实数学结果的形式化证明作出贡献的技能。

为了理解证明背后的思想，考虑映射 \(g\) 在 \(\alpha\) 中的像。在这个像上，\(g\) 的逆是有定义的，并且给出它与 \(\beta\) 的一个双射。

问题在于，如果 \(g\) 不是满射，这个双射没有覆盖图中阴影区域；该区域非空。另一种做法是用 \(f\) 把整个 \(\alpha\) 映到 \(\beta\)，但那样的问题是，如果 \(f\) 不是满射，就会遗漏 \(\beta\) 中的一些元素。

现在考虑从 \(\alpha\) 到自身的复合 \(g \circ f\)。因为该复合是单射，它在 \(\alpha\) 与其像之间形成一个双射，从而在 \(\alpha\) 内部得到一个缩小的 \(\alpha\) 副本。

这个复合把内侧阴影环映到另一个同类集合，我们可以把它看作更小的同心阴影环，如此继续。于是得到一个同心的阴影环序列，每个环都与下一个环双射对应。如果把每个环映到下一个环，并保持 \(\alpha\) 的非阴影部分不变，就得到 \(\alpha\) 与 \(g\) 的像之间的一个双射。再与 \(g^{-1}\) 复合，就得到所需的 \(\alpha\) 与 \(\beta\) 之间的双射。

可以更简单地描述这个双射。设 \(A\) 为阴影区域序列的并，并如下定义 \(h : \alpha \to \beta\)：

\[\begin{split}h(x) = \begin{cases} f(x) & \text{if $x \in A$} \\ g^{-1}(x) & \text{otherwise.} \end{cases}\end{split}\]

换言之，在阴影部分使用 \(f\)，在其他地方使用 \(g\) 的逆。所得映射 \(h\) 是单射，因为每个组成部分都是单射，且两个组成部分的像互不相交。要看它是满射，设给定 \(\beta\) 中的 \(y\)，并考虑 \(g(y)\)。若 \(g(y)\) 位于某个阴影区域，则它不能在第一个环中，所以有 \(g(y) = g(f(x))\)，其中 \(x\) 位于前一个环中。由 \(g\) 的单射性，得到 \(h(x) = f(x) = y\)。若 \(g(y)\) 不在阴影区域，则由 \(h\) 的定义有 \(h(g(y)) = y\)。无论哪种情况，\(y\) 都在 \(h\) 的像中。

这个论证听起来应当是可信的，但细节相当微妙。形式化证明不仅会增强我们对结果的信心，也会帮助我们更好地理解它。由于证明使用经典逻辑，我们告诉 Lean：我们的定义通常不是可计算的。

noncomputable section
open Classical
variable {α β : Type*} [Nonempty β]

标注 [Nonempty β] 指明 β 非空。我们使用它，是因为将要用来构造 \(g^{-1}\) 的 Mathlib 原语需要这一点。当 \(\beta\) 为空时，定理的情形是平凡的；虽然把形式化推广到涵盖该情形并不困难，但我们不费这个功夫。具体地说，我们需要假设 [Nonempty β] 来使用 Mathlib 中定义的运算 invFun。给定 x : α，invFun g x 在存在 x 在 β 中的原像时选择一个这样的原像，否则返回 β 的任意元素。若 g 是单射，则函数 invFun g 总是一个左逆；若 g 是满射，则它总是一个右逆。

#check (invFun g : α → β)
#check (leftInverse_invFun : Injective g → LeftInverse (invFun g) g)
#check (leftInverse_invFun : Injective g → ∀ y, invFun g (g y) = y)
#check (invFun_eq : (∃ y, g y = x) → g (invFun g x) = x)

我们如下定义与阴影区域之并对应的集合。

variable (f : α → β) (g : β → α)

def sbAux : ℕ → Set α
  | 0 => univ \ g '' univ
  | n + 1 => g '' (f '' sbAux n)

def sbSet :=
  ⋃ n, sbAux f g n

定义 sbAux 是递归定义的一个例子，我们将在下一章解释。它定义了一个集合序列

\[\begin{split}S_0 &= \alpha ∖ g(\beta) \\ S_{n+1} &= g(f(S_n)).\end{split}\]

定义 sbSet 对应证明草图中的集合 \(A = \bigcup_{n \in \mathbb{N}} S_n\)。上面描述的函数 \(h\) 现在定义如下：

def sbFun (x : α) : β :=
  if x ∈ sbSet f g then f x else invFun g x

我们将需要这样一个事实：我们对 \(g^{-1}\) 的定义，在 \(A\) 的补集上是右逆，也就是说，在 \(\alpha\) 的非阴影区域上是右逆。这是因为最外侧环 \(S_0\) 等于 \(\alpha \setminus g(\beta)\)，所以 \(A\) 的补集包含于 \(g(\beta)\)。因此，对补集中的每个 \(x\)，都存在 \(y\) 使得 \(g(y) = x\)。（由 \(g\) 的单射性，这个 \(y\) 是唯一的；但下一个定理只说明 invFun g x 返回某个满足 g y = x 的 y。）

请逐步查看下面的证明，确保理解其中发生了什么，并补全剩余部分。最后需要使用 invFun_eq。注意，这里用 sbAux 重写，会把 sbAux f g 0 替换为相应定义方程的右端。

theorem sb_right_inv {x : α} (hx : x ∉ sbSet f g) : g (invFun g x) = x := by
  have : x ∈ g '' univ := by
    contrapose! hx
    rw [sbSet, mem_iUnion]
    use 0
    rw [sbAux, mem_diff]
    sorry
  have : ∃ y, g y = x := by
    sorry
  sorry

现在转向证明 \(h\) 是单射。非形式地说，证明如下。首先，假设 \(h(x_1) = h(x_2)\)。若 \(x_1\) 在 \(A\) 中，则 \(h(x_1) = f(x_1)\)，并且可以如下证明 \(x_2\) 也在 \(A\) 中。若它不在，则 \(h(x_2) = g^{-1}(x_2)\)。由 \(f(x_1) = h(x_1) = h(x_2)\) 得到 \(g(f(x_1)) = x_2\)。根据 \(A\) 的定义，由于 \(x_1\) 在 \(A\) 中，\(x_2\) 也在 \(A\) 中，矛盾。因此，若 \(x_1\) 在 \(A\) 中，则 \(x_2\) 也在；此时有 \(f(x_1) = h(x_1) = h(x_2) = f(x_2)\)。由 \(f\) 的单射性推出 \(x_1 = x_2\)。对称论证表明，若 \(x_2\) 在 \(A\) 中，则 \(x_1\) 也在，从而同样推出 \(x_1 = x_2\)。

唯一剩下的可能性是 \(x_1\) 与 \(x_2\) 都不在 \(A\) 中。在这种情况下，有 \(g^{-1}(x_1) = h(x_1) = h(x_2) = g^{-1}(x_2)\)。对等式两边应用 \(g\)，得到 \(x_1 = x_2\)。

我们再次鼓励你逐步查看下面的证明，看看这个论证如何在 Lean 中展开。看看能否使用 sb_right_inv 完成证明。

theorem sb_injective (hf : Injective f) : Injective (sbFun f g) := by
  set A := sbSet f g with A_def
  set h := sbFun f g with h_def
  intro x₁ x₂ (hxeq : h x₁ = h x₂)
  show x₁ = x₂
  simp only [h_def, sbFun, ← A_def] at hxeq
  by_cases xA : x₁ ∈ A ∨ x₂ ∈ A
  · wlog x₁A : x₁ ∈ A generalizing x₁ x₂ hxeq xA
    · symm
      apply this hxeq.symm xA.symm (xA.resolve_left x₁A)
    have x₂A : x₂ ∈ A := by
      apply _root_.not_imp_self.mp
      intro (x₂nA : x₂ ∉ A)
      rw [if_pos x₁A, if_neg x₂nA] at hxeq
      rw [A_def, sbSet, mem_iUnion] at x₁A
      have x₂eq : x₂ = g (f x₁) := by
        sorry
      rcases x₁A with ⟨n, hn⟩
      rw [A_def, sbSet, mem_iUnion]
      use n + 1
      simp [sbAux]
      exact ⟨x₁, hn, x₂eq.symm⟩
    sorry
  push_neg at xA
  sorry

这个证明引入了一些新策略。首先，注意 set 策略，它为 sbSet f g 和 sb_fun f g 分别引入缩写 A 和 h。我们把相应的定义方程命名为 A_def 和 h_def。这些缩写是定义性的，也就是说，必要时 Lean 有时会自动展开它们。但并非总是如此；例如使用 rw 时，通常需要显式使用 A_def 和 h_def。因此，这些定义带来一种权衡：它们可以使表达式更短、更可读，但有时也要求我们做更多工作。

更有趣的策略是 wlog，它封装了上面非形式证明中的对称性论证。我们现在不详细讨论它，但请注意它恰好完成了我们想要的事情。若把鼠标悬停在该策略上，可以查看它的文档。

满射性的证明更容易。给定 \(\beta\) 中的 \(y\)，我们根据 \(g(y)\) 是否在 \(A\) 中分两种情况。如果在，它不可能位于最外侧环 \(S_0\)，因为按定义该环与 \(g\) 的像不相交。因此，它对某个 \(n\) 是 \(S_{n+1}\) 的元素。这意味着它形如 \(g(f(x))\)，其中 \(x\) 在 \(S_n\) 中。由 \(g\) 的单射性，得到 \(f(x) = y\)。若 \(g(y)\) 在 \(A\) 的补集中，则立即有 \(h(g(y)) = y\)，证明完成。

我们再次鼓励你逐步查看证明并补全缺失部分。策略 rcases n with _ | n 会把情况分为 g y ∈ sbAux f g 0 和 g y ∈ sbAux f g (n + 1)。在两种情形中，调用 simp [sbAux] 都会应用 sbAux 的相应定义方程。

theorem sb_surjective (hg : Injective g) : Surjective (sbFun f g) := by
  set A := sbSet f g with A_def
  set h := sbFun f g with h_def
  intro y
  by_cases gyA : g y ∈ A
  · rw [A_def, sbSet, mem_iUnion] at gyA
    rcases gyA with ⟨n, hn⟩
    rcases n with _ | n
    · simp [sbAux] at hn
    simp [sbAux] at hn
    rcases hn with ⟨x, xmem, hx⟩
    use x
    have : x ∈ A := by
      rw [A_def, sbSet, mem_iUnion]
      exact ⟨n, xmem⟩
    rw [h_def, sbFun, if_pos this]
    apply hg hx

  sorry

现在可以把一切组合起来。最终陈述简短漂亮，而证明使用了 Bijective h 展开为 Injective h ∧ Surjective h 这一事实。

theorem schroeder_bernstein {f : α → β} {g : β → α} (hf : Injective f) (hg : Injective g) :
    ∃ h : α → β, Bijective h :=
  ⟨sbFun f g, sb_injective f g hf, sb_surjective f g hg⟩
