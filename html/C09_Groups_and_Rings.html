9. 群与环

我们在Section 2.2中看到如何对群和环中的运算进行推理。随后在Section 7.2中，我们看到如何定义抽象代数结构，例如群结构，以及高斯整数上的环结构这类具体实例。Chapter 8解释了 Mathlib 如何处理抽象结构的层级。

本章将更详细地处理群与环。我们无法涵盖 Mathlib 中关于这些主题的所有内容，尤其是 Mathlib 还在不断增长。不过，我们会提供进入库的入口，并展示基本概念如何使用。这里与Chapter 8的讨论有一些重叠，但我们将重点放在如何使用 Mathlib，而不是这些主题处理方式背后的设计决策。因此，理解某些例子可能需要回顾Chapter 8的背景。

9.1. 幺半群与群

9.1.1. 幺半群及其态射

抽象代数课程常常从群开始，然后推进到环、域和向量空间。讨论环上的乘法时，这会带来一些扭曲，因为乘法运算并非来自群结构，但许多证明可以从群论逐字搬到这个新情境中。用纸笔做数学时，最常见的补救办法是把这些证明留作练习。一种效率稍低但更安全、也更适合形式化的做法，是使用monoid。类型 M 上的幺半群结构是一个内部合成律，满足结合律并有单位元。幺半群主要用于同时容纳群和环的乘法结构。不过也有许多自然例子；例如，自然数集合配备加法后形成幺半群。

从实践角度看，使用 Mathlib 时大多可以忽略幺半群。但在浏览 Mathlib 文件寻找引理时，需要知道它们存在。否则，你可能会在群论文件中寻找某个陈述，而它其实在幺半群相关文件中，因为它并不要求元素可逆。

类型 M 上幺半群结构的类型写作 Monoid M。函数 Monoid 是类型类，因此几乎总会以实例隐式参数出现（换言之，在方括号中）。默认情况下，Monoid 对运算使用乘法记号；若要使用加法记号，则用 AddMonoid。这些结构的交换版本会在 Monoid 前加前缀 Comm。

example {M : Type*} [Monoid M] (x : M) : x * 1 = x := mul_one x

example {M : Type*} [AddCommMonoid M] (x y : M) : x + y = y + x := add_comm x y

注意，虽然库中有 AddMonoid，但对非交换运算使用加法记号通常会造成混淆。

M 与 N 之间幺半群态射的类型称为 MonoidHom M N，记作 M →* N。当把这样的态射应用到 M 的元素上时，Lean 会自动把它看作从 M 到 N 的函数。加法版本称为 AddMonoidHom，记作 M →+ N。

example {M N : Type*} [Monoid M] [Monoid N] (x y : M) (f : M →* N) : f (x * y) = f x * f y :=
  f.map_mul x y

example {M N : Type*} [AddMonoid M] [AddMonoid N] (f : M →+ N) : f 0 = 0 :=
  f.map_zero

这些态射是打包映射，也就是说，它们把一个映射及其若干性质打包在一起。记住，Section 8.2解释了打包映射；这里我们只指出一个略显遗憾的后果：不能用普通函数复合来复合这些映射。相反，需要使用 MonoidHom.comp 和 AddMonoidHom.comp。

example {M N P : Type*} [AddMonoid M] [AddMonoid N] [AddMonoid P]
    (f : M →+ N) (g : N →+ P) : M →+ P := g.comp f

9.1.2. 群及其态射

关于群，我们还有更多内容要说；群就是额外满足每个元素都有逆元这一性质的幺半群。

example {G : Type*} [Group G] (x : G) : x * x⁻¹ = 1 := mul_inv_cancel x

类似于前面见过的 ring 策略，也有一个 group 策略可以证明任意群中成立的恒等式。（等价地，它证明自由群中成立的恒等式。）

example {G : Type*} [Group G] (x y z : G) : x * (y * z) * (x * z)⁻¹ * (x * y * x⁻¹)⁻¹ = 1 := by
  group

还有一个用于交换加法群恒等式的策略，称为 abel。

example {G : Type*} [AddCommGroup G] (x y z : G) : z + x + (y - z - x) = y := by
  abel

有趣的是，群态射不过是群之间的幺半群态射。因此，可以复制粘贴前面的一个例子，把 Monoid 替换为 Group。

example {G H : Type*} [Group G] [Group H] (x y : G) (f : G →* H) : f (x * y) = f x * f y :=
  f.map_mul x y

当然，我们确实会得到一些新性质，例如下面这个：

example {G H : Type*} [Group G] [Group H] (x : G) (f : G →* H) : f (x⁻¹) = (f x)⁻¹ :=
  f.map_inv x

你可能担心，构造群态射会要求做不必要的工作，因为幺半群态射的定义要求单位元映到单位元，而对群态射来说这是自动的。实践中，额外工作并不困难；不过，为避免它，有一个函数可以从群之间与合成律相容的函数构造群态射。

example {G H : Type*} [Group G] [Group H] (f : G → H) (h : ∀ x y, f (x * y) = f x * f y) :
    G →* H :=
  MonoidHom.mk' f h

还有一个群（或幺半群）同构类型 MulEquiv，记作 ≃*（加法记号中对应 AddEquiv，记作 ≃+）。f : G ≃* H 的逆是 MulEquiv.symm f : H ≃* G，f 和 g 的复合是 MulEquiv.trans f g，G 的恒等同构是 M̀ulEquiv.refl G。使用匿名投影记号，前两者可分别写作 f.symm 和 f.trans g。必要时，该类型的元素会自动强制转换为态射和函数。

example {G H : Type*} [Group G] [Group H] (f : G ≃* H) :
    f.trans f.symm = MulEquiv.refl G :=
  f.self_trans_symm

可以使用 MulEquiv.ofBijective 从双射态射构造同构。这样做会使逆函数不可计算。

noncomputable example {G H : Type*} [Group G] [Group H]
    (f : G →* H) (h : Function.Bijective f) :
    G ≃* H :=
  MulEquiv.ofBijective f h

9.1.3. 子群

正如群态射是打包的，G 的子群也是打包结构，由 G 中的一个集合及相关闭包性质组成。

example {G : Type*} [Group G] (H : Subgroup G) {x y : G} (hx : x ∈ H) (hy : y ∈ H) :
    x * y ∈ H :=
  H.mul_mem hx hy

example {G : Type*} [Group G] (H : Subgroup G) {x : G} (hx : x ∈ H) :
    x⁻¹ ∈ H :=
  H.inv_mem hx

在上面的例子中，重要的是要理解 Subgroup G 是 G 的子群类型，而不是谓词 IsSubgroup H，其中 H 是 Set G 的元素。Subgroup G 配备了到 Set G 的强制转换，以及 G 上的成员谓词。关于如何以及为何这样做，可参见Section 8.3。

当然，两个子群相同当且仅当它们有相同的元素。这个事实已注册给 ext 策略使用；它可以像证明两个集合相等一样，用来证明两个子群相等。

例如，要陈述并证明 ℤ 是 ℚ 的加法子群，我们真正想做的是构造一个类型为 AddSubgroup ℚ 的项，其到 Set ℚ 的投影为 ℤ，或者更准确地说，是 ℤ 在 ℚ 中的像。

example : AddSubgroup ℚ where
  carrier := Set.range ((↑) : ℤ → ℚ)
  add_mem' := by
    rintro _ _ ⟨n, rfl⟩ ⟨m, rfl⟩
    use n + m
    simp
  zero_mem' := by
    use 0
    simp
  neg_mem' := by
    rintro _ ⟨n, rfl⟩
    use -n
    simp

借助类型类，Mathlib 知道群的子群继承群结构。

example {G : Type*} [Group G] (H : Subgroup G) : Group H := inferInstance

这个例子有些微妙。对象 H 不是类型，但 Lean 会把它解释为 G 的子类型，从而自动强制转换为类型。因此，上面的例子可以更显式地重述为：

example {G : Type*} [Group G] (H : Subgroup G) : Group {x : G // x ∈ H} := inferInstance

拥有类型 Subgroup G 而不是谓词 IsSubgroup : Set G → Prop 的一个重要好处是，可以很容易地给 Subgroup G 赋予额外结构。尤其重要的是，相对于包含关系，它具有完备格结构。例如，我们不是有一个说明 G 的两个子群之交仍为子群的引理，而是使用格运算 ⊓ 来构造这个交。随后可以把关于格的任意引理应用到该构造上。

让我们检查，两个子群下确界的底层集合按定义确实就是它们的交。

example {G : Type*} [Group G] (H H' : Subgroup G) :
    ((H ⊓ H' : Subgroup G) : Set G) = (H : Set G) ∩ (H' : Set G) := rfl

对底层集合的交使用不同记号可能看起来奇怪，但这种对应关系不能迁移到上确界运算和集合并上，因为子群的并通常不是子群。相反，需要使用由并生成的子群，这通过 Subgroup.closure 完成。

example {G : Type*} [Group G] (H H' : Subgroup G) :
    ((H ⊔ H' : Subgroup G) : Set G) = Subgroup.closure ((H : Set G) ∪ (H' : Set G)) := by
  rw [Subgroup.sup_eq_closure]

另一个微妙点是，G 本身不具有类型 Subgroup G，所以需要一种方式谈论把 G 看成 G 的子群。这同样由格结构提供：全子群就是该格的顶元素。

example {G : Type*} [Group G] (x : G) : x ∈ (⊤ : Subgroup G) := trivial

类似地，该格的底元素是唯一元素为单位元的子群。

example {G : Type*} [Group G] (x : G) : x ∈ (⊥ : Subgroup G) ↔ x = 1 := Subgroup.mem_bot

作为操作群和子群的练习，可以定义子群被母群中一个元素共轭后的结果。

def conjugate {G : Type*} [Group G] (x : G) (H : Subgroup G) : Subgroup G where
  carrier := {a : G | ∃ h, h ∈ H ∧ a = x * h * x⁻¹}
  one_mem' := by
    dsimp
    sorry
  inv_mem' := by
    dsimp
    sorry
  mul_mem' := by
    dsimp
    sorry

把前两个主题联系起来，可以使用群态射对子群进行推出和拉回。Mathlib 中的命名约定是把这些操作称为 map 和 comap。这些不是常见的数学术语，但优点是比“推出”和“直接像”更短。

example {G H : Type*} [Group G] [Group H] (G' : Subgroup G) (f : G →* H) : Subgroup H :=
  Subgroup.map f G'

example {G H : Type*} [Group G] [Group H] (H' : Subgroup H) (f : G →* H) : Subgroup G :=
  Subgroup.comap f H'

#check Subgroup.mem_map
#check Subgroup.mem_comap

特别地，底子群在态射 f 下的原像是一个子群，称为 f 的kernel；而 f 的值域也是一个子群。

example {G H : Type*} [Group G] [Group H] (f : G →* H) (g : G) :
    g ∈ MonoidHom.ker f ↔ f g = 1 :=
  f.mem_ker

example {G H : Type*} [Group G] [Group H] (f : G →* H) (h : H) :
    h ∈ MonoidHom.range f ↔ ∃ g : G, f g = h :=
  f.mem_range

作为操作群态射和子群的练习，让我们证明一些初等性质。它们在 Mathlib 中已经被证明；若想从这些练习中获益，不要太快使用 exact?。

section exercises
variable {G H : Type*} [Group G] [Group H]

open Subgroup

example (φ : G →* H) (S T : Subgroup H) (hST : S ≤ T) : comap φ S ≤ comap φ T := by
  sorry

example (φ : G →* H) (S T : Subgroup G) (hST : S ≤ T) : map φ S ≤ map φ T := by
  sorry

variable {K : Type*} [Group K]

-- Remember you can use the `ext` tactic to prove an equality of subgroups.
example (φ : G →* H) (ψ : H →* K) (U : Subgroup K) :
    comap (ψ.comp φ) U = comap φ (comap ψ U) := by
  sorry

-- Pushing a subgroup along one homomorphism and then another is equal to
-- pushing it forward along the composite of the homomorphisms.
example (φ : G →* H) (ψ : H →* K) (S : Subgroup G) :
    map (ψ.comp φ) S = map ψ (S.map φ) := by
  sorry

end exercises

让我们用两个非常经典的结果结束对 Mathlib 中子群的介绍。拉格朗日定理说明，有限群的子群基数整除该群的基数。Sylow 第一定理是拉格朗日定理的一个著名部分逆命题。

虽然 Mathlib 的这一角落部分设置为允许计算，我们仍可使用下面的 open scoped 命令告诉 Lean 使用非构造逻辑。

open scoped Classical


example {G : Type*} [Group G] (G' : Subgroup G) : Nat.card G' ∣ Nat.card G :=
  ⟨G'.index, mul_comm G'.index _ ▸ G'.index_mul_card.symm⟩

open Subgroup

example {G : Type*} [Group G] [Finite G] (p : ℕ) {n : ℕ} [Fact p.Prime]
    (hdvd : p ^ n ∣ Nat.card G) : ∃ K : Subgroup G, Nat.card K = p ^ n :=
  Sylow.exists_subgroup_card_pow_prime p hdvd

接下来的两个练习推出拉格朗日引理的一个推论。（这同样已经在 Mathlib 中，所以不要太快使用 exact?。）

lemma eq_bot_iff_card {G : Type*} [Group G] {H : Subgroup G} :
    H = ⊥ ↔ Nat.card H = 1 := by
  suffices (∀ x ∈ H, x = 1) ↔ ∃ x ∈ H, ∀ a ∈ H, a = x by
    simpa [eq_bot_iff_forall, Nat.card_eq_one_iff_exists]
  sorry

#check card_dvd_of_le

lemma inf_bot_of_coprime {G : Type*} [Group G] (H K : Subgroup G)
    (h : (Nat.card H).Coprime (Nat.card K)) : H ⊓ K = ⊥ := by
  sorry

9.1.4. 具体群

也可以在 Mathlib 中操作具体群，尽管这通常比使用抽象理论更复杂。例如，给定任意类型 X，X 的置换群是 Equiv.Perm X。特别地，对称群 \(\mathfrak{S}_n\) 是 Equiv.Perm (Fin n)。可以陈述关于该群的抽象结果，例如说若 X 有限，则 Equiv.Perm X 由轮换生成。

open Equiv

example {X : Type*} [Finite X] : Subgroup.closure {σ : Perm X | Perm.IsCycle σ} = ⊤ :=
  Perm.closure_isCycle

也可以完全具体地计算轮换的实际乘积。下面使用命令 #simp，它会在给定表达式上调用 simp 策略。记号 c[] 用于定义循环置换。在例子中，结果是 ℕ 的一个置换。可以在出现的第一个数字上使用类型标注，例如 (1 : Fin 5)，使其成为 Perm (Fin 5) 中的计算。

#simp [mul_assoc] c[1, 2, 3] * c[2, 3, 4]

处理具体群的另一种方式是使用自由群和群表示。类型 α 上的自由群是 FreeGroup α，包含映射是 FreeGroup.of : α → FreeGroup α。例如，定义一个有三个元素的类型 S，这些元素记为 a、b 和 c，并定义相应自由群中的元素 ab⁻¹。

section FreeGroup

inductive S | a | b | c

open S

def myElement : FreeGroup S := (.of a) * (.of b)⁻¹

注意，我们给出了定义的预期类型，使 Lean 知道 .of 表示 FreeGroup.of。

自由群的泛性质体现为等价 FreeGroup.lift。例如，定义一个从 FreeGroup S 到 Perm (Fin 5) 的群态射，它把 a 送到 c[1, 2, 3]，把 b 送到 c[2, 3, 1]，并把 c 送到 c[2, 3]。

def myMorphism : FreeGroup S →* Perm (Fin 5) :=
  FreeGroup.lift fun | .a => c[1, 2, 3]
                     | .b => c[2, 3, 1]
                     | .c => c[2, 3]

作为最后一个具体例子，来看如何定义由单个元素生成且其三次方为一的群（因此该群同构于 \(\mathbb{Z}/3\)），并构造从该群到 Perm (Fin 5) 的态射。

作为恰有一个元素的类型，我们将使用 Unit，其唯一元素记作 ()。函数 PresentedGroup 接受一个关系集合，即某个自由群元素的集合，并返回该自由群对由这些关系生成的正规子群取商得到的群。（我们将在Section 9.1.6中看到如何处理更一般的商。）由于我们以某种方式把这件事隐藏在定义背后，所以使用 deriving Group 强制在 myGroup 上创建一个群实例。

def myGroup := PresentedGroup {.of () ^ 3} deriving Group

表示群的泛性质保证，从该群出发的态射可以由把关系送到目标群单位元的函数构造。因此需要这样一个函数，以及该条件成立的证明。然后可以把这个证明传给 PresentedGroup.toGroup，得到所需的群态射。

def myMap : Unit → Perm (Fin 5)
| () => c[1, 2, 3]

lemma compat_myMap :
    ∀ r ∈ ({.of () ^ 3} : Set (FreeGroup Unit)), FreeGroup.lift myMap r = 1 := by
  rintro _ rfl
  simp
  decide

def myNewMorphism : myGroup →* Perm (Fin 5) := PresentedGroup.toGroup compat_myMap

end FreeGroup

9.1.5. 群作用

群论与数学其他部分相互作用的一种重要方式是使用群作用。群 G 在某个类型 X 上的作用，不过是从 G 到 Equiv.Perm X 的态射。因此，从某种意义上说，群作用已经被前面的讨论覆盖了。但我们不希望随身携带这个态射；相反，希望它尽可能由 Lean 自动推断。因此有一个类型类 MulAction G X。这种设置的缺点是，同一个群在同一个类型上有多个作用时，需要一些迂回办法，例如定义类型同义词，并为每个同义词携带不同的类型类实例。

特别地，这允许我们用 g • x 表示群元素 g 对点 x 的作用。

noncomputable section GroupActions

example {G X : Type*} [Group G] [MulAction G X] (g g': G) (x : X) :
    g • (g' • x) = (g * g') • x :=
  (mul_smul g g' x).symm

还有用于加法群的版本，称为 AddAction，其中作用记作 +ᵥ。例如，它用于仿射空间的定义。

example {G X : Type*} [AddGroup G] [AddAction G X] (g g' : G) (x : X) :
    g +ᵥ (g' +ᵥ x) = (g + g') +ᵥ x :=
  (add_vadd g g' x).symm

底层群态射称为 MulAction.toPermHom。

open MulAction

example {G X : Type*} [Group G] [MulAction G X] : G →* Equiv.Perm X :=
  toPermHom G X

作为说明，来看如何定义 Cayley 同构嵌入：把任意群 G 嵌入置换群，即 Perm G。为了实例化 [MulAction G G]，Lean 会找到由左乘给出的标准群作用，这正是我们想要的。

def CayleyIsoMorphism (G : Type*) [Group G] : G ≃* (toPermHom G G).range :=
  Equiv.Perm.subgroupOfMulAction G G

注意，在上面的定义之前，没有任何内容真正要求拥有群而非幺半群（事实上，只要类型配备乘法运算即可）。

当我们想把 X 分解为轨道时，群条件才真正进入图景。X 上相应的等价关系称为 MulAction.orbitRel。它没有被声明为全局实例。

example {G X : Type*} [Group G] [MulAction G X] : Setoid X := orbitRel G X

利用这一点，可以陈述 X 在 G 的作用下被划分为轨道。更准确地说，我们得到 X 与依赖积 (ω : orbitRel.Quotient G X) × (orbit G (Quotient.out' ω)) 之间的双射，其中 Quotient.out' ω 只是选择一个投影到 ω 的元素。回忆一下，这个依赖积的元素是对 ⟨ω, x⟩，其中 x 的类型 orbit G (Quotient.out' ω) 依赖于 ω。

example {G X : Type*} [Group G] [MulAction G X] :
    X ≃ (ω : orbitRel.Quotient G X) × (orbit G (Quotient.out ω)) :=
  MulAction.selfEquivSigmaOrbits G X

特别地，当 X 有限时，可以把它与 Fintype.card_congr 和 Fintype.card_sigma 结合起来，推出 X 的基数等于各轨道基数之和。此外，轨道与 G 在稳定子左平移作用下的商之间双射。子群通过左平移的这个作用被用于定义群对子群的商，记号为 /，所以可以使用下面这个简洁陈述。

example {G X : Type*} [Group G] [MulAction G X] (x : X) :
    orbit G x ≃ G ⧸ stabilizer G x :=
  MulAction.orbitEquivQuotientStabilizer G x

结合上述两个结果的一个重要特例，是 X 为群 G，并配备子群 H 通过平移给出的作用。在这种情况下，所有稳定子都是平凡的，所以每个轨道都与 H 双射，于是得到：

example {G : Type*} [Group G] (H : Subgroup G) : G ≃ (G ⧸ H) × H :=
  groupEquivQuotientProdSubgroup

这是上面所见拉格朗日定理版本的概念性变体。注意，这个版本不作有限性假设。

作为本节练习，请使用前一个练习中定义的 conjugate，构造群在其子群上由共轭给出的作用。

variable {G : Type*} [Group G]

lemma conjugate_one (H : Subgroup G) : conjugate 1 H = H := by
  sorry

instance : MulAction G (Subgroup G) where
  smul := conjugate
  one_smul := by
    sorry
  mul_smul := by
    sorry

end GroupActions

9.1.6. 商群

在上面关于子群作用于群的讨论中，我们看到了商 G ⧸ H。一般而言，它只是一个类型。当且仅当 H 是正规子群时，它才能被赋予群结构，使商映射成为群态射（而该群结构随后是唯一的）。

正规性假设是一个类型类 Subgroup.Normal，因此类型类推断可以使用它来导出商上的群结构。

noncomputable section QuotientGroup

example {G : Type*} [Group G] (H : Subgroup G) [H.Normal] : Group (G ⧸ H) := inferInstance

example {G : Type*} [Group G] (H : Subgroup G) [H.Normal] : G →* G ⧸ H :=
  QuotientGroup.mk' H

商群的泛性质通过 QuotientGroup.lift 使用：只要群态射 φ 的核包含 N，它就下降到 G ⧸ N。

example {G : Type*} [Group G] (N : Subgroup G) [N.Normal] {M : Type*}
    [Group M] (φ : G →* M) (h : N ≤ MonoidHom.ker φ) : G ⧸ N →* M :=
  QuotientGroup.lift N φ h

上面片段中目标群称为 M，这暗示在 M 上有幺半群结构其实已经足够。

一个重要特例是 N = ker φ。在这种情况下，下降得到的态射是单射，并得到到其像上的群同构。这个结果通常称为第一同构定理。

example {G : Type*} [Group G] {M : Type*} [Group M] (φ : G →* M) :
    G ⧸ MonoidHom.ker φ →* MonoidHom.range φ :=
  QuotientGroup.quotientKerEquivRange φ

把泛性质应用到态射 φ : G →* G' 与商群投影 Quotient.mk' N' 的复合，也可以得到从 G ⧸ N 到 G' ⧸ N' 的态射。对 φ 所需的条件通常表述为“φ 应把 N 映入 N'”。但这等价于要求 φ 把 N' 拉回到 N 之上；后一条件更便于使用，因为拉回的定义不含存在量词。

example {G G': Type*} [Group G] [Group G']
    {N : Subgroup G} [N.Normal] {N' : Subgroup G'} [N'.Normal]
    {φ : G →* G'} (h : N ≤ Subgroup.comap φ N') : G ⧸ N →* G' ⧸ N':=
  QuotientGroup.map N N' φ h

需要记住一个微妙点：类型 G ⧸ N 确实依赖于 N（直至定义相等），所以有一个证明说明两个正规子群 N 和 M 相等，并不足以使相应商类型按定义相等。不过，在这种情况下，泛性质确实给出一个同构。

example {G : Type*} [Group G] {M N : Subgroup G} [M.Normal]
    [N.Normal] (h : M = N) : G ⧸ M ≃* G ⧸ N := QuotientGroup.quotientMulEquivOfEq h

作为本节最后一组练习，我们将证明：若有限群 G 的正规子群 H 与 K 互不相交，并且它们基数的乘积等于 G 的基数，则 G 同构于 H × K。回忆，在此语境中“互不相交”表示 H ⊓ K = ⊥。

首先稍微操作一下拉格朗日引理，此时不假设子群正规或互不相交。

9.2. 环

9.2.1. 环、单位、态射与子环

类型 R 上环结构的类型是 Ring R。假设乘法可交换的变体是 CommRing R。我们已经看到，策略 ring 会证明任意由交换环公理推出的等式。

example {R : Type*} [CommRing R] (x y : R) : (x + y) ^ 2 = x ^ 2 + y ^ 2 + 2 * x * y := by ring

更特殊的变体不要求 R 上的加法形成群，只要求它形成加法幺半群。相应的类型类是 Semiring R 和 CommSemiring R。自然数类型是 CommSemiring R 的一个重要实例，任何取值于自然数的函数类型也是如此。另一个重要例子是环中理想的类型，稍后会讨论。策略 ring 的名称有双重误导性，因为它假设交换性，却也能在半环中工作。换言之，它适用于任意 CommSemiring。

example (x y : ℕ) : (x + y) ^ 2 = x ^ 2 + y ^ 2 + 2 * x * y := by ring

环和半环类还有一些版本，不假设乘法单位元存在，也不假设乘法满足结合律。这里不讨论这些版本。

一些传统上在环论入门中讲授的概念，实际上只涉及底层乘法幺半群。一个突出例子是环的单位的定义。每个（乘法）幺半群 M 都有一个谓词 IsUnit : M → Prop，断言双侧逆元存在；有一个单位类型 Units M，记作 Mˣ；并有到 M 的强制转换。类型 Units M 把一个可逆元素、它的逆，以及保证二者确实互为逆的性质打包在一起。这个实现细节主要在定义可计算函数时相关。在大多数情形中，可以用 IsUnit.unit {x : M} : IsUnit x → Mˣ 构造单位。在交换情形中，还有 Units.mkOfMulEqOne (x y : M) : x * y = 1 → Mˣ，它把 x 视为单位来构造。

example (x : ℤˣ) : x = 1 ∨ x = -1 := Int.units_eq_one_or x

example {M : Type*} [Monoid M] (x : Mˣ) : (x : M) * x⁻¹ = 1 := Units.mul_inv x

example {M : Type*} [Monoid M] : Group Mˣ := inferInstance

两个（半）环 R 与 S 之间环态射的类型是 RingHom R S，记作 R →+* S。

example {R S : Type*} [Ring R] [Ring S] (f : R →+* S) (x y : R) :
    f (x + y) = f x + f y := f.map_add x y

example {R S : Type*} [Ring R] [Ring S] (f : R →+* S) : Rˣ →* Sˣ :=
  Units.map f

同构版本是 RingEquiv，记号为 ≃+*。

与子幺半群和子群一样，对环 R 的子环也有类型 Subring R；但这个类型远不如子群类型有用，因为不能用子环对环取商。

example {R : Type*} [Ring R] (S : Subring R) : Ring S := inferInstance

还要注意，RingHom.range 产生一个子环。

9.2.2. 理想与商

由于历史原因，Mathlib 只对交换环有理想理论。（环库最初是为快速推进现代代数几何基础而开发的。）因此，本节将使用交换（半）环。R 的理想被定义为把 R 看作 R-模时的子模。模将在后面的线性代数章节中介绍，但这个实现细节通常可以安全忽略，因为大多数（但不是全部）相关引理都在理想这一特殊语境中重新陈述。不过，匿名投影记号并不总会按预期工作。例如，下面片段中不能把 Ideal.Quotient.mk I 替换为 I.Quotient.mk，因为其中有两个 .，所以它会被解析为 (Ideal.Quotient I).mk；但 Ideal.Quotient 本身并不存在。

example {R : Type*} [CommRing R] (I : Ideal R) : R →+* R ⧸ I :=
  Ideal.Quotient.mk I

example {R : Type*} [CommRing R] {a : R} {I : Ideal R} :
    Ideal.Quotient.mk I a = 0 ↔ a ∈ I :=
  Ideal.Quotient.eq_zero_iff_mem

商环的泛性质是 Ideal.Quotient.lift。

example {R S : Type*} [CommRing R] [CommRing S] (I : Ideal R) (f : R →+* S)
    (H : I ≤ RingHom.ker f) : R ⧸ I →+* S :=
  Ideal.Quotient.lift I f H

特别地，它导出环的第一同构定理。

example {R S : Type*} [CommRing R] [CommRing S](f : R →+* S) :
    R ⧸ RingHom.ker f ≃+* f.range :=
  RingHom.quotientKerEquivRange f

理想相对于包含关系形成完备格结构，同时也形成半环结构。这两个结构之间相互配合良好。

variable {R : Type*} [CommRing R] {I J : Ideal R}

example : I + J = I ⊔ J := rfl

example {x : R} : x ∈ I + J ↔ ∃ a ∈ I, ∃ b ∈ J, a + b = x := by
  simp [Submodule.mem_sup]

example : I * J ≤ J := Ideal.mul_le_left

example : I * J ≤ I := Ideal.mul_le_right

example : I * J ≤ I ⊓ J := Ideal.mul_le_inf

可以使用环态射分别通过 Ideal.map 和 Ideal.comap 推出和拉回理想。照常，后者更方便使用，因为它不涉及存在量词。这解释了为什么在陈述允许构造商环之间态射的条件时使用它。

example {R S : Type*} [CommRing R] [CommRing S] (I : Ideal R) (J : Ideal S) (f : R →+* S)
    (H : I ≤ Ideal.comap f J) : R ⧸ I →+* S ⧸ J :=
  Ideal.quotientMap J f H

一个微妙点是，类型 R ⧸ I 确实依赖于 I（直至定义相等），所以有一个证明说明两个理想 I 和 J 相等，并不足以使相应商类型按定义相等。不过，泛性质在这种情况下确实提供一个同构。

example {R : Type*} [CommRing R] {I J : Ideal R} (h : I = J) : R ⧸ I ≃+* R ⧸ J :=
  Ideal.quotEquivOfEq h

现在可以把中国剩余同构作为例子给出。注意带指标下确界符号 ⨅ 与类型的大积符号 Π 之间的区别。根据字体不同，它们可能很难区分。

example {R : Type*} [CommRing R] {ι : Type*} [Fintype ι] (f : ι → Ideal R)
    (hf : ∀ i j, i ≠ j → IsCoprime (f i) (f j)) : (R ⧸ ⨅ i, f i) ≃+* Π i, R ⧸ f i :=
  Ideal.quotientInfRingEquivPiQuotient f hf

中国剩余定理的初等版本，即关于 ZMod 的陈述，可以很容易从前一个定理推出：

open BigOperators PiNotation

example {ι : Type*} [Fintype ι] (a : ι → ℕ) (coprime : ∀ i j, i ≠ j → (a i).Coprime (a j)) :
    ZMod (∏ i, a i) ≃+* Π i, ZMod (a i) :=
  ZMod.prodEquivPi a coprime

作为一组练习，我们将在一般情形中重新证明中国剩余定理。

首先，需要使用商环的泛性质，把定理中出现的映射定义为一个环态射。

theorem isCoprime_Inf {I : Ideal R} {J : ι → Ideal R} {s : Finset ι}
    (hf : ∀ j ∈ s, IsCoprime I (J j)) : IsCoprime I (⨅ j ∈ s, J j) := by
  classical
  simp_rw [isCoprime_iff_add] at *
  induction s using Finset.induction with
  | empty =>
      simp
  | @insert i s _ hs =>
      rw [Finset.iInf_insert, inf_comm, one_eq_top, eq_top_iff, ← one_eq_top]
      set K := ⨅ j ∈ s, J j
      calc
        1 = I + K                  := sorry
        _ = I + K * (I + J i)      := sorry
        _ = (1 + K) * I + K * J i  := sorry
        _ ≤ I + K ⊓ J i            := sorry

lemma chineseMap_surj [Fintype ι] {I : ι → Ideal R}

  intro g
  choose f hf using fun i ↦ Ideal.Quotient.mk_surjective (g i)

      sorry
    sorry
  choose e he using key
  use mk _ (∑ i, f i * e i)
  sorry

9.2.3. 代数与多项式

给定交换（半）环 R，一个algebra over R 的代数，是一个半环 A，并配备从 R 到 A 的环态射，且该态射的像与 A 的每个元素交换。这被编码为类型类 Algebra R A。从 R 到 A 的态射称为结构映射，在 Lean 中记作 algebraMap R A : R →+* A。对某个 r : R，用 algebraMap R A r 乘以 a : A，称为 a 由 r 作标量乘法，记作 r • a。注意，这里的代数概念有时称为associative unital algebra，以强调存在更一般的代数概念。

algebraMap R A 是环态射这一事实，把许多关于标量乘法的性质打包在一起，例如下面这些：

example {R A : Type*} [CommRing R] [Ring A] [Algebra R A] (r r' : R) (a : A) :
    (r + r') • a = r • a + r' • a :=
  add_smul r r' a

example {R A : Type*} [CommRing R] [Ring A] [Algebra R A] (r r' : R) (a : A) :
    (r * r') • a = r • r' • a :=
  mul_smul r r' a

两个 R-代数 A 与 B 之间的态射，是与 R 中元素的标量乘法交换的环态射。它们是打包态射，类型为 AlgHom R A B，记作 A →ₐ[R] B。

非交换代数的重要例子包括自同态代数和方阵代数，二者都会在线性代数章节中涉及。本章将讨论一个最重要的交换代数例子，即多项式代数。

以 R 为系数的一元多项式代数称为 Polynomial R；一旦打开 Polynomial 命名空间，它就可以写成 R[X]。从 R 到 R[X] 的代数结构映射记作 C，表示“常数”，因为相应的多项式函数总是常函数。不定元记作 X。

open Polynomial

example {R : Type*} [CommRing R] : R[X] := X

example {R : Type*} [CommRing R] (r : R) := X - C r

在上面第一个例子中，给 Lean 预期类型至关重要，因为无法从定义体确定它。在第二个例子中，由于 r 的类型已知，目标多项式代数可以从我们使用 C r 推断出来。

因为 C 是从 R 到 R[X] 的环态射，所以在 R[X] 中计算之前，可以使用所有环态射引理，例如 map_zero、map_one、map_mul 和 map_pow。例如：

example {R : Type*} [CommRing R] (r : R) : (X + C r) * (X - C r) = X ^ 2 - C (r ^ 2) := by
  rw [C.map_pow]
  ring

可以使用 Polynomial.coeff 访问系数。

example {R : Type*} [CommRing R] (r:R) : (C r).coeff 0 = r := by simp

example {R : Type*} [CommRing R] : (X ^ 2 + 2 * X + C 3 : R[X]).coeff 1 = 2 := by simp

定义多项式次数总是棘手，因为零多项式是特殊情形。Mathlib 有两个变体：Polynomial.natDegree : R[X] → ℕ 把零多项式的次数指定为 0，而 Polynomial.degree : R[X] → WithBot ℕ 指定为 ⊥。后者中，WithBot ℕ 可以看作 ℕ ∪ {-∞}，只是 -∞ 记为 ⊥，这与完备格中底元素使用同一符号。这个特殊值用作零多项式的次数，并且对加法具有吸收性。（它对乘法几乎也具有吸收性，例外是 ⊥ * 0 = 0。）

从数学本质上说，degree 版本是正确的。例如，它允许我们陈述乘积次数的预期公式（假设基环没有零因子）。

example {R : Type*} [Semiring R] [NoZeroDivisors R] {p q : R[X]} :
    degree (p * q) = degree p + degree q :=
  Polynomial.degree_mul

而 natDegree 版本则需要假设多项式非零。

example {R : Type*} [Semiring R] [NoZeroDivisors R] {p q : R[X]} (hp : p ≠ 0) (hq : q ≠ 0) :
    natDegree (p * q) = natDegree p + natDegree q :=
  Polynomial.natDegree_mul hp hq

然而，ℕ 比 WithBot ℕ 好用得多，所以 Mathlib 同时提供两个版本，并提供在二者之间转换的引理。此外，在计算复合的次数时，natDegree 是更方便的定义。多项式的复合是 Polynomial.comp，并且有：

example {R : Type*} [Semiring R] [NoZeroDivisors R] {p q : R[X]} :
    natDegree (comp p q) = natDegree p * natDegree q :=
  Polynomial.natDegree_comp

多项式给出多项式函数：任意多项式都可以使用 Polynomial.eval 在 R 上求值。

example {R : Type*} [CommRing R] (P: R[X]) (x : R) := P.eval x

example {R : Type*} [CommRing R] (r : R) : (X - C r).eval r = 0 := by simp

特别地，有一个谓词 IsRoot，它对 R 中使多项式为零的元素 r 成立。

example {R : Type*} [CommRing R] (P : R[X]) (r : R) : IsRoot P r ↔ P.eval r = 0 := Iff.rfl

我们希望说，若 R 没有零因子，则一个多项式的根数（按重数计）至多为其次数。但零多项式的情形再次令人棘手。因此，Mathlib 定义 Polynomial.roots，把多项式 P 送到一个多重集：若 P 为零，则该有限集定义为空；否则为 P 的根，带重数。这个定义只在底层环为整环时可用，因为否则该定义没有良好性质。

example {R : Type*} [CommRing R] [IsDomain R] (r : R) : (X - C r).roots = {r} :=
  roots_X_sub_C r

example {R : Type*} [CommRing R] [IsDomain R] (r : R) (n : ℕ):
    ((X - C r) ^ n).roots = n • {r} :=
  by simp

Polynomial.eval 和 Polynomial.roots 都只考虑系数环。它们无法表达 X ^ 2 - 2 : ℚ[X] 在 ℝ 中有根，或 X ^ 2 + 1 : ℝ[X] 在 ℂ 中有根。为此，需要 Polynomial.aeval，它会在任意 R-代数中对 P : R[X] 求值。更精确地说，给定一个半环 A 和一个 Algebra R A 实例，Polynomial.aeval 把每个元素 a : A 送到 R-代数态射 R[X] → A，该态射把 X 求值为 a。由于 AlgHom 有到函数的强制转换，可以把它应用到多项式上。但 aeval 不以多项式作为参数，所以不能像上面的 P.eval 一样使用点记号。

example : aeval Complex.I (X ^ 2 + 1 : ℝ[X]) = 0 := by simp

在此语境中，与 roots 对应的函数是 aroots，它接受一个多项式，再接受一个代数，并输出一个多重集（对于零多项式有与 roots 相同的注意事项）。

open Complex Polynomial

example : aroots (X ^ 2 + 1 : ℝ[X]) ℂ = {Complex.I, -I} := by
  suffices roots (X ^ 2 + 1 : ℂ[X]) = {I, -I} by simpa [aroots_def]
  have factored : (X ^ 2 + 1 : ℂ[X]) = (X - C I) * (X - C (-I)) := by
    have key : (C I * C I : ℂ[X]) = -1 := by simp [← C_mul]
    rw [C_neg]
    linear_combination key
  have p_ne_zero : (X - C I) * (X - C (-I)) ≠ 0 := by
    intro H
    apply_fun eval 0 at H
    simp [eval] at H
  simp only [factored, roots_mul p_ne_zero, roots_X_sub_C]
  rfl

-- Mathlib knows about D'Alembert-Gauss theorem: ``ℂ`` is algebraically closed.
example : IsAlgClosed ℂ := inferInstance

更一般地，给定环态射 f : R →+* S，可以用 Polynomial.eval₂ 在 S 中的一点对 P : R[X] 求值。它产生从 R[X] 到 S 的实际函数，因为它不假设存在 Algebra R S 实例，所以点记号会按预期工作。

#check (Complex.ofRealHom : ℝ →+* ℂ)

example : (X ^ 2 + 1 : ℝ[X]).eval₂ Complex.ofRealHom Complex.I = 0 := by simp

最后简要提一下多元多项式。给定交换半环 R，以 R 为系数、由类型 σ 索引不定元的 R-代数是 MVPolynomial σ R。给定 i : σ，相应多项式是 MvPolynomial.X i。（照常，可以打开 MVPolynomial 命名空间，把它缩写为 X i。）例如，如果想要两个不定元，可以用 Fin 2 作为 σ，并把定义 \(\mathbb{R}^2`\) 中单位圆的多项式写作：

open MvPolynomial

def circleEquation : MvPolynomial (Fin 2) ℝ := X 0 ^ 2 + X 1 ^ 2 - 1

回忆，函数应用具有很高优先级，因此上面的表达式读作 (X 0) ^ 2 + (X 1) ^ 2 - 1。可以对它求值，以确认坐标为 \((1, 0)\) 的点在圆上。回忆，记号 ![...] 表示 Fin n → X 的元素，其中自然数 n 由参数个数决定，而类型 X 由参数类型决定。

example : MvPolynomial.eval ![1, 0] circleEquation = 0 := by simp [circleEquation]
