6. 离散数学

Discrete Mathematics研究有限集合、对象与结构。我们可以计数有限集合的元素，可以计算其元素上的有限和或有限积，可以计算最大值与最小值，等等。我们还可以研究由某些生成函数经有限次应用而生成的对象，可以用结构递归定义函数，并用结构归纳证明定理。本章介绍 Mathlib 中支持这些工作的若干部分。

6.1. 有限集与有限类型

在 Mathlib 中处理有限集合与有限类型可能令人困惑，因为库提供了多种处理方式。本节将讨论其中最常用的几种。

我们已经在Section 5.2和Section 5.3中见过类型 Finset。顾名思义，类型 Finset α 的一个元素就是由类型 α 的元素构成的有限集。我们称这类对象为“有限集”（finset）。数据类型 Finset 被设计为具有计算解释；Finset α 上许多基本操作都假定 α 具有可判定相等性，这保证存在一种算法可检验 a : α 是否为有限集 s 的元素。

section
variable {α : Type*} [DecidableEq α] (a : α) (s t : Finset α)

#check a ∈ s
#check s ∩ t

end

如果移除声明 [DecidableEq α]，Lean 会在 #check s ∩ t 这一行报错，因为它无法计算交。尽管如此，所有你期望能够进行计算的数据类型都有可判定相等性；而若通过打开 Classical 命名空间并声明 noncomputable section 来以经典方式工作，就可以对任意类型的元素所成有限集进行推理。

有限集支持集合所支持的大多数集合论操作：

open Finset

variable (a b c : Finset ℕ)
variable (n : ℕ)

#check a ∩ b
#check a ∪ b
#check a \ b
#check (∅ : Finset ℕ)

example : a ∩ (b ∪ c) = (a ∩ b) ∪ (a ∩ c) := by
  ext x; simp only [mem_inter, mem_union]; tauto

example : a ∩ (b ∪ c) = (a ∩ b) ∪ (a ∩ c) := by rw [inter_union_distrib_left]

注意，我们已经打开了 Finset 命名空间，其中包含有限集专用的定理。如果逐步查看下面最后一个例子，会看到先应用 ext 再应用 simp 会把该恒等式化归为命题逻辑问题。作为练习，你可以尝试把Chapter 4中的一些集合恒等式迁移到有限集上来证明。

你已经见过记号 Finset.range n，它表示自然数的有限集 \(\{ 0, 1, \ldots, n-1 \}\)。Finset 还允许通过列举元素来定义有限集：

#check ({0, 2, 5} : Finset Nat)

def example1 : Finset ℕ := {0, 1, 2}

有多种方法可以让 Lean 识别：以这种方式给出的集合中，元素的顺序和重复出现并不重要。

example : ({0, 1, 2} : Finset ℕ) = {1, 2, 0} := by decide

example : ({0, 1, 2} : Finset ℕ) = {0, 1, 1, 2} := by decide

example : ({0, 1} : Finset ℕ) = {1, 0} := by rw [Finset.pair_comm]

example (x : Nat) : ({x, x} : Finset ℕ) = {x} := by simp

example (x y z : Nat) : ({x, y, z, y, z, x} : Finset ℕ) = {x, y, z} := by
  ext i; simp [or_comm, or_left_comm]

example (x y z : Nat) : ({x, y, z, y, z, x} : Finset ℕ) = {x, y, z} := by
  ext i; simp; tauto

可以使用 insert 向一个有限集中加入单个元素，也可以使用 Finset.erase 删除单个元素。注意，erase 位于 Finset 命名空间中，而 insert 位于根命名空间中。

example (s : Finset ℕ) (a : ℕ) (h : a ∉ s) : (insert a s |>.erase a) = s :=
  Finset.erase_insert h

example (s : Finset ℕ) (a : ℕ) (h : a ∈ s) : insert a (s.erase a) = s :=
  Finset.insert_erase h

事实上，{0, 1, 2} 只是 insert 0 (insert 1 (singleton 2)) 的记号。

set_option pp.notation false in
#check ({0, 1, 2} : Finset ℕ)

给定有限集 s 和谓词 P，可以用集合构造记号 {x ∈ s | P x} 定义 s 中满足 P 的元素所成集合。这是 Finset.filter P s 的记号，也可以写作 s.filter P。

example : {m ∈ range n | Even m} = (range n).filter Even := rfl
example : {m ∈ range n | Even m ∧ m ≠ 3} = (range n).filter (fun m ↦ Even m ∧ m ≠ 3) := rfl

example : {m ∈ range 10 | Even m} = {0, 2, 4, 6, 8} := by decide

Mathlib 知道，有限集在函数下的像仍然是有限集。

#check (range 5).image (fun x ↦ x * 2)

example : (range 5).image (fun x ↦ x * 2) = {x ∈ range 10 | Even x} := by decide

Lean 也知道两个有限集的笛卡儿积 s ×ˢ t 是有限集，并且有限集的幂集也是有限集。（注意，记号 s ×ˢ t 对集合同样适用。）

#check s ×ˢ t
#check s.powerset

按有限集的元素来定义操作是微妙的，因为任何这样的定义都必须与元素给出的顺序无关。当然，总可以通过复合已有操作来定义函数。另一种做法是使用 Finset.fold 将一个二元运算折叠到各个元素上，前提是该运算满足结合律和交换律；这些性质保证结果与应用运算的顺序无关。有限和、有限积以及有限并正是以这种方式定义的。在下面最后一个例子中，biUnion 表示“有界指标并”。用通常的数学记号，该表达式可写为 \(\bigcup_{i ∈ s} g(i)\)。

#check Finset.fold

def f (n : ℕ) : Int := (↑n)^2

#check (range 5).fold (fun x y : Int ↦ x + y) 0 f
#eval (range 5).fold (fun x y : Int ↦ x + y) 0 f

#check ∑ i ∈ range 5, i^2
#check ∏ i ∈ range 5, i + 1

variable (g : Nat → Finset Int)

#check (range 5).biUnion g

有限集上有一个自然的归纳原则：要证明每个有限集都具有某性质，只需证明空集具有该性质，并证明当向一个有限集加入一个新元素时该性质得以保持。（下一个例子的归纳步骤中需要在 @insert 里使用符号 @，以便给参数 a 和 s 命名，因为它们已被标记为隐式参数。）

#check Finset.induction

example {α : Type*} [DecidableEq α] (f : α → ℕ)  (s : Finset α) (h : ∀ x ∈ s, f x ≠ 0) :
    ∏ x ∈ s, f x ≠ 0 := by
  induction s using Finset.induction_on with
  | empty => simp
  | @insert a s anins ih =>
    rw [prod_insert anins]
    apply mul_ne_zero
    · apply h; apply mem_insert_self
    apply ih
    intros x xs
    exact h x (mem_insert_of_mem xs)

若 s 是有限集，则 Finset.Nonempty s 被定义为 ∃ x, x ∈ s。可以使用经典选择从非空有限集中选取一个元素。类似地，库定义了 Finset.toList s，它使用选择以某种顺序选取 s 的元素。

noncomputable example (s : Finset ℕ) (h : s.Nonempty) : ℕ := Classical.choose h

example (s : Finset ℕ) (h : s.Nonempty) : Classical.choose h ∈ s := Classical.choose_spec h

noncomputable example (s : Finset ℕ) : List ℕ := s.toList

example (s : Finset ℕ) (a : ℕ) : a ∈ s.toList ↔ a ∈ s := mem_toList

可以使用 Finset.min 和 Finset.max 从线性序中元素构成的有限集中选取最小或最大元素；类似地，也可以对格中元素构成的有限集使用 Finset.inf 和 Finset.sup。但这里有一个细节：空有限集的最小元素应当是什么？你可以检查下面这些函数的带撇号版本，它们增加了有限集非空这一前提。无撇号版本 Finset.min 和 Finset.max 分别向输出类型加入一个顶元素或底元素，用来处理有限集为空的情形。无撇号版本 Finset.inf 和 Finset.sup 则分别假定该格配备了顶元素或底元素。

#check Finset.min
#check Finset.min'
#check Finset.max
#check Finset.max'
#check Finset.inf
#check Finset.inf'
#check Finset.sup
#check Finset.sup'

example : Finset.Nonempty {2, 6, 7} := ⟨6, by trivial⟩
example : Finset.min' {2, 6, 7} ⟨6, by trivial⟩ = 2 := by trivial

每个有限集 s 都有一个有限基数 Finset.card s；当 Finset 命名空间打开时，它可写作 #s。

#check Finset.card

#eval (range 5).card

example (s : Finset ℕ) : s.card = #s := by rfl

example (s : Finset ℕ) : s.card = ∑ _i ∈ s, 1 := by rw [card_eq_sum_ones]

example (s : Finset ℕ) : s.card = ∑ _i ∈ s, 1 := by simp

下一节将专门讨论关于基数的推理。

在形式化数学时，常常必须决定是用集合还是用类型来表述定义和定理。使用类型往往能简化记号和证明，但使用某个类型的子集则可能更灵活。有限集的类型化对应物是fintype，即某个 α 上的类型 Fintype α。按照定义，有限类型只是一个配备了有限集 univ 的数据类型，而该有限集包含其全部元素。

variable {α : Type*} [Fintype α]

example : ∀ x : α, x ∈ Finset.univ := by
  intro x; exact mem_univ x

Fintype.card α 等于相应有限集的基数。

example : Fintype.card α = (Finset.univ : Finset α).card := rfl

我们已经见过有限类型的一个典型例子，即对每个 n 给出的类型 Fin n。Lean 能够识别有限类型在诸如积这样的操作下封闭。

example : Fintype.card (Fin 5) = 5 := by simp
example : Fintype.card ((Fin 5) × (Fin 3)) = 15 := by simp

Finset α 的任意元素 s 都可以强制转换为一个类型 (↑s : Type α)，即 α 中属于 s 的元素所成的子类型。此外，Lean 知道 ↑s 是有限类型。

variable (s : Finset ℕ)

example : (↑s : Type) = {x : ℕ // x ∈ s} := rfl
example : Fintype.card ↑s = s.card := by simp

Lean 和 Mathlib 使用type class inference来跟踪有限类型上的额外结构，也就是包含所有元素的全体有限集。换言之，可以把有限类型看作配备了这项额外数据的代数结构。Chapter 7将说明这一机制如何工作。

6.2. 计数论证

计数的技巧是组合数学的核心组成部分。Mathlib 含有若干用于计数有限集元素的基本恒等式：

open Finset

variable {α β : Type*} [DecidableEq α] [DecidableEq β] (s t : Finset α) (f : α → β)

example : #(s ×ˢ t) = #s * #t := by rw [card_product]
example : #(s ×ˢ t) = #s * #t := by simp

example : #(s ∪ t) = #s + #t - #(s ∩ t) := by rw [card_union]

example (h : Disjoint s t) : #(s ∪ t) = #s + #t := by rw [card_union_of_disjoint h]
example (h : Disjoint s t) : #(s ∪ t) = #s + #t := by simp [h]

example (h : Function.Injective f) : #(s.image f) = #s := by rw [card_image_of_injective _ h]

example (h : Set.InjOn f s) : #(s.image f) = #s := by rw [card_image_of_injOn h]

打开 Finset 命名空间后，可以使用记号 #s 表示 s.card，也可以使用 card_union 等缩短后的名称。

Mathlib 也可以计数有限类型的元素：

open Fintype

variable {α β : Type*} [Fintype α] [Fintype β]

example : card (α × β) = card α * card β := by simp

example : card (α ⊕ β) = card α + card β := by simp

example (n : ℕ) : card (Fin n → α) = (card α)^n := by simp

variable {n : ℕ} {γ : Fin n → Type*} [∀ i, Fintype (γ i)]

example : card ((i : Fin n) → γ i) = ∏ i, card (γ i) := by simp

example : card (Σ i, γ i) = ∑ i, card (γ i) := by simp

当 Fintype 命名空间未打开时，必须使用 Fintype.card 而不是 card。

下面是一个计算有限集基数的例子：计算 range n 与其一个副本的并，其中这个副本被平移了超过 n 的距离。该计算要求证明并中的两个集合不交；证明的第一行产生旁条件 Disjoint (range n) (image (fun i ↦ m + i) (range n))，它在证明末尾得到证明。谓词 Disjoint 对我们来说过于一般，不能直接使用，但定理 disjoint_iff_ne 会把它化为可用的形式。

#check Disjoint

example (m n : ℕ) (h : m ≥ n) :
    card (range n ∪ (range n).image (fun i ↦ m + i)) = 2 * n := by
  rw [card_union_of_disjoint, card_range, card_image_of_injective, card_range]; omega
  . apply add_right_injective
  . simp [disjoint_iff_ne]; omega

在本节中，omega 将是处理算术计算与不等式的主力工具。

下面是一个更有趣的例子。考虑 \(\{0, \ldots, n\} \times \{0, \ldots, n\}\) 中由满足 \(i < j\) 的点 \((i, j)\) 构成的子集。如果把它们看成坐标平面中的格点，它们构成以 \((0, 0)\) 和 \((n, n)\) 为顶点的正方形的上三角部分，但不包括对角线。整个正方形的基数为 \((n + 1)^2\)；减去对角线的大小后再除以二，可知该三角形的基数为 \(n (n + 1) / 2\)。

另一种做法是注意该三角形各行的大小为 \(0, 1, \ldots, n\)，因此其基数就是前 \(n\) 个正整数之和。下面证明中的第一个 have 把三角形描述为各行之并，其中第 \(j\) 行由数 \(0, 1, ..., j - 1\) 与 \(j\) 配对而成。在下面的证明中，记号 (., j) 是函数 fun i ↦ (i, j) 的缩写。证明的其余部分只是关于有限集基数的计算。

def triangle (n : ℕ) : Finset (ℕ × ℕ) := {p ∈ range (n+1) ×ˢ range (n+1) | p.1 < p.2}

example (n : ℕ) : #(triangle n) = (n + 1) * n / 2 := by
  have : triangle n = (range (n+1)).biUnion (fun j ↦ (range j).image (., j)) := by
    ext p
    simp only [triangle, mem_filter, mem_product, mem_range, mem_biUnion, mem_image]
    constructor
    . rintro ⟨⟨hp1, hp2⟩, hp3⟩
      use p.2, hp2, p.1, hp3
    . rintro ⟨p1, hp1, p2, hp2, rfl⟩
      omega
  rw [this, card_biUnion]; swap
  · -- take care of disjointness first
    intro x _ y _ xney
    simp [disjoint_iff_ne, xney]
  -- continue the calculation
  transitivity (∑ i ∈ range (n + 1), i)
  · congr; ext i
    rw [card_image_of_injective, card_range]
    intros i1 i2; simp
  rw [sum_range_id]; rfl

下面的证明变体使用有限类型而非有限集来完成计算。类型 α ≃ β 是 α 与 β 之间等价的类型，由正向映射、反向映射以及证明二者互为逆的证明组成。证明中的第一个 have 表明 triangle n 等价于 i 遍历 Fin (n + 1) 时各 Fin i 的不交并。有趣的是，正向函数和反向函数是用策略构造出来的，而不是显式写出的。由于它们只是搬运数据和信息，rfl 即可证明它们互为逆。

之后，rw [←Fintype.card_coe] 将 #(triangle n) 重写为子类型 { x // x ∈ triangle n } 的基数，证明的其余部分就是计算。

example (n : ℕ) : #(triangle n) = (n + 1) * n / 2 := by
  have : triangle n ≃ Σ i : Fin (n + 1), Fin i.val :=
    { toFun := by
        rintro ⟨⟨i, j⟩, hp⟩
        have : (i ≤ n ∧ j ≤ n) ∧ i < j := by simpa [triangle] using hp
        exact ⟨⟨j, by linarith⟩, ⟨i, by linarith⟩⟩
      invFun := by
        rintro ⟨i, j⟩
        use ⟨j, i⟩
        suffices j ≤ n ∧ i ≤ n by simpa [triangle]
        constructor <;> linarith [i.2, j.2]
      left_inv := by intro i; rfl
      right_inv := by intro i; rfl }
  rw [←Fintype.card_coe]
  trans; apply (Fintype.card_congr this)
  rw [Fintype.card_sigma, sum_fin_eq_sum_range]
  convert Finset.sum_range_id (n + 1)
  simp_all

还有另一种方法。下面证明的第一行把问题化归为证明 2 * #(triangle n) = (n + 1) * n。我们可以通过证明两个三角形的副本恰好填满矩形 range n ×ˢ range (n + 1) 来做到这一点。作为练习，请尝试补全该计算的步骤。在解答中，我们在倒数第二步大量依赖 omega，但遗憾的是仍须手工完成相当一部分工作。

example (n : ℕ) : #(triangle n) = (n + 1) * n / 2 := by
  apply Nat.eq_div_of_mul_eq_right (by norm_num)
  let turn (p : ℕ × ℕ) : ℕ × ℕ := (n - 1 - p.1, n - p.2)
  calc 2 * #(triangle n)
      = #(triangle n) + #(triangle n) := by
          sorry
    _ = #(triangle n) + #(triangle n |>.image turn) := by
          sorry
    _ = #(range n ×ˢ range (n + 1)) := by
          sorry
    _ = (n + 1) * n := by
          sorry

你可以自行确信：若在 triangle 的定义中把 n 替换为 n + 1，并把 < 替换为 ≤，得到的只是向下平移后的同一个三角形。下面的练习要求你用这一事实证明两个三角形大小相同。

def triangle' (n : ℕ) : Finset (ℕ × ℕ) := {p ∈ range n ×ˢ range n | p.1 ≤ p.2}

example (n : ℕ) : #(triangle' n) = #(triangle n) := by sorry

本节最后给出一个例子和一个练习，它们来自 Bhavik Mehta 于 2023 年在 Lean for the Curious Mathematician 中关于组合数学的tutorial。设有一个二分图，其顶点集为 s 和 t，并且对 s 中每个 a，至少有三条边从 a 发出；对 t 中每个 b，至多有一条边进入 b。于是图中的边总数至少是 s 的基数的三倍，且至多是 t 的基数，由此推出 s 的基数的三倍不超过 t 的基数。下面的定理实现了这个论证，其中我们用关系 r 表示图的边。该证明是一个优雅的计算。

open Classical
variable (s t : Finset ℕ) (a b : ℕ)

theorem doubleCounting {α β : Type*} (s : Finset α) (t : Finset β)
    (r : α → β → Prop)
    (h_left : ∀ a ∈ s, 3 ≤ #{b ∈ t | r a b})
    (h_right : ∀ b ∈ t, #{a ∈ s | r a b} ≤ 1) :
    3 * #(s) ≤ #(t) := by
  calc 3 * #(s)
      = ∑ a ∈ s, 3                               := by simp [mul_comm]
    _ ≤ ∑ a ∈ s, #({b ∈ t | r a b})              := sum_le_sum h_left
    _ = ∑ a ∈ s, ∑ b ∈ t, if r a b then 1 else 0 := by simp
    _ = ∑ b ∈ t, ∑ a ∈ s, if r a b then 1 else 0 := sum_comm
    _ = ∑ b ∈ t, #({a ∈ s | r a b})              := by simp
    _ ≤ ∑ b ∈ t, 1                               := sum_le_sum h_right
    _ ≤ #(t)                                     := by simp

下面的练习同样取自 Mehta 的教程。设 A 是 range (2 * n) 的一个子集，且有 n + 1 个元素。容易看出，A 必含两个连续整数，从而含有两个互素的元素。如果观看该教程，你会看到为了建立下面这个事实曾花费了大量精力，而如今它可由 omega 自动证明。

example (m k : ℕ) (h : m ≠ k) (h' : m / 2 = k / 2) : m = k + 1 ∨ k = m + 1 := by omega

Mehta 练习的解法使用抽屉原理，其形式为 exists_lt_card_fiber_of_mul_lt_card_of_maps_to，来证明 A 中存在两个不同元素 m 和 k，使得 m / 2 = k / 2。请尝试补全这一事实的论证，然后用它完成证明。

example {n : ℕ} (A : Finset ℕ)
    (hA : #(A) = n + 1)
    (hA' : A ⊆ range (2 * n)) :
    ∃ m ∈ A, ∃ k ∈ A, Nat.Coprime m k := by
  have : ∃ t ∈ range n, 1 < #({u ∈ A | u / 2 = t}) := by
    apply exists_lt_card_fiber_of_mul_lt_card_of_maps_to
    · sorry
    · sorry
  rcases this with ⟨t, ht, ht'⟩
  simp only [one_lt_card, mem_filter] at ht'
  sorry

6.3. 归纳定义的类型

Lean 的基础允许我们定义归纳类型，即其实例自底向上生成的数据类型。例如，由 α 的元素组成的列表数据类型 List α，是从空列表 nil 开始，并不断向列表前端添加元素而生成的。下面我们将定义一种二叉树类型 BinTree，其元素从空树开始生成，并通过把一个新节点连接到两棵已有树上来构造新的树。

在 Lean 中，可以定义对象为无限的归纳类型，例如可数分支的良基树。不过，有限归纳定义在离散数学中很常用，尤其是在与计算机科学相关的离散数学分支中。Lean 不仅提供定义这类类型的方法，也提供归纳原则和递归定义原则。例如，数据类型 List α 是归纳定义的：

namespace MyListSpace

inductive List (α : Type*) where
  | nil  : List α
  | cons : α → List α → List α

end MyListSpace

该归纳定义说明，List α 的每个元素要么是空列表 nil，要么是 cons a as，其中 a 是 α 的一个元素，而 as 是由 α 的元素组成的列表。这两个构造子严格说名为 List.nil 和 List.cons，但当 List 命名空间打开时，可以使用较短的记号。当 List 命名空间未打开时，只要 Lean 期待一个列表，就可以写 .nil 和 .cons a as，Lean 会自动插入 List 限定。在本节中，我们会把临时定义放在诸如 MyListSpace 这样的独立命名空间中，以避免与标准库冲突。在临时命名空间之外，我们则恢复使用标准库定义。

Lean 将记号 [] 定义为 nil，将 :: 定义为 cons；并且可以把 [a, b, c] 写成 a :: b :: c :: []。函数 append 和 map 按如下方式递归定义：

def append {α : Type*} : List α → List α → List α
  | [],      bs => bs
  | a :: as, bs => a :: append as bs

def map {α β : Type*} (f : α → β) : List α → List β
  | []      => []
  | a :: as => f a :: map f as

#eval append [1, 2, 3] [4, 5, 6]
#eval map (fun n => n^2) [1, 2, 3, 4, 5]

注意这里有一个基本情形和一个递归情形。在每种情形中，这两个定义子句都按定义成立：

theorem nil_append {α : Type*} (as : List α) : append [] as = as := rfl

theorem cons_append {α : Type*} (a : α) (as : List α) (bs : List α) :
    append (a :: as) bs = a :: append as bs := rfl

theorem map_nil {α β : Type*} (f : α → β) : map f [] = [] := rfl

theorem map_cons {α β : Type*} (f : α → β) (a : α) (as : List α) :
    map f (a :: as) = f a :: map f as := rfl

函数 append 和 map 已在标准库中定义，而 append as bs 可以写成 as ++ bs。

Lean 允许你按照定义的结构写出归纳证明。

variable {α β γ : Type*}
variable (as bs cs : List α)
variable (a b c : α)

open List

theorem append_nil : ∀ as : List α, as ++ [] = as
  | [] => rfl
  | a :: as => by rw [cons_append, append_nil as]

theorem map_map (f : α → β) (g : β → γ) :
    ∀ as : List α, map g (map f as) = map (g ∘ f) as
  | [] => rfl
  | a :: as => by rw [map_cons, map_cons, map_cons, map_map f g as]; rfl

也可以使用策略 induction'。

当然，这些定理已经在标准库中。作为练习，请尝试在命名空间 MyListSpace3 中定义一个函数 reverse（以避免与标准的 List.reverse 冲突），用于反转列表。可以用 #eval reverse [1, 2, 3, 4, 5] 测试它。最直接的 reverse 定义需要二次时间，但不必为此担心。可以跳转到标准库中 List.reverse 的定义，查看一个线性时间实现。请尝试证明 reverse (as ++ bs) = reverse bs ++ reverse as 和 reverse (reverse as) = as。可以使用 cons_append 和 append_assoc，但你可能需要提出并证明若干辅助引理。

def reverse : List α → List α := sorry

theorem reverse_append (as bs : List α) : reverse (as ++ bs) = reverse bs ++ reverse as := by
  sorry

theorem reverse_reverse (as : List α) : reverse (reverse as) = as := by sorry

作为另一个例子，考虑下面的二叉树归纳定义，以及用于计算二叉树大小和深度的函数。

inductive BinTree where
  | empty : BinTree
  | node  : BinTree → BinTree → BinTree

namespace BinTree

def size : BinTree → ℕ
  | empty    => 0
  | node l r => size l + size r + 1

def depth : BinTree → ℕ
  | empty    => 0
  | node l r => max (depth l) (depth r) + 1

把空二叉树计为大小为 0、深度为 0 的二叉树是方便的。在文献中，这种数据类型有时称为extended binary trees。包含空树意味着，例如我们可以定义树 node empty (node empty empty)，它由一个根节点、一棵空左子树和一棵只含单个节点的右子树组成。

下面是联系大小与深度的一个重要不等式：

theorem size_le : ∀ t : BinTree, size t ≤ 2^depth t - 1
  | empty    => Nat.zero_le _
  | node l r => by
    simp only [depth, size]
    calc l.size + r.size + 1
      ≤ (2^l.depth - 1) + (2^r.depth - 1) + 1 := by
          gcongr <;> apply size_le
    _ ≤ (2 ^ max l.depth r.depth - 1) + (2 ^ max l.depth r.depth - 1) + 1 := by
          gcongr <;> simp
    _ ≤ 2 ^ (max l.depth r.depth + 1) - 1 := by
          have : 0 < 2 ^ max l.depth r.depth := by simp
          omega

请尝试证明下面这个不等式，它稍微容易一些。记住，如果像前一个定理那样用归纳法证明，必须删除 := by。

theorem depth_le_size : ∀ t : BinTree, depth t ≤ size t := by sorry

还请定义二叉树上的 flip 操作，它递归地交换左子树与右子树。

def flip : BinTree → BinTree := sorry

如果定义正确，下面命题的证明应当是 rfl。

example: flip  (node (node empty (node empty empty)) (node empty empty)) =
    node (node empty empty) (node (node empty empty) empty) := sorry

证明如下命题：

theorem size_flip : ∀ t, size (flip t) = size t := by sorry

本节最后讨论一些形式逻辑。下面给出命题公式的归纳定义。

inductive PropForm : Type where
  | var (n : ℕ)           : PropForm
  | fls                   : PropForm
  | conj (A B : PropForm) : PropForm
  | disj (A B : PropForm) : PropForm
  | impl (A B : PropForm) : PropForm

每个命题公式要么是变量 var n，要么是假常量 fls，要么是形如 conj A B、disj A B 或 impl A B 的复合公式。用通常的数学记号，它们分别常写为 \(p_n\)、\(\bot\)、\(A \wedge B\)、\(A \vee B\) 和 \(A \to B\)。其他命题联结词可以由这些联结词定义；例如，可以把 \(\neg A\) 定义为 \(A \to \bot\)，把 \(A \leftrightarrow B\) 定义为 \((A \to B) \wedge (B \to A)\)。

定义了命题公式这一数据类型之后，我们定义在把布尔真值赋给变量的赋值 v 下，如何求一个命题公式的值。

def eval : PropForm → (ℕ → Bool) → Bool
  | var n,    v => v n
  | fls,      _ => false
  | conj A B, v => A.eval v && B.eval v
  | disj A B, v => A.eval v || B.eval v
  | impl A B, v => ! A.eval v || B.eval v

下一个定义给出公式中出现的变量集合，随后定理表明：若两个真值赋值在某公式的变量上相同，则在这两个赋值下计算该公式会得到相同的值。

def vars : PropForm → Finset ℕ
  | var n    => {n}
  | fls      => ∅
  | conj A B => A.vars ∪ B.vars
  | disj A B => A.vars ∪ B.vars
  | impl A B => A.vars ∪ B.vars

theorem eval_eq_eval : ∀ (A : PropForm) (v1 v2 : ℕ → Bool),
    (∀ n ∈ A.vars, v1 n = v2 n) → A.eval v1 = A.eval v2
  | var n, v1, v2, h    => by simp_all [vars, eval]
  | fls, v1, v2, h      => by simp_all [eval]
  | conj A B, v1, v2, h => by
      simp_all [vars, eval, eval_eq_eval A v1 v2, eval_eq_eval B v1 v2]
  | disj A B, v1, v2, h => by
      simp_all [vars, eval, eval_eq_eval A v1 v2, eval_eq_eval B v1 v2]
  | impl A B, v1, v2, h => by
      simp_all [vars, eval, eval_eq_eval A v1 v2, eval_eq_eval B v1 v2]

注意到这些重复，我们可以更聪明地使用自动化。

theorem eval_eq_eval' (A : PropForm) (v1 v2 : ℕ → Bool) (h : ∀ n ∈ A.vars, v1 n = v2 n) :
    A.eval v1 = A.eval v2 := by
  cases A <;> simp_all [eval, vars, fun A => eval_eq_eval' A v1 v2]

函数 subst A m C 描述把公式 C 代入公式 A 中变量 var m 每一次出现处所得的结果。

def subst : PropForm → ℕ → PropForm → PropForm
  | var n,    m, C => if n = m then C else var n
  | fls,      _, _ => fls
  | conj A B, m, C => conj (A.subst m C) (B.subst m C)
  | disj A B, m, C => disj (A.subst m C) (B.subst m C)
  | impl A B, m, C => impl (A.subst m C) (B.subst m C)

作为例子，证明：若一个变量不出现在公式中，则对该变量作代入没有任何影响：

theorem subst_eq_of_not_mem_vars :
    ∀ (A : PropForm) (n : ℕ) (C : PropForm), n ∉ A.vars → A.subst n C = A := sorry

下面的定理说的是更细致也更有趣的事实：在真值赋值 v 下求 A.subst n C 的值，等同于在一个把 var n 赋为 C 的值的真值赋值下求 A 的值。请尝试证明它。

theorem subst_eval_eq : ∀ (A : PropForm) (n : ℕ) (C : PropForm) (v : ℕ → Bool),
  (A.subst n C).eval v = A.eval (fun m => if m = n then C.eval v else v m) := sorry
