1. 引言

1.1. 入门

本书的目标，是教会读者使用 Lean 4 交互式证明助手来形式化数学。本书假定读者具备一定的数学知识，但所需预备知识并不多。尽管我们将涉及从数论到测度论与分析的诸多例子，重点仍将放在这些领域的初等方面；如果读者尚不熟悉相关内容，也可以在学习过程中逐步掌握。我们也不预设读者具有形式化方法方面的背景。形式化可以看作一种计算机编程：我们将用一种规整的、类似程序语言且 Lean 能够理解的语言，来写出数学定义、定理与证明。作为回报，Lean 会提供反馈和信息，解释表达式并保证它们是良构的，最终还会认证我们的证明确实正确。

关于 Lean 的更多信息，可以参见 Lean 项目页面以及 Lean 社区网页。本教程建立在 Lean 庞大且持续增长的库 Mathlib 之上。我们也强烈建议尚未加入的读者加入 Lean Zulip 在线聊天室。在那里，你会遇到一个活跃而友善的 Lean 爱好者社群，大家乐于回答问题并提供支持。

虽然读者可以在线阅读本书的 PDF 或 HTML 版本，但本书的设计目标是交互式阅读，也就是在 VS Code 编辑器中运行 Lean。开始学习前，请先完成以下步骤：

  1. 按照这些安装说明安装 Lean 4 和 VS Code。

  2. 点击 VS Code 右上角的全称量词符号，选择 Open Project、Download Project，再选择 Mathematics in Lean，以获取本仓库。

  3. 本书每一节都有一个关联的 Lean 文件，其中包含例子和练习。它们位于按章节组织的 MIL 文件夹中。我们强烈建议复制该文件夹，并在副本中实验和完成练习。这样既能保持原文件不变，也能在仓库内容更新时更方便地同步（见下文）。你可以把副本命名为 my_files，或任何你喜欢的名字，并用它创建自己的 Lean 文件。

到这一步，你可以按如下方式在 VS Code 的侧边面板中打开本教材：

  1. 输入 ctrl-shift-P（macOS 中为 command-shift-P）。

  2. 在出现的输入栏中输入 Lean 4: Docs: Show Documentation Resources，然后按回车。（一旦菜单中高亮该项，也可以直接按回车选择。）

  3. 在打开的窗口中点击 Mathematics in Lean。

另一种方式是在云端通过 Codespaces 运行 Lean 和 VS Code。关于如何使用 Codespaces，可以在 Mathematics in Lean 的 项目页面找到说明。即便采用这种方式，我们仍建议按照上文所述，在 MIL 文件夹的副本中工作。

本教材及其关联仓库仍在持续开发中。你可以在 mathematics_in_lean 文件夹中输入 git pull，随后输入 lake exe cache get 来更新仓库。（这假定你没有修改 MIL 文件夹中的内容；这也正是我们建议创建副本的原因。）

我们希望读者在阅读教材时，在 MIL 文件夹中完成练习；教材正文则提供解释、说明与提示。正文中经常会包含如下例子：

#eval "Hello, World!"

你应当能够在相应的 Lean 文件中找到对应例子。点击该行时，VS Code 会在 Lean InfoView 窗口中显示 Lean 的反馈；将光标悬停在 #eval 命令上时，VS Code 会在弹出窗口中显示 Lean 对该命令的响应。我们鼓励你编辑文件，并尝试自己构造例子。

本书还提供了大量具有挑战性的练习，供读者尝试。不要匆忙跳过它们！Lean 关乎以交互方式做数学，而不只是阅读关于数学的文字。完成练习是这种体验的核心。当然，你不必完成所有练习；当你确信自己已经掌握相关技能时，完全可以继续向后阅读。你也总是可以将自己的解答同每一节对应的 solutions 文件夹中的解答进行比较。

1.2. 概览

简单地说，Lean 是一种用于在称为依值类型论的形式语言中构造复杂表达式的工具。

每个表达式都有一个类型，你可以使用 #check 命令将其打印出来。有些表达式的类型形如 ℕ 或 ℕ → ℕ。这些表达式是数学对象。

#check 2 + 2

def f (x : ℕ) :=
  x + 3

#check f

有些表达式的类型是 Prop。这些表达式是数学命题。

#check 2 + 2 = 4

def FermatLastTheorem :=
  ∀ x y z n : ℕ, n > 2 ∧ x * y * z ≠ 0 → x ^ n + y ^ n ≠ z ^ n

#check FermatLastTheorem

还有一些表达式的类型是 P，而 P 本身的类型是 Prop。这样的表达式就是命题 P 的一个证明。

theorem easy : 2 + 2 = 4 :=
  rfl

#check easy

theorem hard : FermatLastTheorem :=
  sorry

#check hard

如果你能够构造一个类型为 FermatLastTheorem 的表达式，并且 Lean 接受它为该类型的项，那么你就完成了一件相当了不起的事。（使用 sorry 是作弊，而 Lean 知道这一点。）至此，你已经知道这场游戏的目标；剩下要学习的只是规则。

本书与配套教程 Theorem Proving in Lean 相辅相成；后者对 Lean 的底层逻辑框架和核心语法作了更系统的介绍。Theorem Proving in Lean 适合那种在使用一台新洗碗机之前，愿意从头到尾读完用户手册的人。如果你更倾向于先按下启动按钮，再慢慢弄清楚如何启用强力洗涤功能，那么从本书开始，并在必要时回头查阅 Theorem Proving in Lean，会更合适。

Mathematics in Lean 与 Theorem Proving in Lean 的另一项区别在于，本书更强调策略的使用。既然我们的目标是构造复杂表达式，Lean 提供了两种途径：我们可以直接写出这些表达式本身（也就是写出合适的文本描述），也可以向 Lean 给出构造这些表达式的指令。例如，下列表达式表示这样一个事实的证明：若 n 为偶数，则 m * n 也是偶数。

example : ∀ m n : Nat, Even n → Even (m * n) := fun m n ⟨k, (hk : n = k + k)⟩ ↦
  have hmn : m * n = m * k + m * k := by rw [hk, mul_add]
  show ∃ l, m * n = l + l from ⟨_, hmn⟩

这个证明项可以压缩成一行：

example : ∀ m n : Nat, Even n → Even (m * n) :=
fun m n ⟨k, hk⟩ ↦ ⟨m * k, by rw [hk, mul_add]⟩

下面则是同一定理的一个策略式证明，其中以 -- 开头的行是注释，因此会被 Lean 忽略：

example : ∀ m n : Nat, Even n → Even (m * n) := by
  -- Say `m` and `n` are natural numbers, and assume `n = 2 * k`.
  rintro m n ⟨k, hk⟩
  -- We need to prove `m * n` is twice a natural number. Let's show it's twice `m * k`.
  use m * k
  -- Substitute for `n`,
  rw [hk]
  -- and now it's obvious.
  ring

在 VS Code 中逐行输入这样的证明时，Lean 会在单独窗口中显示证明状态，告诉你已经建立了哪些事实，以及为了证明定理还剩下哪些任务。你可以逐行移动光标来回放证明，因为 Lean 会持续显示光标所在位置的证明状态。在这个例子中，你会看到证明的第一行引入了 m 和 n（如果愿意，此时也可以给它们另取名字），并且把假设 Even n 分解为某个 k 以及假设 n = 2 * k。第二行 use m * k 声明：我们将通过证明 m * n = 2 * (m * k) 来说明 m * n 是偶数。下一行使用 rw 策略在目标中把 n 替换为 2 * k（rw 是 “rewrite” 的缩写），而 ring 策略则解出由此得到的目标 m * (2 * k) = 2 * (m * k)。

以小步构造证明并获得增量反馈的能力极其强大。正因如此，策略式证明通常比直接写证明项更容易也更快捷。二者之间并没有截然分明的界线：策略式证明可以嵌入证明项中，就像上例中的 by rw [hk, mul_add] 那样。我们也会看到，反过来，在策略式证明中插入一个短小的证明项也常常很有用。尽管如此，本书的重点仍将放在策略的使用上。

在我们的例子中，策略式证明也可以缩减为一行：

example : ∀ m n : Nat, Even n → Even (m * n) := by
  rintro m n ⟨k, hk⟩; use m * k; rw [hk]; ring

这里我们使用策略完成了一些小的证明步骤。但策略也可以提供相当程度的自动化，并为更长的计算和更大的推理步骤给出论证。例如，我们可以调用 Lean 的简化器，并指定若干关于奇偶性的化简规则，从而自动证明我们的定理。

example : ∀ m n : Nat, Even n → Even (m * n) := by
  intros; simp [*, parity_simps]

这两本入门教程之间还有一个重要区别：Theorem Proving in Lean 只依赖核心 Lean 及其内建策略，而 Mathematics in Lean 则建立在 Lean 强大且持续增长的库 Mathlib 之上。因此，我们可以展示如何使用库中的若干数学对象和定理，以及一些非常有用的策略。本书并不打算作为该库的完整概览；社区网页中已有大量文档。我们的目标毋宁是介绍这种形式化背后的思维方式，并指出一些基本入口，使读者能够从容地浏览库并自行寻找所需内容。

交互式定理证明有时令人挫败，而且学习曲线陡峭。不过，Lean 社区对新手非常友好，Lean Zulip 聊天室 中几乎随时都有人可以回答问题。我们希望在那里见到你；也毫不怀疑，过不了多久，你也能够回答这类问题，并为 Mathlib 的发展作出贡献。

因此，如果你愿意接受，这就是你的任务：投入其中，尝试练习，带着问题来到 Zulip，并享受这个过程。不过也请预先知道：交互式定理证明会迫使你以根本上新的方式思考数学和数学推理。你的生活也许从此不再相同。
