11. 拓扑

微积分建立在函数概念之上，函数用于刻画相互依赖的量。例如，研究随时间变化的量是很常见的。极限概念同样是基础性的。我们可以说函数 \(f(x)\) 在 \(x\) 趋近于 \(a\) 时的极限为 \(b\)，也可以说当 \(x\) 趋近于 \(a\) 时，\(f(x)\) 收敛到 \(b\)。等价地，可以说当 \(x\) 趋近于 \(a\) 时，\(f(x)\) 趋近于 \(b\)；或说当 \(x\) 趋向于 \(a\) 时，它趋向 \(b\)。我们已经在Section 3.6中开始考察这类概念。

拓扑学是对极限与连续性的抽象研究。在第 2 章至第 7 章介绍完形式化的基本内容之后，本章将解释 Mathlib 中如何形式化拓扑概念。拓扑抽象不仅适用于大得多的一般性范围，而且有些反直觉的是，它还使我们在具体实例中更容易推理极限和连续性。

拓扑概念建立在若干层数学结构之上。第一层是朴素集合论，如Chapter 4所述。下一层是滤子理论，我们将在Section 11.1中介绍。在此之上，还叠加了拓扑空间、度量空间以及一个稍显特殊的中间概念，称为一致空间。

前几章依赖的数学概念大多可能为你所熟悉；相比之下，滤子概念即使对许多职业数学家也不那么常见。然而，为了有效形式化数学，这一概念是必不可少的。下面说明原因。设 f : ℝ → ℝ 为任意函数。我们可以考虑当 x 趋近于某个值 x₀ 时 f x 的极限，也可以考虑当 x 趋向正无穷或负无穷时 f x 的极限。此外，还可以考虑当 x 从右侧趋近于 x₀（惯常写作 x₀⁺）或从左侧趋近于 x₀⁻ 时 f x 的极限。还有一些变体，其中 x 趋近于 x₀、x₀⁺ 或 x₀⁻，但不允许取值为 x₀ 本身。这样，x 趋近某物的方式至少有八种。还可以把 x 限制为有理数，或对定义域施加其他约束；但这里先只考虑这 8 种情形。

在余定义域上也有类似的多种选择：可以指定 f x 从左侧或右侧趋近某个值，或者趋向正无穷或负无穷，等等。例如，可能想表达：当 x 从右侧趋近 x₀ 且不等于 x₀ 时，f x 趋向 +∞。这会产生 64 种不同的极限陈述，而我们甚至还没有开始处理序列极限，后者曾在Section 3.6中出现。

涉及支撑引理时，问题会进一步复杂化。例如，极限可以复合：若当 x 趋向 x₀ 时 f x 趋向 y₀，且当 y 趋向 y₀ 时 g y 趋向 z₀，则当 x 趋向 x₀ 时 g ∘ f x 趋向 z₀。这里有三个“趋向”概念，每一个都可以按前一段所述的八种方式实例化。于是得到 512 个引理；若都加入库中，数量相当庞大。非形式化地说，数学家通常证明其中两三个，然后只说明其余可以“同样地”证明。形式化数学要求把相关的“相同方式”完全明确化，而这正是布尔巴基的滤子理论所做到的。

11.1. 滤子

类型 X 上的滤子是 X 的若干集合组成的族，并满足下面将列出的三个条件。这个概念支撑两类相关想法：

  • 极限，包括上文讨论的所有极限类型：序列的有限与无限极限，函数在一点或无穷远处的有限与无限极限，等等。
  • 最终成立的性质，包括对充分大的 n : ℕ 成立、在某点 x 的足够近处成立、对足够接近的点对成立，或在测度论意义下几乎处处成立。对偶地，滤子也可以表达频繁成立的性质：对任意大的 n、在给定点任意邻域中的某一点，等等。

与这些描述对应的滤子将在本节稍后定义，但我们已经可以先给出它们的名称：

  • (atTop : Filter ℕ)，由那些包含某个 N 对应的集合 {n | n ≥ N} 的 ℕ 的子集组成
  • 𝓝 x，由拓扑空间中 x 的邻域组成
  • 𝓤 X，由一致空间的 entourage 组成（一致空间推广了度量空间和拓扑群）
  • μ.ae，由相对于测度 μ 其补集测度为零的集合组成。

一般定义如下：滤子 F : Filter X 是由若干集合组成的族 F.sets : Set (Set X)，并满足下列条件：

  • 条件 F.univ_sets : univ ∈ F.sets
  • 条件 F.sets_of_superset : ∀ {U V}, U ∈ F.sets → U ⊆ V → V ∈ F.sets
  • 条件 F.inter_sets : ∀ {U V}, U ∈ F.sets → V ∈ F.sets → U ∩ V ∈ F.sets。

第一个条件说，X 的全体元素组成的集合属于 F.sets。第二个条件说，若 U 属于 F.sets，则任何包含 U 的集合也属于 F.sets。第三个条件说，F.sets 在有限交下封闭。在 Mathlib 中，滤子 F 被定义为一个打包了 F.sets 及其三个性质的结构；但这些性质不携带额外数据，因此混同 F 与 F.sets 很方便。于是我们把 U ∈ F 定义为 U ∈ F.sets。这解释了为什么有些提及 U ∈ F 的引理名称中会出现 sets 一词。

把滤子理解为定义了“充分大”集合的概念，可能会有所帮助。第一个条件于是说 univ 是充分大的；第二个条件说，包含充分大集合的集合仍然充分大；第三个条件说，两个充分大集合的交仍然充分大。

更有用的想法可能是，把类型 X 上的滤子视为 Set X 的广义元素。例如，atTop 是“非常大的数的集合”，而 𝓝 x₀ 是“非常接近 x₀ 的点的集合”。这种观点的一个体现是，可以把任意 s : Set X 关联到所谓的主滤子，它由所有包含 s 的集合组成。这个定义已在 Mathlib 中，并有记号 𝓟（局部化在 Filter 命名空间中）。为了演示，我们请你借此机会在这里完成这个定义。

def principal {α : Type*} (s : Set α) : Filter α
    where
  sets := { t | s ⊆ t }
  univ_sets := sorry
  sets_of_superset := sorry
  inter_sets := sorry

作为第二个例子，我们请你定义滤子 atTop : Filter ℕ。（也可以用任何带预序的类型替代 ℕ。）

example : Filter ℕ :=
  { sets := { s | ∃ a, ∀ b, a ≤ b → b ∈ s }
    univ_sets := sorry
    sets_of_superset := sorry
    inter_sets := sorry }

还可以直接定义任意 x : ℝ 的邻域滤子 𝓝 x。在实数中，x 的一个邻域是包含开区间 \((x_0 - \varepsilon, x_0 + \varepsilon)\) 的集合；该开区间在 Mathlib 中定义为 Ioo (x₀ - ε) (x₀ + ε)。（这种邻域概念只是 Mathlib 中更一般构造的一个特例。）

有了这些例子，已经可以如下定义函数 f : X → Y 沿某个 F : Filter X 收敛到某个 G : Filter Y 的含义：

def Tendsto₁ {X Y : Type*} (f : X → Y) (F : Filter X) (G : Filter Y) :=
  ∀ V ∈ G, f ⁻¹' V ∈ F

当 X 为 ℕ 且 Y 为 ℝ 时，Tendsto₁ u atTop (𝓝 x) 等价于说序列 u : ℕ → ℝ 收敛到实数 x。当 X 与 Y 都是 ℝ 时，Tendsto f (𝓝 x₀) (𝓝 y₀) 等价于熟悉的概念 \(\lim_{x \to x₀} f(x) = y₀\)。引言中提到的所有其他极限类型，也都等价于对源和目标上的滤子作适当选择后得到的 Tendsto₁ 实例。

上面的概念 Tendsto₁ 与 Mathlib 中定义的 Tendsto 按定义等价，但后者定义得更加抽象。Tendsto₁ 的定义问题在于，它显式暴露了量词以及 G 的元素，并隐藏了把滤子看作广义集合时得到的直观。通过使用更多代数和集合论机制，可以隐藏量词 ∀ V，并让直观更突出。第一个成分是与任意映射 f : X → Y 对应的pushforward操作 \(f_*\)，在 Mathlib 中记作 Filter.map f。给定 X 上的滤子 F，Filter.map f F : Filter Y 被定义为使 V ∈ Filter.map f F ↔ f ⁻¹' V ∈ F 按定义成立。在示例文件中，我们打开了 Filter 命名空间，因此 Filter.map 可以写作 map。这意味着可以用 Filter Y 上的序关系重写 Tendsto 的定义；这个序关系是成员集合包含关系的反向。换言之，给定 G H : Filter Y，有 G ≤ H ↔ ∀ V : Set Y, V ∈ H → V ∈ G。

def Tendsto₂ {X Y : Type*} (f : X → Y) (F : Filter X) (G : Filter Y) :=
  map f F ≤ G

example {X Y : Type*} (f : X → Y) (F : Filter X) (G : Filter Y) :
    Tendsto₂ f F G ↔ Tendsto₁ f F G :=
  Iff.rfl

滤子上的序关系看起来也许是反的。但请回忆，可以通过嵌入 𝓟 : Set X → Filter X 把 X 上的滤子看成 Set X 的广义元素；该嵌入把任意集合 s 送到相应的主滤子。这个嵌入保持序，因此 Filter 上的序关系确实可以视为广义集合之间的自然包含关系。在这个类比中，推出类似于直接像。事实上，map f (𝓟 s) = 𝓟 (f '' s)。

现在可以直观地理解，为什么序列 u : ℕ → ℝ 收敛到点 x₀ 当且仅当有 map u atTop ≤ 𝓝 x₀。这个不等式表示，“非常大的自然数集合”在 u 下的“直接像”被“包含”在“非常接近 x₀ 的点的集合”中。

如所承诺的，Tendsto₂ 的定义不再显式出现量词或集合。它还利用了推出操作的代数性质。首先，每个 Filter.map f 都是单调的。其次，Filter.map 与复合相容。

#check (@Filter.map_mono : ∀ {α β} {m : α → β}, Monotone (map m))

#check
  (@Filter.map_map :
    ∀ {α β γ} {f : Filter α} {m : α → β} {m' : β → γ}, map m' (map m f) = map (m' ∘ m) f)

这两个性质合在一起，使我们能够证明极限的复合性；它一次性给出引言中所述复合引理的全部 512 个变体，以及更多结果。你可以练习证明下面的陈述：既可以使用以全称量词表述的 Tendsto₁ 定义，也可以使用代数定义，并结合上面的两个引理。

example {X Y Z : Type*} {F : Filter X} {G : Filter Y} {H : Filter Z} {f : X → Y} {g : Y → Z}
    (hf : Tendsto₁ f F G) (hg : Tendsto₁ g G H) : Tendsto₁ (g ∘ f) F H :=
  sorry

推出构造用一个映射把滤子从映射的源推到目标。也有一个方向相反的拉回操作 Filter.comap。它推广了集合上的原像操作。对任意映射 f，Filter.map f 与 Filter.comap f 构成所谓的伽罗瓦连接，也就是说，它们满足

即 Filter.map_le_iff_le_comap : Filter.map f F ≤ G ↔ F ≤ Filter.comap f G

对每个 F 和 G 都成立。这个操作可以用于给出 Tendsto 的另一种表述；它与 Mathlib 中的定义可证明等价，但不按定义等价。

comap 操作可以用来把滤子限制到子类型上。例如，设有 f : ℝ → ℝ、x₀ : ℝ 和 y₀ : ℝ，并且想陈述当 x 在有理数内趋近于 x₀ 时，f x 趋近于 y₀。可以使用强制映射 (↑) : ℚ → ℝ，把滤子 𝓝 x₀ 拉回到 ℚ，并陈述 Tendsto (f ∘ (↑) : ℚ → ℝ) (comap (↑) (𝓝 x₀)) (𝓝 y₀)。

variable (f : ℝ → ℝ) (x₀ y₀ : ℝ)

#check comap ((↑) : ℚ → ℝ) (𝓝 x₀)

#check Tendsto (f ∘ (↑)) (comap ((↑) : ℚ → ℝ) (𝓝 x₀)) (𝓝 y₀)

拉回操作也与复合相容，但它是反变的，也就是说，它会反转参数的顺序。

section
variable {α β γ : Type*} (F : Filter α) {m : γ → β} {n : β → α}

#check (comap_comap : comap m (comap n F) = comap (n ∘ m) F)

end

现在把注意力转向平面 ℝ × ℝ，并尝试理解点 (x₀, y₀) 的邻域与 𝓝 x₀、𝓝 y₀ 之间的关系。有一个积操作 Filter.prod : Filter X → Filter Y → Filter (X × Y)，记作 ×ˢ，它回答了这个问题：

example : 𝓝 (x₀, y₀) = 𝓝 x₀ ×ˢ 𝓝 y₀ :=
  nhds_prod_eq

积操作由拉回操作和 inf 运算定义：

即 F ×ˢ G = (comap Prod.fst F) ⊓ (comap Prod.snd G)。

这里的 inf 运算指的是任意类型 X 上 Filter X 的格结构；其中 F ⊓ G 是同时小于 F 和 G 的最大滤子。因此，inf 运算推广了集合交的概念。

Mathlib 中大量证明会使用上述全部结构（map、comap、inf、sup 和 prod），给出关于收敛性的代数证明，而从不提及滤子的成员。你可以在证明下面引理时练习这一点；必要时可展开 Tendsto 与 Filter.prod 的定义。

#check le_inf_iff

example (f : ℕ → ℝ × ℝ) (x₀ y₀ : ℝ) :
    Tendsto f atTop (𝓝 (x₀, y₀)) ↔
      Tendsto (Prod.fst ∘ f) atTop (𝓝 x₀) ∧ Tendsto (Prod.snd ∘ f) atTop (𝓝 y₀) :=
  sorry

有序类型 Filter X 实际上是一个完备格，也就是说，它有底元和顶元，并且 X 上任意滤子集合都有一个 Inf 和一个 Sup。

注意，给定滤子定义中的第二个性质（若 U 属于 F，则任何比 U 更大的集合也属于 F），第一个性质（X 的所有元素组成的集合属于 F）等价于 F 不是空的集合族。不要把这一点与“空集是否为 F 的元素”这一更微妙的问题混淆。滤子的定义并不禁止 ∅ ∈ F；但如果空集在 F 中，那么每个集合都在 F 中，也就是说 ∀ U : Set X, U ∈ F。此时，F 是一个相当平凡的滤子，正是完备格 Filter X 的底元。这与布尔巴基的滤子定义不同，后者不允许滤子包含空集。

因为我们的定义包含平凡滤子，有些引理中就需要显式假设非平凡性。不过作为回报，理论具有更好的整体性质。我们已经看到，包含平凡滤子给出了底元。它还允许定义 principal : Set X → Filter X，把 ∅ 映到 ⊥，而无需添加排除空集的前提。它也同样允许无前提地定义拉回操作。事实上，可能出现 comap f F = ⊥ 但 F ≠ ⊥ 的情况。例如，给定 x₀ : ℝ 和 s : Set ℝ，在与 s 对应的子类型到实数的强制映射下，𝓝 x₀ 的拉回非平凡，当且仅当 x₀ 属于 s 的闭包。

为了管理那些确实需要假设某个滤子非平凡的引理，Mathlib 有一个类型类 Filter.NeBot，库中也有假设 (F : Filter X) [F.NeBot] 的引理。例如，实例数据库知道 (atTop : Filter ℕ).NeBot，并且知道非平凡滤子的推出仍为非平凡滤子。因此，一个假设 [F.NeBot] 的引理会自动适用于任意序列 u 的 map u atTop。

我们对滤子的代数性质及其与极限关系的巡览基本完成了，但还没有说明为什么这确实重新获得了通常的极限概念。表面上看，Tendsto u atTop (𝓝 x₀) 似乎强于Section 3.6中定义的收敛概念，因为这里要求 x₀ 的每个邻域的原像都属于 atTop，而通常定义只要求标准邻域 Ioo (x₀ - ε) (x₀ + ε) 满足这一点。关键在于，按定义，每个邻域都包含这样一个标准邻域。这个观察引出滤子基的概念。

给定 F : Filter X，若集合族 s : ι → Set X 满足：对每个集合 U，都有 U ∈ F 当且仅当它包含某个 s i，则称它是 F 的基。换言之，形式上说，s 是基当且仅当它满足 ∀ U : Set X, U ∈ F ↔ ∃ i, s i ⊆ U。更灵活的做法是在 ι 上考虑一个谓词，只选择指标类型中的某些值 i。在 𝓝 x₀ 的情形中，我们希望 ι 是 ℝ，把 i 写作 ε，并且该谓词应当选出正的 ε。因此，集合 Ioo  (x₀ - ε) (x₀ + ε) 构成 ℝ 上邻域拓扑的基这一事实，陈述如下：

example (x₀ : ℝ) : HasBasis (𝓝 x₀) (fun ε : ℝ ↦ 0 < ε) fun ε ↦ Ioo (x₀ - ε) (x₀ + ε) :=
  nhds_basis_Ioo_pos x₀

滤子 atTop 也有一个很好的基。引理 Filter.HasBasis.tendsto_iff 允许我们在给定 F 和 G 的基之后，重新表述形如 Tendsto f F G 的陈述。把这些内容结合起来，基本上就得到我们在Section 3.6中使用的收敛概念。

example (u : ℕ → ℝ) (x₀ : ℝ) :
    Tendsto u atTop (𝓝 x₀) ↔ ∀ ε > 0, ∃ N, ∀ n ≥ N, u n ∈ Ioo (x₀ - ε) (x₀ + ε) := by
  have : atTop.HasBasis (fun _ : ℕ ↦ True) Ici := atTop_basis
  rw [this.tendsto_iff (nhds_basis_Ioo_pos x₀)]
  simp

现在说明滤子如何帮助处理“对充分大的数成立”或“对充分接近某点的点成立”的性质。在Section 3.6中，我们经常遇到这样的情形：已知某性质 P n 对充分大的 n 成立，且另一个性质 Q n 也对充分大的 n 成立。使用两次 cases 会得到 N_P 与 N_Q，满足 ∀ n ≥ N_P, P n 和 ∀ n ≥ N_Q, Q n。再使用 set N := max N_P N_Q，最终可以证明 ∀ n ≥ N, P n ∧ Q n。反复这样做会变得繁琐。

可以做得更好：注意陈述“P n 与 Q n 对充分大的 n 成立”意味着有 {n | P n} ∈ atTop 和 {n | Q n} ∈ atTop。由于 atTop 是滤子，atTop 的两个元素之交仍在 atTop 中，因此有 {n | P n ∧ Q n} ∈ atTop。写 {n | P n} ∈ atTop 并不方便，但可以使用更具提示性的记号 ∀ᶠ n in atTop, P n。这里上标 f 代表“Filter”。可以把这个记号理解为：对“非常大的数的集合”中的所有 n，P n 成立。∀ᶠ 记号代表 Filter.Eventually，而引理 Filter.Eventually.and 使用滤子的交性质完成刚才描述的事情：

example (P Q : ℕ → Prop) (hP : ∀ᶠ n in atTop, P n) (hQ : ∀ᶠ n in atTop, Q n) :
    ∀ᶠ n in atTop, P n ∧ Q n :=
  hP.and hQ

这个记号非常方便且直观，因此当 P 是等式或不等式陈述时也有专门版本。例如，令 u 和 v 为两个实数序列，证明若 u n 与 v n 对充分大的 n 相等，则 u 趋向 x₀ 当且仅当 v 趋向 x₀。我们先使用一般的 Eventually，再使用针对等式谓词的专门版本 EventuallyEq。这两个陈述按定义等价，因此同一个证明在两种情形中都适用。

example (u v : ℕ → ℝ) (h : ∀ᶠ n in atTop, u n = v n) (x₀ : ℝ) :
    Tendsto u atTop (𝓝 x₀) ↔ Tendsto v atTop (𝓝 x₀) :=
  tendsto_congr' h

example (u v : ℕ → ℝ) (h : u =ᶠ[atTop] v) (x₀ : ℝ) :
    Tendsto u atTop (𝓝 x₀) ↔ Tendsto v atTop (𝓝 x₀) :=
  tendsto_congr' h

用 Eventually 来回顾滤子的定义很有启发。给定 F : Filter X，对 X 上任意谓词 P 和 Q，

  • 条件 univ ∈ F 保证 (∀ x, P x) → ∀ᶠ x in F, P x，
  • 条件 U ∈ F → U ⊆ V → V ∈ F 保证 (∀ᶠ x in F, P x) → (∀ x, P x → Q x) → ∀ᶠ x in F, Q x，并且
  • 条件 U ∈ F → V ∈ F → U ∩ V ∈ F 保证 (∀ᶠ x in F, P x) → (∀ᶠ x in F, Q x) → ∀ᶠ x in F, P x ∧ Q x。

11.2. 度量空间

上一节的例子集中于实数序列。本节将稍微提高一般性，转向度量空间。度量空间是一个类型 X，配备距离函数 dist : X → X → ℝ；它推广了 X = ℝ 情形中的函数 fun x y ↦ |x - y|。

引入这样的空间很容易，并且我们将检查距离函数所需的所有性质。

variable {X : Type*} [MetricSpace X] (a b c : X)

#check (dist a b : ℝ)
#check (dist_nonneg : 0 ≤ dist a b)
#check (dist_eq_zero : dist a b = 0 ↔ a = b)
#check (dist_comm a b : dist a b = dist b a)
#check (dist_triangle a b c : dist a c ≤ dist a b + dist b c)

注意，还有一些变体允许距离为无穷，或者允许 dist a b 为零而不必有 a = b，或同时允许这两种情况。它们分别称为 EMetricSpace、PseudoMetricSpace 和 PseudoEMetricSpace（这里的“e”代表“extended”）。

注意，从 ℝ 到度量空间的路径跳过了赋范空间这一特殊情形；赋范空间还需要线性代数，并将在微积分章节中解释。

11.2.1. 收敛与连续

使用距离函数，已经可以定义收敛序列以及度量空间之间的连续函数。实际上，它们是在下一节涉及的更一般设置中定义的，但我们有用距离重新表述定义的引理。

example {u : ℕ → X} {a : X} :
    Tendsto u atTop (𝓝 a) ↔ ∀ ε > 0, ∃ N, ∀ n ≥ N, dist (u n) a < ε :=
  Metric.tendsto_atTop

example {X Y : Type*} [MetricSpace X] [MetricSpace Y] {f : X → Y} :
    Continuous f ↔
      ∀ x : X, ∀ ε > 0, ∃ δ > 0, ∀ x', dist x' x < δ → dist (f x') (f x) < ε :=
  Metric.continuous_iff

许多引理带有连续性假设，因此我们最终会证明大量连续性结果，并且有一个专门完成这项任务的 continuity 策略。先证明下面练习中将要用到的一个连续性陈述。注意，Lean 知道如何把两个度量空间的积视为度量空间；它选择上确界范数，因此 dist (x₁, y₁) (x₂, y₂) = max (dist x₁ x₂) (dist y₁ y₂) 由自反性成立。因此，考虑从 X × X 到 ℝ 的连续函数是有意义的。特别地，距离函数的非柯里化版本就是这样的函数。

example {X Y : Type*} [MetricSpace X] [MetricSpace Y] {f : X → Y} (hf : Continuous f) :
    Continuous fun p : X × X ↦ dist (f p.1) (f p.2) := by continuity

这个策略有些慢，因此知道如何手动完成也很有用。首先需要使用：函数 fun p : X × X ↦ f p.1 是连续的，因为它是 f 与投影 prod.fst 的复合；其中 f 根据假设 hf 连续，而 prod.fst 的连续性正是引理 continuous_fst 的内容。复合性质是 Continuous.comp，它位于 Continuous 命名空间中，因此可以使用点记号把 Continuous.comp hf continuous_fst 压缩为 hf.comp continuous_fst；后者实际上更可读，因为它确实读作把我们的假设与引理复合。对第二个分量也可以类似处理，得到 fun p : X × X ↦ f p.2 的连续性。然后用 Continuous.prod_mk 组装这两个连续性，得到 (hf.comp continuous_fst).prod_mk (hf.comp continuous_snd) : Continuous (fun p : X × X ↦ (f p.1, f p.2))，再复合一次即可得到完整证明。

example {X Y : Type*} [MetricSpace X] [MetricSpace Y] {f : X → Y} (hf : Continuous f) :
    Continuous fun p : X × X ↦ dist (f p.1) (f p.2) :=
  continuous_dist.comp ((hf.comp continuous_fst).prodMk (hf.comp continuous_snd))

即使像上面那样大量使用点记号，通过 Continuous.comp 把 Continuous.prod_mk 与 continuous_dist 组合起来仍显得笨拙。更严重的问题是，这个漂亮的证明需要相当多预先规划。Lean 接受上面的证明项，是因为它是一个完整项，证明了一个与目标按定义等价的陈述；关键是展开函数复合的定义。事实上，我们的目标函数 fun p : X × X ↦ dist (f p.1) (f p.2) 并不是以复合形式呈现的。我们给出的证明项证明的是 dist ∘ (fun p : X × X ↦ (f p.1, f p.2)) 的连续性，而它恰好与目标函数按定义相等。但如果尝试用策略逐步构造这个证明，先写 apply continuous_dist.comp，Lean 的 elaborator 将无法识别出复合，从而拒绝应用该引理。当涉及类型的积时，这种情况尤其明显。

这里更适合应用的引理是 Continuous.dist {f g : X → Y} : Continuous f → Continuous g → Continuous (fun x ↦ dist (f x) (g x))；它对 Lean 的 elaborator 更友好，并且在直接给出完整证明项时也能产生更短的证明，下面两个对上述陈述的新证明展示了这一点：

example {X Y : Type*} [MetricSpace X] [MetricSpace Y] {f : X → Y} (hf : Continuous f) :
    Continuous fun p : X × X ↦ dist (f p.1) (f p.2) := by
  apply Continuous.dist
  exact hf.comp continuous_fst
  exact hf.comp continuous_snd

example {X Y : Type*} [MetricSpace X] [MetricSpace Y] {f : X → Y} (hf : Continuous f) :
    Continuous fun p : X × X ↦ dist (f p.1) (f p.2) :=
  (hf.comp continuous_fst).dist (hf.comp continuous_snd)

注意，如果没有来自复合的 elaboration 问题，压缩证明的另一种方式是使用 Continuous.prod_map；它有时很有用，并给出

既要在更利于 elaboration 的版本和更短的输入版本之间取舍，多少令人遗憾；因此我们用 Continuous.fst' 提供的最后一点压缩来结束这段讨论。它允许把 hf.comp continuous_fst 压缩为 hf.fst'（对 snd 也类似），从而得到最后一个证明；此时已经接近晦涩。

    Continuous fun p : X × X ↦ dist (f p.1) (f p.2) :=
  hf.fst'.dist hf.snd'

11.2.2. 球、开集与闭集

一旦有了距离函数，最重要的几何定义就是（开）球和闭球。

variable (r : ℝ)

example : Metric.ball a r = { b | dist b a < r } :=
  rfl

example : Metric.closedBall a r = { b | dist b a ≤ r } :=
  rfl

注意，这里的 r 是任意实数，没有符号限制。当然，某些陈述确实需要半径条件。

example (hr : 0 < r) : a ∈ Metric.ball a r :=
  Metric.mem_ball_self hr

example (hr : 0 ≤ r) : a ∈ Metric.closedBall a r :=
  Metric.mem_closedBall_self hr

有了球之后，就可以定义开集。开集实际上是在下一节涉及的更一般设置中定义的，但我们有用球重新表述该定义的引理。

example (s : Set X) : IsOpen s ↔ ∀ x ∈ s, ∃ ε > 0, Metric.ball x ε ⊆ s :=
  Metric.isOpen_iff

于是，闭集就是补集为开集的集合。它们的重要性质是对极限封闭。一个集合的闭包是包含它的最小闭集。

11.2.3. 紧性

紧性是重要的拓扑概念。它把度量空间中具有与实数线段相同性质的子集同其他区间区分开来：

  • 任意取值于紧集的序列都有一个在该集合中收敛的子序列。
  • 任意定义在非空紧集上且取实数值的连续函数都是有界的，并且在某处达到其界（这称为极值定理）。
  • 紧集是闭集。

先检查实数中的单位区间确实是紧集，然后在一般度量空间中检查上面的紧集性质。第二个陈述只需要在给定集合上的连续性，所以将使用 ContinuousOn 而不是 Continuous，并且会分别给出最小值和最大值的陈述。当然，这些结果都由更一般的版本推出，其中一些会在后面的章节中讨论。

example : IsCompact (Set.Icc 0 1 : Set ℝ) :=
  isCompact_Icc

example {s : Set X} (hs : IsCompact s) {u : ℕ → X} (hu : ∀ n, u n ∈ s) :
    ∃ a ∈ s, ∃ φ : ℕ → ℕ, StrictMono φ ∧ Tendsto (u ∘ φ) atTop (𝓝 a) :=
  hs.tendsto_subseq hu

example {s : Set X} (hs : IsCompact s) (hs' : s.Nonempty) {f : X → ℝ}
      (hfs : ContinuousOn f s) :
    ∃ x ∈ s, ∀ y ∈ s, f x ≤ f y :=
  hs.exists_isMinOn hs' hfs

example {s : Set X} (hs : IsCompact s) (hs' : s.Nonempty) {f : X → ℝ}
      (hfs : ContinuousOn f s) :
    ∃ x ∈ s, ∀ y ∈ s, f y ≤ f x :=
  hs.exists_isMaxOn hs' hfs

example {s : Set X} (hs : IsCompact s) : IsClosed s :=
  hs.isClosed

也可以用一个额外的取值于 Prop 的类型类，指定一个度量空间整体上是紧的：

example {X : Type*} [MetricSpace X] [CompactSpace X] : IsCompact (univ : Set X) :=
  isCompact_univ

在紧度量空间中，任意闭集都是紧的；这就是 IsClosed.isCompact。

11.2.5. 完备性

度量空间中的柯西序列，是各项彼此越来越接近的序列。表达这一思想有若干等价方式。特别地，收敛序列是柯西序列。反命题只在所谓完备空间中成立。

example (u : ℕ → X) :
    CauchySeq u ↔ ∀ ε > 0, ∃ N : ℕ, ∀ m ≥ N, ∀ n ≥ N, dist (u m) (u n) < ε :=
  Metric.cauchySeq_iff

example (u : ℕ → X) :
    CauchySeq u ↔ ∀ ε > 0, ∃ N : ℕ, ∀ n ≥ N, dist (u n) (u N) < ε :=
  Metric.cauchySeq_iff'

example [CompleteSpace X] (u : ℕ → X) (hu : CauchySeq u) :
    ∃ x, Tendsto u atTop (𝓝 x) :=
  cauchySeq_tendsto_of_complete hu

我们将通过证明一个方便的判别准则来练习使用这个定义；它是 Mathlib 中某个判别准则的特例。这也是在几何语境中练习使用大和式的好机会。除了滤子部分的说明之外，你可能还需要 tendsto_pow_atTop_nhds_zero_of_lt_one、Tendsto.mul 和 dist_le_range_sum_dist。

theorem cauchySeq_of_le_geometric_two' {u : ℕ → X}
    (hu : ∀ n : ℕ, dist (u n) (u (n + 1)) ≤ (1 / 2) ^ n) : CauchySeq u := by
  rw [Metric.cauchySeq_iff']
  intro ε ε_pos
  obtain ⟨N, hN⟩ : ∃ N : ℕ, 1 / 2 ^ N * 2 < ε := by sorry
  use N
  intro n hn
  obtain ⟨k, rfl : n = N + k⟩ := le_iff_exists_add.mp hn
  calc
    dist (u (N + k)) (u N) = dist (u (N + 0)) (u (N + k)) := sorry
    _ ≤ ∑ i  ∈ range k, dist (u (N + i)) (u (N + (i + 1))) := sorry
    _ ≤ ∑ i  ∈ range k, (1 / 2 : ℝ) ^ (N + i) := sorry
    _ = 1 / 2 ^ N * ∑ i  ∈ range k, (1 / 2 : ℝ) ^ i := sorry
    _ ≤ 1 / 2 ^ N * 2 := sorry
    _ < ε := sorry

现在准备迎接本节最后的难题：完备度量空间的贝尔定理！下面的证明骨架展示了一些有趣技巧。它使用感叹号版本的 choose 策略（你应当尝试去掉这个感叹号看看会发生什么），并展示如何在证明中间使用 Nat.rec_on 归纳地定义某个对象。

open Metric

example [CompleteSpace X] (f : ℕ → Set X) (ho : ∀ n, IsOpen (f n)) (hd : ∀ n, Dense (f n)) :
    Dense (⋂ n, f n) := by
  let B : ℕ → ℝ := fun n ↦ (1 / 2) ^ n
  have Bpos : ∀ n, 0 < B n
  sorry
  /- Translate the density assumption into two functions `center` and `radius` associating
    to any n, x, δ, δpos a center and a positive radius such that
    `closedBall center radius` is included both in `f n` and in `closedBall x δ`.
    We can also require `radius ≤ (1/2)^(n+1)`, to ensure we get a Cauchy sequence later. -/
  have :
    ∀ (n : ℕ) (x : X),
      ∀ δ > 0, ∃ y : X, ∃ r > 0, r ≤ B (n + 1) ∧ closedBall y r ⊆ closedBall x δ ∩ f n :=
    by sorry
  choose! center radius Hpos HB Hball using this
  intro x
  rw [mem_closure_iff_nhds_basis nhds_basis_closedBall]
  intro ε εpos
  /- `ε` is positive. We have to find a point in the ball of radius `ε` around `x`
    belonging to all `f n`. For this, we construct inductively a sequence
    `F n = (c n, r n)` such that the closed ball `closedBall (c n) (r n)` is included
    in the previous ball and in `f n`, and such that `r n` is small enough to ensure
    that `c n` is a Cauchy sequence. Then `c n` converges to a limit which belongs
    to all the `f n`. -/
  let F : ℕ → X × ℝ := fun n ↦
    Nat.recOn n (Prod.mk x (min ε (B 0)))
      fun n p ↦ Prod.mk (center n p.1 p.2) (radius n p.1 p.2)
  let c : ℕ → X := fun n ↦ (F n).1
  let r : ℕ → ℝ := fun n ↦ (F n).2
  have rpos : ∀ n, 0 < r n := by sorry
  have rB : ∀ n, r n ≤ B n := by sorry
  have incl : ∀ n, closedBall (c (n + 1)) (r (n + 1)) ⊆ closedBall (c n) (r n) ∩ f n := by
    sorry
  have cdist : ∀ n, dist (c n) (c (n + 1)) ≤ B n := by sorry
  have : CauchySeq c := cauchySeq_of_le_geometric_two' cdist
  -- as the sequence `c n` is Cauchy in a complete space, it converges to a limit `y`.
  rcases cauchySeq_tendsto_of_complete this with ⟨y, ylim⟩
  -- this point `y` will be the desired point. We will check that it belongs to all
  -- `f n` and to `ball x ε`.
  use y
  have I : ∀ n, ∀ m ≥ n, closedBall (c m) (r m) ⊆ closedBall (c n) (r n) := by sorry
  have yball : ∀ n, y ∈ closedBall (c n) (r n) := by sorry
  sorry

11.3. 拓扑空间

11.3.1. 基本概念

现在进一步提高一般性，引入拓扑空间。我们将回顾定义拓扑空间的两种主要方式，然后解释为什么拓扑空间范畴比度量空间范畴表现得好得多。注意，这里不会使用 Mathlib 的范畴论，只是采取一种略带范畴意味的观点。

从度量空间过渡到拓扑空间的第一种理解方式是：只保留开集概念（或等价地，闭集概念）。从这个观点看，拓扑空间是一个类型，配备一族称为开集的集合。这个集合族必须满足下面给出的若干公理（这份数据略有冗余，但我们将忽略这一点）。

section
variable {X : Type*} [TopologicalSpace X]

example : IsOpen (univ : Set X) :=
  isOpen_univ

example : IsOpen (∅ : Set X) :=
  isOpen_empty

example {ι : Type*} {s : ι → Set X} (hs : ∀ i, IsOpen (s i)) : IsOpen (⋃ i, s i) :=
  isOpen_iUnion hs

example {ι : Type*} [Fintype ι] {s : ι → Set X} (hs : ∀ i, IsOpen (s i)) :
    IsOpen (⋂ i, s i) :=
  isOpen_iInter_of_finite hs

闭集于是定义为补集为开集的集合。拓扑空间之间的函数称为（全局）连续，当且仅当所有开集的原像都是开集。

variable {Y : Type*} [TopologicalSpace Y]

example {f : X → Y} : Continuous f ↔ ∀ s, IsOpen s → IsOpen (f ⁻¹' s) :=
  continuous_def

用这个定义已经可以看到，与度量空间相比，拓扑空间只保留了足以讨论连续函数的信息：一个类型上的两个拓扑结构相同，当且仅当它们具有相同的连续函数（事实上，恒等函数在两个方向都连续，当且仅当这两个结构具有相同的开集）。

然而，一旦转向一点处连续，就能看到基于开集的方法的局限。在 Mathlib 中，我们经常把拓扑空间理解为这样的类型：每个点 x 都配有一个邻域滤子 𝓝 x（相应函数 X → Filter X 满足后面解释的若干条件）。请回忆滤子部分中这些对象的两个相关作用。首先，𝓝 x 被看作 X 中接近 x 的点组成的广义集合。其次，它给出一种方式，用于表达对任意谓词 P : X → Prop，该谓词在足够接近 x 的点处成立。现在陈述 f : X → Y 在 x 处连续。完全滤子化的表述是：f 下接近 x 的点组成的广义集合的直接像，包含于接近 f x 的点组成的广义集合。回忆这可以写作 map f (𝓝 x) ≤ 𝓝 (f x) 或 Tendsto f (𝓝 x) (𝓝 (f x))。

example {f : X → Y} {x : X} : ContinuousAt f x ↔ map f (𝓝 x) ≤ 𝓝 (f x) :=
  Iff.rfl

也可以同时把邻域看作普通集合，而把邻域滤子看作广义集合来表述：“对 f x 的任意邻域 U，所有足够接近 x 的点都会被送入 U”。注意，证明再次是 Iff.rfl；这个观点与前一个观点按定义等价。

example {f : X → Y} {x : X} : ContinuousAt f x ↔ ∀ U ∈ 𝓝 (f x), ∀ᶠ x in 𝓝 x, f x ∈ U :=
  Iff.rfl

现在解释如何在两种观点之间转换。用开集来表述时，可以简单地把 𝓝 x 的成员定义为包含某个含有 x 的开集的集合。

example {x : X} {s : Set X} : s ∈ 𝓝 x ↔ ∃ t, t ⊆ s ∧ IsOpen t ∧ x ∈ t :=
  mem_nhds_iff

反向转换时，需要讨论 𝓝 : X → Filter X 必须满足什么条件，才能成为某个拓扑的邻域函数。

第一个约束是：把 𝓝 x 看作广义集合时，它包含把单点集 {x} 看作广义集合得到的 pure x（解释这个奇怪名称会偏离太远，所以暂且接受它）。另一种说法是：如果某个谓词对接近 x 的点成立，那么它在 x 处成立。

example (x : X) : pure x ≤ 𝓝 x :=
  pure_le_nhds x

example (x : X) (P : X → Prop) (h : ∀ᶠ y in 𝓝 x, P y) : P x :=
  h.self_of_nhds

然后有一个更微妙的要求：对任意谓词 P : X → Prop 和任意 x，若对接近 x 的 y 有 P y 成立，则对接近 x 的 y 且接近 y 的 z，有 P z 成立。更精确地说，有：

example {P : X → Prop} {x : X} (h : ∀ᶠ y in 𝓝 x, P y) : ∀ᶠ y in 𝓝 x, ∀ᶠ z in 𝓝 y, P z :=
  eventually_eventually_nhds.mpr h

这两个结果刻画了哪些函数 X → Filter X 是 X 上某个拓扑空间结构的邻域函数。仍然有一个函数 TopologicalSpace.mkOfNhds : (X → Filter X) → TopologicalSpace X；但只有当输入满足上述两个约束时，它才会把输入作为邻域函数返回。更精确地说，有一个引理 TopologicalSpace.nhds_mkOfNhds，它以另一种方式说明这一点；下一个练习将从我们上面的表述推出这种另一种表述。

example {α : Type*} (n : α → Filter α) (H₀ : ∀ a, pure a ≤ n a)
    (H : ∀ a : α, ∀ p : α → Prop, (∀ᶠ x in n a, p x) → ∀ᶠ y in n a, ∀ᶠ x in n y, p x) :
    ∀ a, ∀ s ∈ n a, ∃ t ∈ n a, t ⊆ s ∧ ∀ a' ∈ t, s ∈ n a' := by
  sorry
end

注意，TopologicalSpace.mkOfNhds 并不经常使用；但仍然值得知道，在何种精确意义上，邻域滤子就是拓扑空间结构的全部内容。

为了在 Mathlib 中高效使用拓扑空间，接下来需要知道的是：我们大量使用 TopologicalSpace : Type u → Type u 的形式性质。从纯数学观点看，这些形式性质非常清楚地说明了拓扑空间如何解决度量空间存在的问题。从这个观点看，拓扑空间解决的问题是：度量空间的函子性很弱，并且总体上范畴性质很差。除此之外，前面也已经讨论过，度量空间包含大量拓扑上无关的几何信息。

先关注函子性。可以在子集上诱导度量空间结构；等价地，可以沿单射拉回度量结构。但这基本上就是全部。度量结构不能沿一般映射拉回，也不能推出，即使沿满射也不行。

特别地，不存在一种合理的方式给度量空间的商或不可数个度量空间的积赋予距离。举例来说，考虑类型 ℝ → ℝ，把它看成由 ℝ 指标的若干份 ℝ 的积。我们希望能够说函数序列的逐点收敛是一种正当的收敛概念。但 ℝ → ℝ 上不存在给出这一收敛概念的距离。相关地，也不存在一种距离，使得映射 f : X → (ℝ → ℝ) 连续当且仅当对每个 t : ℝ，fun x ↦ f x t 连续。

现在回顾用于解决这些问题的数据。首先，可以用任意映射 f : X → Y 从一侧向另一侧推出或拉回拓扑。这两个操作构成伽罗瓦连接。

variable {X Y : Type*}

example (f : X → Y) : TopologicalSpace X → TopologicalSpace Y :=
  TopologicalSpace.coinduced f

example (f : X → Y) : TopologicalSpace Y → TopologicalSpace X :=
  TopologicalSpace.induced f

example (f : X → Y) (T_X : TopologicalSpace X) (T_Y : TopologicalSpace Y) :
    TopologicalSpace.coinduced f T_X ≤ T_Y ↔ T_X ≤ TopologicalSpace.induced f T_Y :=
  coinduced_le_iff_le_induced

这些操作与函数复合相容。照常，推出是协变的，拉回是反变的；参见 coinduced_compose 与 induced_compose。纸笔记号中，我们将用 \(f_*T\) 表示 TopologicalSpace.coinduced f T，用 \(f^*T\) 表示 TopologicalSpace.induced f T。

下一个重要组成部分是：对任意给定的 X，TopologicalSpace X 上有完备格结构。如果把拓扑主要看作开集数据，那么会期望 TopologicalSpace X 上的序关系来自 Set (Set X)，也就是说，会期望当集合 u 对 t 开时它也对 t' 开，从而有 t ≤ t'。然而我们已经知道，Mathlib 比开集更重视邻域；所以对任意 x : X，我们希望从拓扑空间到邻域的映射 fun T : TopologicalSpace X ↦ @nhds X T x 保序。并且我们知道 Filter X 上的序关系被设计成保证 principal : Set X → Filter X 保序，从而可把滤子视为广义集合。因此，我们在 TopologicalSpace X 上使用的序关系与来自 Set (Set X) 的序关系相反。

example {T T' : TopologicalSpace X} : T ≤ T' ↔ ∀ s, T'.IsOpen s → T.IsOpen s :=
  Iff.rfl

现在可以把推出（或拉回）操作与序关系结合起来，重新得到连续性。

example (T_X : TopologicalSpace X) (T_Y : TopologicalSpace Y) (f : X → Y) :
    Continuous f ↔ TopologicalSpace.coinduced f T_X ≤ T_Y :=
  continuous_iff_coinduced_le

借助这个定义以及推出与复合的相容性，可以免费得到如下泛性质：对任意拓扑空间 \(Z\)，函数 \(g : Y → Z\) 关于拓扑 \(f_*T_X\) 连续，当且仅当 \(g ∘ f\) 连续。

\[\begin{split}g \text{ continuous } &⇔ g_*(f_*T_X) ≤ T_Z \\ &⇔ (g ∘ f)_* T_X ≤ T_Z \\ &⇔ g ∘ f \text{ continuous}\end{split}\]
example {Z : Type*} (f : X → Y) (T_X : TopologicalSpace X) (T_Z : TopologicalSpace Z)
      (g : Y → Z) :
    @Continuous Y Z (TopologicalSpace.coinduced f T_X) T_Z g ↔
      @Continuous X Z T_X T_Z (g ∘ f) := by
  rw [continuous_iff_coinduced_le, coinduced_compose, continuous_iff_coinduced_le]

于是已经得到了商拓扑（把投影映射作为 f）。这还没有用到对所有 X，TopologicalSpace X 都是完备格。现在看看所有这些结构如何用抽象废话证明积拓扑的存在。上面考虑过 ℝ → ℝ 的情形；现在考虑一般情形：对某个 ι : Type* 和 X : ι → Type*，取 Π i, X i。我们希望对任意拓扑空间 Z 和任意函数 f : Z → Π i, X i，f 连续当且仅当对所有 i，(fun x ↦ x i) ∘ f 连续。用 \(p_i\) 表示投影 (fun (x : Π i, X i) ↦ x i)，在纸面上探索这个约束：

\[\begin{split}(∀ i, p_i ∘ f \text{ continuous}) &⇔ ∀ i, (p_i ∘ f)_* T_Z ≤ T_{X_i} \\ &⇔ ∀ i, (p_i)_* f_* T_Z ≤ T_{X_i}\\ &⇔ ∀ i, f_* T_Z ≤ (p_i)^*T_{X_i}\\ &⇔ f_* T_Z ≤ \inf \left[(p_i)^*T_{X_i}\right]\end{split}\]

因此可以看出，Π i, X i 上我们想要的拓扑是什么：

example (ι : Type*) (X : ι → Type*) (T_X : ∀ i, TopologicalSpace (X i)) :
    (Pi.topologicalSpace : TopologicalSpace (∀ i, X i)) =
      ⨅ i, TopologicalSpace.induced (fun x ↦ x i) (T_X i) :=
  rfl

至此，我们结束了这一巡览：Mathlib 如何通过更具函子性的理论、以及在任意固定类型上具有完备格结构，来理解拓扑空间如何修正度量空间理论的缺陷。

11.3.2. 分离性与可数性

我们看到，拓扑空间范畴具有非常好的性质。代价是会出现相当病态的拓扑空间。可以对拓扑空间施加若干假设，使其行为更接近度量空间。最重要的是 T2Space，也称为“Hausdorff”，它保证极限唯一。更强的分离性质是 T3Space，它还保证 RegularSpace 性质：每个点都有由闭邻域组成的基。

example [TopologicalSpace X] [T2Space X] {u : ℕ → X} {a b : X} (ha : Tendsto u atTop (𝓝 a))
    (hb : Tendsto u atTop (𝓝 b)) : a = b :=
  tendsto_nhds_unique ha hb

example [TopologicalSpace X] [RegularSpace X] (a : X) :
    (𝓝 a).HasBasis (fun s : Set X ↦ s ∈ 𝓝 a ∧ IsClosed s) id :=
  closed_nhds_basis a

注意，在每个拓扑空间中，每个点按定义都有一个开邻域基。

example [TopologicalSpace X] {x : X} :
    (𝓝 x).HasBasis (fun t : Set X ↦ t ∈ 𝓝 x ∧ IsOpen t) id :=
  nhds_basis_opens' x

现在的主要目标，是证明允许由连续性作延拓的基本定理。取布尔巴基《一般拓扑》I.8.5 定理 1 中的非平凡方向：

设 \(X\) 为拓扑空间，\(A\) 为 \(X\) 的稠密子集，\(f : A → Y\) 为从 \(A\) 到 \(T_3\) 空间 \(Y\) 的连续映射。若对 \(X\) 中每个 \(x\)，当 \(y\) 在 \(A\) 中趋向 \(x\) 时，\(f(y)\) 在 \(Y\) 中趋向某个极限，则存在 \(f\) 到 \(X\) 的连续延拓 \(φ\)。

实际上，Mathlib 包含上面引理的更一般版本 IsDenseInducing.continuousAt_extend；但这里我们坚持使用布尔巴基版本。

请记住，给定 A : Set X，↥A 是与 A 对应的子类型；Lean 会在需要时自动插入那个特殊的上箭头。并且（包含）强制映射是 (↑) : A → X。假设“在保持于 \(A\) 中的同时趋向 \(x\)”对应于拉回滤子 comap (↑) (𝓝 x)。

先证明一个辅助引理，把它抽出是为了简化上下文（特别是这里不需要 Y 是拓扑空间）。

theorem aux {X Y A : Type*} [TopologicalSpace X] {c : A → X}
      {f : A → Y} {x : X} {F : Filter Y}
      (h : Tendsto f (comap c (𝓝 x)) F) {V' : Set Y} (V'_in : V' ∈ F) :
    ∃ V ∈ 𝓝 x, IsOpen V ∧ c ⁻¹' V ⊆ f ⁻¹' V' := by
  sorry

现在转向连续延拓定理的主证明。

当 Lean 需要 ↥A 上的拓扑时，它会自动使用诱导拓扑。唯一相关的引理是 nhds_induced (↑) : ∀ a : ↥A, 𝓝 a = comap (↑) (𝓝 ↑a)（这实际上是关于诱导拓扑的一般引理）。

证明提纲如下：

主假设和选择公理给出一个函数 φ，使得 ∀ x, Tendsto f (comap (↑) (𝓝 x)) (𝓝 (φ x))（由于 Y 是 Hausdorff 的，φ 完全确定；但直到要证明 φ 确实延拓 f 时，才需要这一点）。

先证明 φ 连续。固定任意 x : X。由于 Y 是正则的，只需检查对 φ x 的每个闭邻域 V'，都有 φ ⁻¹' V' ∈ 𝓝 x。极限假设通过上面的辅助引理给出某个 V ∈ 𝓝 x，满足 IsOpen V ∧ (↑) ⁻¹' V ⊆ f ⁻¹' V'。由于 V ∈ 𝓝 x，只需证明 V ⊆ φ ⁻¹' V'，即 ∀ y ∈ V, φ y ∈ V'。固定 V 中的 y。因为 V 是开的，它是 y 的邻域。特别地，(↑) ⁻¹' V ∈ comap (↑) (𝓝 y)；进而有 f ⁻¹' V' ∈ comap (↑) (𝓝 y)。此外，由于 A 稠密，comap (↑) (𝓝 y) ≠ ⊥。因为已知 Tendsto f (comap (↑) (𝓝 y)) (𝓝 (φ y))，这推出 φ y ∈ closure V'；又因为 V' 是闭的，便证明了 φ y ∈ V'。

剩下的是证明 φ 延拓 f。这一步中会用到 f 的连续性，以及 Y 是 Hausdorff 的事实。

example [TopologicalSpace X] [TopologicalSpace Y] [T3Space Y] {A : Set X}
    (hA : ∀ x, x ∈ closure A) {f : A → Y} (f_cont : Continuous f)
    (hf : ∀ x : X, ∃ c : Y, Tendsto f (comap (↑) (𝓝 x)) (𝓝 c)) :
    ∃ φ : X → Y, Continuous φ ∧ ∀ a : A, φ a = f a := by
  sorry

#check HasBasis.tendsto_right_iff

除了分离性之外，使拓扑空间更接近度量空间的主要假设类型是可数性假设。其中最主要的是第一可数性，它要求每个点都有一个可数邻域基。特别地，这保证可以用序列来理解集合闭包。

example [TopologicalSpace X] [FirstCountableTopology X]
      {s : Set X} {a : X} :
    a ∈ closure s ↔ ∃ u : ℕ → X, (∀ n, u n ∈ s) ∧ Tendsto u atTop (𝓝 a) :=
  mem_closure_iff_seq_limit

11.3.3. 紧性

现在讨论拓扑空间中的紧性如何定义。照常，有多种理解方式，而 Mathlib 采用滤子版本。

首先需要定义滤子的聚点。给定拓扑空间 X 上的滤子 F，点 x : X 称为 F 的聚点，当且仅当把 F 视为广义集合时，它与接近 x 的点组成的广义集合有非空交。

于是可以说，集合 s 是紧的，当且仅当每个包含在 s 中的非空广义集合 F，即每个满足 F ≤ 𝓟 s 的滤子，都在 s 中有聚点。

variable [TopologicalSpace X]

example {F : Filter X} {x : X} : ClusterPt x F ↔ NeBot (𝓝 x ⊓ F) :=
  Iff.rfl

example {s : Set X} :
    IsCompact s ↔ ∀ (F : Filter X) [NeBot F], F ≤ 𝓟 s → ∃ a ∈ s, ClusterPt a F :=
  Iff.rfl

例如，若 F 是 map u atTop，即 atTop 这个非常大的自然数组成的广义集合在 u : ℕ → X 下的像，则假设 F ≤ 𝓟 s 意味着对充分大的 n，u n 属于 s。说 x 是 map u atTop 的聚点，就是说非常大自然数的像与接近 x 的点的集合相交。当 𝓝 x 有可数基时，可以把它解释为：u 有一个收敛到 x 的子序列；这就恢复了度量空间中紧性的样子。

example [FirstCountableTopology X] {s : Set X} {u : ℕ → X} (hs : IsCompact s)
    (hu : ∀ n, u n ∈ s) : ∃ a ∈ s, ∃ φ : ℕ → ℕ, StrictMono φ ∧ Tendsto (u ∘ φ) atTop (𝓝 a) :=
  hs.tendsto_subseq hu

聚点在连续函数下表现良好。

variable [TopologicalSpace Y]

example {x : X} {F : Filter X} {G : Filter Y} (H : ClusterPt x F) {f : X → Y}
    (hfx : ContinuousAt f x) (hf : Tendsto f F G) : ClusterPt (f x) G :=
  ClusterPt.map H hfx hf

作为练习，我们将证明紧集在连续映射下的像是紧的。除了已经见过的内容之外，还应使用 Filter.push_pull 和 NeBot.of_map。

example [TopologicalSpace Y] {f : X → Y} (hf : Continuous f) {s : Set X} (hs : IsCompact s) :
    IsCompact (f '' s) := by
  intro F F_ne F_le
  have map_eq : map f (𝓟 s ⊓ comap f F) = 𝓟 (f '' s) ⊓ F := by sorry
  have Hne : (𝓟 s ⊓ comap f F).NeBot := by sorry
  have Hle : 𝓟 s ⊓ comap f F ≤ 𝓟 s := inf_le_left
  sorry

也可以用开覆盖来表达紧性：s 是紧的，当且仅当每个覆盖 s 的开集族都有一个有限子族仍然覆盖它。

example {ι : Type*} {s : Set X} (hs : IsCompact s) (U : ι → Set X) (hUo : ∀ i, IsOpen (U i))
    (hsU : s ⊆ ⋃ i, U i) : ∃ t : Finset ι, s ⊆ ⋃ i ∈ t, U i :=
  hs.elim_finite_subcover U hUo hsU
