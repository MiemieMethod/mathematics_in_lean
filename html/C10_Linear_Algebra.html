10. 线性代数

10.1. 向量空间与线性映射

10.1.1. 向量空间

我们将直接从抽象线性代数开始，即任意域上的向量空间中的线性代数。不过，关于矩阵的内容可见Section 10.4.1，该节在逻辑上并不依赖这套抽象理论。实际上，Mathlib 处理的是更一般的线性代数，其中会出现“模”这一术语；但暂时我们可以把这看作一种略显特殊的命名习惯。

要表达“设 \(K\) 为域，设 \(V\) 为 \(K\) 上的向量空间”（并使它们成为后续结果的隐式参数），可以写作：

variable {K : Type*} [Field K] {V : Type*} [AddCommGroup V] [Module K V]

我们在Chapter 8中解释过，为什么需要两个分开的类型类 [AddCommGroup V] [Module K V]。简短地说，数学上我们想表达的是：具有 \(K\)-向量空间结构蕴含具有加法交换群结构。我们当然可以把这一点告诉 Lean。但这样一来，每当 Lean 需要在某个类型 \(V\) 上寻找这样的群结构时，它就会开始寻找相对于某个完全未指定的域 \(K\) 的向量空间结构，而这个域无法从 \(V\) 推断出来。这会严重损害类型类合成系统的工作。

向量 v 乘以标量 a 记作 a • v。下面的例子列出这一运算与加法相互作用的若干代数规则。当然，simp 或 apply? 都可以找到这些证明。还有一个 module 策略，它会解决由向量空间和域的公理推出的目标，正如在交换环中使用 ring 策略、在群中使用 group 策略一样。不过仍然有必要记住，在引理名称中，标量乘法通常缩写为 smul。

example (a : K) (u v : V) : a • (u + v) = a • u + a • v :=
  smul_add a u v

example (a b : K) (u : V) : (a + b) • u = a • u + b • u :=
  add_smul a b u

example (a b : K) (u : V) : a • b • u = b • a • u :=
  smul_comm a b u

给较熟悉代数的读者补充一点：正如术语所暗示的，Mathlib 的线性代数也涵盖（不一定交换的）环上的模。事实上，它甚至涵盖半环上的半模。若你认为自己并不需要这种一般性，可以思考下面的例子；它很好地概括了理想作用于子模时的大量代数规则：

example {R M : Type*} [CommSemiring R] [AddCommMonoid M] [Module R M] :
    Module (Ideal R) (Submodule R M) :=
  inferInstance

要表达 M 是环 R 上的模，可以写作：

section
variable {R : Type*} [Ring R] {M : Type*} [AddCommGroup M] [Module R M]
end

如果把这与声明向量空间的方式比较，可以看到两个差异。除了变量改名之外，唯一的变化就是把假设 Field K 替换为假设 Ring R。

要表达 M 是半环 R 上的半模，我们把假设 Ring R 替换为假设 Semiring R，同时把 AddCommGroup M 替换为 AddCommMonoid M；这反映了半模没有取负运算：

section
variable {R : Type*} [Semiring R] {M : Type*} [AddCommMonoid M] [Module R M]
end

即使 R 有取负运算，Lean 也不能自动推断 M 有取负运算，因为对任意给定的 M，它先验上无法猜出正确的 R。因此，如果我们声明 R 是环，且 M 是 R 上的模，那么要么必须声明 M 是一个 AddCommGroup，要么手动把相应实例加入上下文：

example {R : Type*} [Ring R] {M : Type*} [AddCommMonoid M] [Module R M] : M → M :=
  -- Make sure to use `letI` (instead of `haveI` or `let`) to make it easier to
  -- identify the existing additive structure from the `AddCommMonoid M` hypothesis
  -- with the new one introduced in the line below.
  letI : AddCommGroup M := Module.addCommMonoidToAddCommGroup R
  fun x => - x

10.1.2. 线性映射

接下来需要线性映射。与群态射一样，Mathlib 中的线性映射是打包映射，也就是由一个映射及其线性性质的证明组成的对象。在应用这些打包映射时，它们会被转换为通常的函数。关于这种设计，参见Chapter 8。

两个 K-向量空间 V 与 W 之间的线性映射类型记作 V →ₗ[K] W。下标 l 表示 linear。初看之下，在这个记号中指定 K 或许有些奇怪；但当多个域同时出现时，这一点非常关键。例如，从 \(ℂ\) 到 \(ℂ\) 的实线性映射是所有形如 \(z ↦ az + b\bar{z}\) 的映射，而复线性映射只有形如 \(z ↦ az\) 的映射，这一区别在复分析中至关重要。

variable {W : Type*} [AddCommGroup W] [Module K W]

variable (φ : V →ₗ[K] W)

example (a : K) (v : V) : φ (a • v) = a • φ v :=
  map_smul φ a v

example (v w : V) : φ (v + w) = φ v + φ w :=
  map_add φ v w

注意，V →ₗ[K] W 本身也带有有意义的代数结构（这也是将这些映射打包的动机之一）。它是一个 K-向量空间，因此可以对线性映射相加，也可以用标量乘以线性映射。

variable (ψ : V →ₗ[K] W)

#check (2 • φ + ψ : V →ₗ[K] W)

使用打包映射的一个代价是，不能直接使用通常的函数复合。我们需要使用 LinearMap.comp，或者使用记号 ∘ₗ。

variable (θ : W →ₗ[K] V)

#check (φ.comp θ : W →ₗ[K] W)
#check (φ ∘ₗ θ : W →ₗ[K] W)

构造线性映射主要有两种方式。第一种是直接构造该结构，给出函数以及线性性的证明。照常，结构代码操作可以简化这一步：你可以输入 example : V →ₗ[K] V := _，然后在下划线处使用“Generate a skeleton”代码操作生成骨架。

example : V →ₗ[K] V where
  toFun v := 3 • v
  map_add' _ _ := smul_add ..
  map_smul' _ _ := smul_comm ..

你可能会疑惑，为什么 LinearMap 的证明字段名称以撇号结尾。这是因为它们定义时，到函数的强制转换尚未定义，所以这些字段要用 LinearMap.toFun 来表述。随后，它们又以关于到函数的强制转换的形式被重新陈述为 LinearMap.map_add 和 LinearMap.map_smul。事情还没有到此为止。我们还希望有一个适用于任何保持加法的（打包）映射的 map_add 版本，例如加法群态射、线性映射、连续线性映射、K-代数映射等。这个版本就是根命名空间中的 map_add。中间版本 LinearMap.map_add 有些冗余，但允许使用点记号，有时很方便。map_smul 也有类似情况；一般框架见Chapter 8。

#check (φ.map_add' : ∀ x y : V, φ.toFun (x + y) = φ.toFun x + φ.toFun y)
#check (φ.map_add : ∀ x y : V, φ (x + y) = φ x + φ y)
#check (map_add φ : ∀ x y : V, φ (x + y) = φ x + φ y)

还可以借助各种组合子，从 Mathlib 中已经定义好的线性映射构造新的线性映射。例如，上面的例子已经作为 LinearMap.lsmul K V 3 存在。这里把 K 和 V 作为显式参数有若干原因。最直接的原因是，如果只有 LinearMap.lsmul 3，Lean 无法推断 V，甚至也无法推断 K。此外，LinearMap.lsmul K V 本身也是一个有意义的对象：它的类型是 K →ₗ[K] V →ₗ[K] V，意思是一个从 K（视为自身上的向量空间）到从 V 到 V 的 K-线性映射空间的 K-线性映射。

#check (LinearMap.lsmul K V 3 : V →ₗ[K] V)
#check (LinearMap.lsmul K V : K →ₗ[K] V →ₗ[K] V)

还有一个线性同构类型 LinearEquiv，记作 V ≃ₗ[K] W。若 f : V ≃ₗ[K] W，则其逆为 f.symm : W ≃ₗ[K] V；f 与 g 的复合是 f.trans g，也记作 f ≪≫ₗ g；V 的恒等同构是 LinearEquiv.refl K V。必要时，这个类型的元素会自动强制转换为态射和函数。

example (f : V ≃ₗ[K] W) : f ≪≫ₗ f.symm = LinearEquiv.refl K V :=
  f.self_trans_symm

可以使用 LinearEquiv.ofBijective 从一个双射态射构造同构。这样做会使逆函数成为不可计算对象。

noncomputable example (f : V →ₗ[K] W) (h : Function.Bijective f) : V ≃ₗ[K] W :=
  .ofBijective f h

注意，在上面的例子中，Lean 利用已声明的类型来理解 .ofBijective 指的是 LinearEquiv.ofBijective，无需打开任何命名空间。

10.1.3. 向量空间的和与积

可以用直和与直积从已有向量空间构造新的向量空间。先考虑两个向量空间的情形。此时和与积没有区别，可以直接使用积类型。在下面的代码片段中，我们只是展示如何把所有结构映射（嵌入与投影）作为线性映射得到，以及如何使用泛性质来构造进入积和离开和的线性映射；如果你不熟悉范畴论中和与积的区别，可以忽略泛性质这一术语，只关注下面例子的类型。

section binary_product

variable {W : Type*} [AddCommGroup W] [Module K W]
variable {U : Type*} [AddCommGroup U] [Module K U]
variable {T : Type*} [AddCommGroup T] [Module K T]

-- First projection map
example : V × W →ₗ[K] V := LinearMap.fst K V W

-- Second projection map
example : V × W →ₗ[K] W := LinearMap.snd K V W

-- Universal property of the product
example (φ : U →ₗ[K] V) (ψ : U →ₗ[K] W) : U →ₗ[K]  V × W := LinearMap.prod φ ψ

-- The product map does the expected thing, first component
example (φ : U →ₗ[K] V) (ψ : U →ₗ[K] W) : LinearMap.fst K V W ∘ₗ LinearMap.prod φ ψ = φ := rfl

-- The product map does the expected thing, second component
example (φ : U →ₗ[K] V) (ψ : U →ₗ[K] W) : LinearMap.snd K V W ∘ₗ LinearMap.prod φ ψ = ψ := rfl

-- We can also combine maps in parallel
example (φ : V →ₗ[K] U) (ψ : W →ₗ[K] T) : (V × W) →ₗ[K] (U × T) := φ.prodMap ψ

-- This is simply done by combining the projections with the universal property
example (φ : V →ₗ[K] U) (ψ : W →ₗ[K] T) :
  φ.prodMap ψ = (φ ∘ₗ .fst K V W).prod (ψ ∘ₗ .snd K V W) := rfl

-- First inclusion map
example : V →ₗ[K] V × W := LinearMap.inl K V W

-- Second inclusion map
example : W →ₗ[K] V × W := LinearMap.inr K V W

-- Universal property of the sum (aka coproduct)
example (φ : V →ₗ[K] U) (ψ : W →ₗ[K] U) : V × W →ₗ[K] U := φ.coprod ψ

-- The coproduct map does the expected thing, first component
example (φ : V →ₗ[K] U) (ψ : W →ₗ[K] U) : φ.coprod ψ ∘ₗ LinearMap.inl K V W = φ :=
  LinearMap.coprod_inl φ ψ

-- The coproduct map does the expected thing, second component
example (φ : V →ₗ[K] U) (ψ : W →ₗ[K] U) : φ.coprod ψ ∘ₗ LinearMap.inr K V W = ψ :=
  LinearMap.coprod_inr φ ψ

-- The coproduct map is defined in the expected way
example (φ : V →ₗ[K] U) (ψ : W →ₗ[K] U) (v : V) (w : W) :
    φ.coprod ψ (v, w) = φ v + ψ w :=
  rfl

end binary_product

现在转向任意族向量空间的和与积。这里我们只说明如何定义一个向量空间族，以及如何使用和与积的泛性质。注意，直和记号限定在 DirectSum 命名空间中，并且直和的泛性质要求指标类型具有可判定相等性（这多少是实现层面的偶然现象）。

section families
open DirectSum

variable {ι : Type*} [DecidableEq ι]
         (V : ι → Type*) [∀ i, AddCommGroup (V i)] [∀ i, Module K (V i)]

-- The universal property of the direct sum assembles maps from the summands to build
-- a map from the direct sum
example (φ : Π i, (V i →ₗ[K] W)) : (⨁ i, V i) →ₗ[K] W :=
  DirectSum.toModule K ι W φ

-- The universal property of the direct product assembles maps into the factors
-- to build a map into the direct product
example (φ : Π i, (W →ₗ[K] V i)) : W →ₗ[K] (Π i, V i) :=
  LinearMap.pi φ

-- The projection maps from the product
example (i : ι) : (Π j, V j) →ₗ[K] V i := LinearMap.proj i

-- The inclusion maps into the sum
example (i : ι) : V i →ₗ[K] (⨁ i, V i) := DirectSum.lof K ι V i

-- The inclusion maps into the product
example (i : ι) : V i →ₗ[K] (Π i, V i) := LinearMap.single K V i

-- In case `ι` is a finite type, there is an isomorphism between the sum and product.
example [Fintype ι] : (⨁ i, V i) ≃ₗ[K] (Π i, V i) :=
  linearEquivFunOnFintype K ι V

end families

10.2. 子空间与商空间

10.2.1. 子空间

正如线性映射是打包的，V 的线性子空间也是一个打包结构，由 V 中的一个集合（称为该子空间的载体）以及相关闭包性质组成。这里同样出现“模”而不是“向量空间”一词，是因为 Mathlib 的线性代数实际工作在更一般的语境中。

section
variable {K : Type*} [Field K] {V : Type*} [AddCommGroup V] [Module K V]

example (U : Submodule K V) {x y : V} (hx : x ∈ U) (hy : y ∈ U) :
    x + y ∈ U :=
  U.add_mem hx hy

example (U : Submodule K V) {x : V} (hx : x ∈ U) (a : K) :
    a • x ∈ U :=
  U.smul_mem a hx
end

在上面的例子中，必须理解 Submodule K V 是 V 的 K-线性子空间的类型，而不是谓词 IsSubmodule U，其中 U 是 Set V 的元素。Submodule K V 带有到 Set V 的强制转换，并且在 V 上有成员谓词。关于这样做的方式与原因，参见Section 8.3。

当然，两个子空间相同当且仅当它们具有相同元素。这个事实已注册给 ext 策略使用；因此可以像证明两个集合相等一样，用该策略证明两个子空间相等。

例如，要陈述并证明 ℝ 是 ℂ 的一个 ℝ-线性子空间，我们真正想构造的是一个类型为 Submodule ℝ ℂ 的项，使其到 Set ℂ 的投影为 ℝ；更准确地说，是 ℝ 在 ℂ 中的像。

noncomputable example : Submodule ℝ ℂ where
  carrier := Set.range ((↑) : ℝ → ℂ)
  add_mem' := by
    rintro _ _ ⟨n, rfl⟩ ⟨m, rfl⟩
    use n + m
    simp
  zero_mem' := by
    use 0
    simp
  smul_mem' := by
    rintro c - ⟨a, rfl⟩
    use c*a
    simp

Submodule 中证明字段末尾的撇号与 LinearMap 中的情形类似。这些字段用 carrier 字段来表述，因为它们定义时 MemberShip 实例尚未定义。随后，我们上面见过的 Submodule.add_mem、Submodule.zero_mem 和 Submodule.smul_mem 会取代这些字段。

作为一个操作子空间和线性映射的练习，你将定义线性映射下子空间的原像（当然，下面会看到 Mathlib 已经知道这一构造）。请记住，Set.mem_preimage 可以用来改写涉及成员关系和原像的命题。除了上面讨论过的关于 LinearMap 与 Submodule 的引理之外，这是唯一需要的引理。

variable {K : Type*} [Field K] {V : Type*} [AddCommGroup V] [Module K V]

def preimage {W : Type*} [AddCommGroup W] [Module K W] (φ : V →ₗ[K] W) (H : Submodule K W) :
    Submodule K V where
  carrier := φ ⁻¹' H
  zero_mem' := by
    sorry
  add_mem' := by
    sorry
  smul_mem' := by
    sorry

借助类型类，Mathlib 知道向量空间的子空间继承向量空间结构。

example (U : Submodule K V) : Module K U := inferInstance

这个例子有些微妙。对象 U 并不是一个类型，但 Lean 会自动把它强制转换为类型，方式是把它解释为 V 的子类型。因此，上面的例子可以更显式地重述为：

example (U : Submodule K V) : Module K {x : V // x ∈ U} := inferInstance

10.2.2. 完备格结构与内直和

使用类型 Submodule K V 而不是谓词 IsSubmodule : Set V → Prop 的一个重要好处，是可以很容易地给 Submodule K V 赋予额外结构。尤其重要的是，按包含关系它具有完备格结构。例如，我们不会使用一个断言 V 的两个子空间的交仍为子空间的引理，而是使用格运算 ⊓ 来构造交。随后便可以对这一构造应用任意关于格的引理。

让我们检查两个子空间的下确界的底层集合确实按定义就是它们的交。

example (H H' : Submodule K V) :
    ((H ⊓ H' : Submodule K V) : Set V) = (H : Set V) ∩ (H' : Set V) := rfl

对于底层集合的交使用一个不同记号，看起来也许有些奇怪；但这种对应并不能延伸到上确界运算与集合并，因为子空间的并通常不是子空间。相反，需要使用由并集生成的子空间，这由 Submodule.span 完成。

example (H H' : Submodule K V) :
    ((H ⊔ H' : Submodule K V) : Set V) = Submodule.span K ((H : Set V) ∪ (H' : Set V)) := by
  simp [Submodule.span_union]

另一个微妙点是，V 本身并不具有类型 Submodule K V，所以需要一种方式来谈论把 V 视为 V 的子空间。这同样由格结构提供：全子空间就是这个格的顶元。

example (x : V) : x ∈ (⊤ : Submodule K V) := trivial

类似地，这个格的底元是唯一元素为零元素的子空间。

example (x : V) : x ∈ (⊥ : Submodule K V) ↔ x = 0 := Submodule.mem_bot K

特别地，我们可以讨论若干子空间构成（内部）直和的情形。对于两个子空间，使用通用谓词 IsCompl；它适用于任何有界偏序类型。对于一般的子空间族，则使用 DirectSum.IsInternal。

-- If two subspaces are in direct sum then they span the whole space.
example (U V : Submodule K V) (h : IsCompl U V) :
  U ⊔ V = ⊤ := h.sup_eq_top

-- If two subspaces are in direct sum then they intersect only at zero.
example (U V : Submodule K V) (h : IsCompl U V) :
  U ⊓ V = ⊥ := h.inf_eq_bot

section
open DirectSum
variable {ι : Type*} [DecidableEq ι]

-- If subspaces are in direct sum then they span the whole space.
example (U : ι → Submodule K V) (h : DirectSum.IsInternal U) :
  ⨆ i, U i = ⊤ := h.submodule_iSup_eq_top

-- If subspaces are in direct sum then they pairwise intersect only at zero.
example {ι : Type*} [DecidableEq ι] (U : ι → Submodule K V) (h : DirectSum.IsInternal U)
    {i j : ι} (hij : i ≠ j) : U i ⊓ U j = ⊥ :=
  (h.submodule_iSupIndep.pairwiseDisjoint hij).eq_bot

-- Those conditions characterize direct sums.
#check DirectSum.isInternal_submodule_iff_iSupIndep_and_iSup_eq_top

-- The relation with external direct sums: if a family of subspaces is
-- in internal direct sum then the map from their external direct sum into `V`
-- is a linear isomorphism.
noncomputable example {ι : Type*} [DecidableEq ι] (U : ι → Submodule K V)
    (h : DirectSum.IsInternal U) : (⨁ i, U i) ≃ₗ[K] V :=
  LinearEquiv.ofBijective (coeLinearMap U) h
end

10.2.3. 由集合张成的子空间

除了从已有子空间构造子空间之外，还可以从任意集合 s 出发，使用 Submodule.span K s 构造包含 s 的最小子空间。纸笔数学中常说这个空间由 s 中元素的所有线性组合组成。但在形式化中，更高效的做法往往是使用由 Submodule.span_le 表达的泛性质，以及伽罗瓦连接的整个理论。

example {s : Set V} (E : Submodule K V) : Submodule.span K s ≤ E ↔ s ⊆ E :=
  Submodule.span_le

example : GaloisInsertion (Submodule.span K) ((↑) : Submodule K V → Set V) :=
  Submodule.gi K V

当这些还不够时，可以使用相应的归纳原理 Submodule.span_induction。它保证：只要某个性质对零、对 s 中元素成立，并且在加法和标量乘法下稳定，那么它就对 s 的张成子空间中的每个元素成立。

作为练习，我们重新证明 Submodule.mem_sup 的一个方向。请记住，可以使用 module 策略关闭那些由 V 上各代数运算的公理推出的目标。

example {S T : Submodule K V} {x : V} (h : x ∈ S ⊔ T) :
    ∃ s ∈ S, ∃ t ∈ T, x = s + t  := by
  rw [← S.span_eq, ← T.span_eq, ← Submodule.span_union] at h
  induction h using Submodule.span_induction with
  | mem y h =>
      sorry
  | zero =>
      sorry
  | add x y hx hy hx' hy' =>
      sorry
  | smul a x hx hx' =>
      sorry

10.2.4. 子空间的推出与拉回

如前所述，现在说明如何用线性映射推出和拉回子空间。照 Mathlib 的惯例，第一个操作称为 map，第二个操作称为 comap。

section

variable {W : Type*} [AddCommGroup W] [Module K W] (φ : V →ₗ[K] W)

variable (E : Submodule K V) in
#check (Submodule.map φ E : Submodule K W)

variable (F : Submodule K W) in
#check (Submodule.comap φ F : Submodule K V)

注意，这些操作位于 Submodule 命名空间中，因此可以使用点记号，把 E.map φ 写成 Submodule.map φ E 的形式；不过这样读起来相当别扭（尽管一些 Mathlib 贡献者确实采用这种写法）。

特别地，线性映射的值域和核都是子空间。这些特殊情形足够重要，因此有专门的声明。

example : LinearMap.range φ = .map φ ⊤ := LinearMap.range_eq_map φ

example : LinearMap.ker φ = .comap φ ⊥ := Submodule.comap_bot φ -- or `rfl`

注意，不能把 LinearMap.ker φ 写成 φ.ker，因为 LinearMap.ker 还适用于保持更多结构的映射类，所以它并不期待一个类型以 LinearMap 开始的参数，点记号在这里无法工作。然而，在等式右端我们可以使用另一种点记号。由于在 elaboration 左端之后，Lean 期待一个类型为 Submodule K V 的项，它会把 .comap 解释为 Submodule.comap。

下面的引理给出这些子模与 φ 的性质之间的关键关系。

open Function LinearMap

example : Injective φ ↔ ker φ = ⊥ := ker_eq_bot.symm

example : Surjective φ ↔ range φ = ⊤ := range_eq_top.symm

作为练习，我们证明 map 与 comap 的伽罗瓦连接性质。可以使用下面的引理，但这并非必要，因为这些结论按定义即成立。

#check Submodule.mem_map_of_mem
#check Submodule.mem_map
#check Submodule.mem_comap

example (E : Submodule K V) (F : Submodule K W) :
    Submodule.map φ E ≤ F ↔ E ≤ Submodule.comap φ F := by
  sorry

10.2.5. 商空间

商向量空间使用一般的商记号（输入 \quot，而不是通常的 /）。到商空间的投影是 Submodule.mkQ，其泛性质是 Submodule.liftQ。

variable (E : Submodule K V)

example : Module K (V ⧸ E) := inferInstance

example : V →ₗ[K] V ⧸ E := E.mkQ

example : ker E.mkQ = E := E.ker_mkQ

example : range E.mkQ = ⊤ := E.range_mkQ

example (hφ : E ≤ ker φ) : V ⧸ E →ₗ[K] W := E.liftQ φ hφ

example (F : Submodule K W) (hφ : E ≤ .comap φ F) : V ⧸ E →ₗ[K] W ⧸ F := E.mapQ F φ hφ

noncomputable example : (V ⧸ LinearMap.ker φ) ≃ₗ[K] range φ := φ.quotKerEquivRange

open Submodule

#check Submodule.comap_map_eq

10.3. 自同态

线性映射的一个重要特例是自同态：从一个向量空间到自身的线性映射。它们很有趣，因为它们形成一个 K-代数。特别地，可以把 K 系数多项式代入其中，也可以讨论它们的特征值和特征向量。

Mathlib 使用缩写 Module.End K V := V →ₗ[K] V；当大量使用这类对象时，这很方便（尤其是在打开 Module 命名空间之后）。

variable {K : Type*} [Field K] {V : Type*} [AddCommGroup V] [Module K V]

variable {W : Type*} [AddCommGroup W] [Module K W]


open Polynomial Module LinearMap End

example (φ ψ : End K V) : φ * ψ = φ ∘ₗ ψ :=
  End.mul_eq_comp φ ψ -- `rfl` would also work

-- evaluating `P` on `φ`
example (P : K[X]) (φ : End K V) : V →ₗ[K] V :=
  aeval φ P

-- evaluating `X` on `φ` gives back `φ`
example (φ : End K V) : aeval φ (X : K[X]) = φ :=
  aeval_X φ

作为一个操作自同态、子空间和多项式的练习，我们证明（二元）核引理：对任意自同态 \(φ\) 以及任意两个互素多项式 \(P\) 和 \(Q\)，有 \(\ker P(φ) ⊕ \ker Q(φ) = \ker \big(PQ(φ)\big)\)。

注意，IsCoprime x y 被定义为 ∃ a b, a * x + b * y = 1。

#check Submodule.eq_bot_iff
#check Submodule.mem_inf
#check LinearMap.mem_ker

example (P Q : K[X]) (h : IsCoprime P Q) (φ : End K V) : ker (aeval φ P) ⊓ ker (aeval φ Q) = ⊥ := by
  sorry

#check Submodule.add_mem_sup
#check map_mul
#check End.mul_apply
#check LinearMap.ker_le_ker_comp

example (P Q : K[X]) (h : IsCoprime P Q) (φ : End K V) :
    ker (aeval φ P) ⊔ ker (aeval φ Q) = ker (aeval φ (P*Q)) := by
  sorry

现在转向特征空间与特征值的讨论。与自同态 \(φ\) 和标量 \(a\) 对应的特征空间，是 \(φ - aId\) 的核。特征空间对所有 a 都有定义，尽管只有非零时才有意义。不过，特征向量按定义是特征空间中的非零元素。相应的谓词是 End.HasEigenvector。

example (φ : End K V) (a : K) : φ.eigenspace a = LinearMap.ker (φ - a • 1) :=
  End.eigenspace_def

此外还有谓词 End.HasEigenvalue，以及对应的子类型 End.Eigenvalues。

example (φ : End K V) (a : K) : φ.HasEigenvalue a ↔ φ.eigenspace a ≠ ⊥ :=
  Iff.rfl

example (φ : End K V) (a : K) : φ.HasEigenvalue a ↔ ∃ v, φ.HasEigenvector a v  :=
  ⟨End.HasEigenvalue.exists_hasEigenvector, fun ⟨_, hv⟩ ↦ φ.hasEigenvalue_of_hasEigenvector hv⟩

example (φ : End K V) : φ.Eigenvalues = {a // φ.HasEigenvalue a} :=
  rfl

-- Eigenvalue are roots of the minimal polynomial
example (φ : End K V) (a : K) : φ.HasEigenvalue a → (minpoly K φ).IsRoot a :=
  φ.isRoot_of_hasEigenvalue

-- In finite dimension, the converse is also true (we will discuss dimension below)
example [FiniteDimensional K V] (φ : End K V) (a : K) :
    φ.HasEigenvalue a ↔ (minpoly K φ).IsRoot a :=
  φ.hasEigenvalue_iff_isRoot

-- Cayley-Hamilton
example [FiniteDimensional K V] (φ : End K V) : aeval φ φ.charpoly = 0 :=
  φ.aeval_self_charpoly

10.4. 矩阵、基与维数

10.4.1. 矩阵

在介绍抽象向量空间的基之前，我们回到更初等的设置：域 \(K\) 上的 \(K^n\) 中的线性代数。这里的主要对象是向量和矩阵。对具体向量，可以使用 ![…] 记号，其中分量用逗号分隔。对具体矩阵，可以使用 !![…] 记号，行之间用分号分隔，每行中的元素用逗号分隔。当项具有可计算类型，例如 ℕ 或 ℚ 时，可以使用 eval 命令尝试基本运算。

section matrices

-- Adding vectors
#eval ![1, 2] + ![3, 4]  -- ![4, 6]

-- Adding matrices
#eval !![1, 2; 3, 4] + !![3, 4; 5, 6]  -- !![4, 6; 8, 10]

-- Multiplying matrices
#eval !![1, 2; 3, 4] * !![3, 4; 5, 6]  -- !![13, 16; 29, 36]

必须理解，这里使用 #eval 只对探索有意义，并不意在取代 Sage 之类的计算机代数系统。这里用于矩阵的数据表示在计算上完全不高效。它使用函数而不是数组，并针对证明而非计算优化。#eval 所使用的虚拟机也并未为这种用途优化。

注意，矩阵记号列出的是行，但向量记号既不是行向量也不是列向量。矩阵从左侧（相应地，从右侧）与向量相乘时，会把该向量解释为行向量（相应地，列向量）。这对应于操作 Matrix.vecMul（记号为 ᵥ*）和 Matrix.mulVec（记号为 *ᵥ`）。这些记号限定在 Matrix 命名空间中，因此我们需要打开该命名空间。

open Matrix

-- matrices acting on vectors on the left
#eval !![1, 2; 3, 4] *ᵥ ![1, 1] -- ![3, 7]

-- matrices acting on vectors on the left, resulting in a size one matrix
#eval !![1, 2] *ᵥ ![1, 1]  -- ![3]

-- matrices acting on vectors on the right
#eval  ![1, 1, 1] ᵥ* !![1, 2; 3, 4; 5, 6] -- ![9, 12]

为了生成由某个向量指定、且各行或各列相同的矩阵，可以使用 Matrix.replicateRow 和 Matrix.replicateCol；其参数为行或列的指标类型以及该向量。例如，可以得到单行或单列矩阵（更准确地说，是行或列由 Fin 1 作指标的矩阵）。

#eval replicateRow (Fin 1) ![1, 2] -- !![1, 2]

#eval replicateCol (Fin 1) ![1, 2] -- !![1; 2]

其他熟悉的运算包括向量点积、矩阵转置，以及方阵的行列式和迹。

-- vector dot product
#eval ![1, 2] ⬝ᵥ ![3, 4] -- `11`

-- matrix transpose
#eval !![1, 2; 3, 4]ᵀ -- `!![1, 3; 2, 4]`

-- determinant
#eval !![(1 : ℤ), 2; 3, 4].det -- `-2`

-- trace
#eval !![(1 : ℤ), 2; 3, 4].trace -- `5`

当项没有可计算类型时，例如它们是实数，我们不能指望 #eval 提供帮助。此外，这类求值若要用于证明，会显著扩大可信代码基，即检查证明时必须信任的那部分 Lean。

因此，在证明中也应使用 simp 和 norm_num 策略；若只是快速探索，则可使用它们对应的命令形式。

#simp !![(1 : ℝ), 2; 3, 4].det -- `4 - 2*3`

#norm_num !![(1 : ℝ), 2; 3, 4].det -- `-2`

#norm_num !![(1 : ℝ), 2; 3, 4].trace -- `5`

variable (a b c d : ℝ) in
#simp !![a, b; c, d].det -- `a * d – b * c`

方阵的下一个重要运算是求逆。正如数的除法总是有定义，并且在除以零时返回人为指定的值零一样，矩阵求逆也对所有矩阵有定义，并在矩阵不可逆时返回零矩阵。

更精确地说，有一个一般函数 Ring.inverse，它在任意环中完成这一点；对任意矩阵 A，A⁻¹ 被定义为 Ring.inverse A.det • A.adjugate。根据克拉默法则，当 A 的行列式不为零时，这确实是 A 的逆。

#norm_num [Matrix.inv_def] !![(1 : ℝ), 2; 3, 4]⁻¹ -- !![-2, 1; 3 / 2, -(1 / 2)]

当然，这个定义真正有用的情形是可逆矩阵。有一个一般类型类 Invertible 可用于记录这一事实。例如，下一个例子中的 simp 调用会使用引理 inv_mul_of_invertible，而该引理有一个 Invertible 类型类假设；因此只有当类型类合成系统能找到它时，该引理才会触发。这里我们用一个 have 语句使这一事实可用。

example : !![(1 : ℝ), 2; 3, 4]⁻¹ * !![(1 : ℝ), 2; 3, 4] = 1 := by
  have : Invertible !![(1 : ℝ), 2; 3, 4] := by
    apply Matrix.invertibleOfIsUnitDet
    norm_num
  simp

在这个完全具体的情形中，也可以使用 norm_num 机制，并用 apply? 找到最后一行：

example : !![(1 : ℝ), 2; 3, 4]⁻¹ * !![(1 : ℝ), 2; 3, 4] = 1 := by
  norm_num [Matrix.inv_def]
  exact one_fin_two.symm

上面所有具体矩阵的行和列都由某个 n 的 Fin n 作指标（行与列的大小不必相同）。但有时用任意有限类型给矩阵作指标更方便。例如，有限图的邻接矩阵，其行和列自然由图的顶点作指标。

事实上，如果只是想定义矩阵而不定义其上的任何运算，甚至不需要指标类型有限，系数也可以是任意类型，不必有任何代数结构。因此，Mathlib 简单地把 Matrix m n α 定义为 m → n → α，其中 m、n 和 α 是任意类型；到目前为止我们使用的矩阵具有诸如 Matrix (Fin 2) (Fin 2) ℝ 这样的类型。当然，代数运算需要对 m、n 和 α 作更多假设。

注意，我们不直接使用 m → n → α 的主要原因，是为了让类型类系统理解我们的意图。例如，对一个环 R，类型 n → R 带有逐点乘法；类似地，m → n → R 也带有这种运算，而这不是我们希望用于矩阵的乘法。

在下面第一个例子中，我们强制 Lean 展开 Matrix 的定义，使其接受该命题有意义，然后通过检查所有项来证明它。

但接下来的两个例子显示，Lean 对 Fin 2 → Fin 2 → ℤ 使用逐点乘法，而对 Matrix (Fin 2) (Fin 2) ℤ 使用矩阵乘法。

section

example : (fun _ ↦ 1 : Fin 2 → Fin 2 → ℤ) = !![1, 1; 1, 1] := by
  ext i j
  fin_cases i <;> fin_cases j <;> rfl

example : (fun _ ↦ 1 : Fin 2 → Fin 2 → ℤ) * (fun _ ↦ 1 : Fin 2 → Fin 2 → ℤ) = !![1, 1; 1, 1] := by
  ext i j
  fin_cases i <;> fin_cases j <;> rfl

example : !![1, 1; 1, 1] * !![1, 1; 1, 1] = !![2, 2; 2, 2] := by
  norm_num

为了把矩阵定义为函数，同时又不丢失 Matrix 对类型类合成带来的好处，可以使用函数与矩阵之间的等价 Matrix.of。这个等价在背后是用 Equiv.refl 定义的。

例如，可以定义与向量 v 对应的范德蒙德矩阵。

example {n : ℕ} (v : Fin n → ℝ) :
    Matrix.vandermonde v = Matrix.of (fun i j : Fin n ↦ v i ^ (j : ℕ)) :=
  rfl
end
end matrices

10.4.2. 基

现在讨论向量空间的基。非形式化地说，这一概念有多种定义方式。可以使用泛性质；可以说基是一族线性无关且张成全空间的向量；也可以把这些性质合并，直接说基是一族向量，使得每个向量都能唯一地写成基向量的线性组合。还有一种说法是，基给出了与基域 K 的幂的线性同构，其中 K 被视为自身上的向量空间。

这个同构版本实际上是 Mathlib 底层采用的定义，其他刻画则由此证明出来。对于“K 的幂”这一想法，在无限基的情形中必须稍加谨慎。事实上，在这个代数语境中，只有有限线性组合有意义。因此，我们需要的参照向量空间不是若干份 K 的直积，而是直和。若某个类型 ι 为基作指标，可以使用 ⨁ i : ι, K；但实际采用的是更专门的写法 ι →₀ K，意思是“从 ι 到 K 的有限支撑函数”，即在 ι 的某个有限集之外取零的函数（这个有限集不是固定的，而依赖于具体函数）。若这样的函数来自基 B，在向量 v 和 i : ι 处求值，会返回 v 在第 i 个基向量上的分量（或坐标）。

以类型 ι 为指标的 V 作为 K-向量空间的基的类型是 Basis ι K V。相应的同构称为 Basis.repr。

variable {K : Type*} [Field K] {V : Type*} [AddCommGroup V] [Module K V]

section

open Module

variable {ι : Type*} (B : Basis ι K V) (v : V) (i : ι)

-- The basis vector with index ``i``
#check (B i : V)

-- the linear isomorphism with the model space given by ``B``
#check (B.repr : V ≃ₗ[K] ι →₀ K)

-- the component function of ``v``
#check (B.repr v : ι →₀ K)

-- the component of ``v`` with index ``i``
#check (B.repr v i : K)

不从这种同构出发，也可以从一族线性无关且张成全空间的向量 b 出发；这就是 Basis.mk。

这族向量张成全空间的假设写作 ⊤ ≤ Submodule.span K (Set.range b)。这里 ⊤ 是 V 的顶子模，也就是把 V 视为自身的子模。这种写法看起来有些曲折，但下面会看到，它几乎按定义等价于更易读的 ∀ v, v ∈ Submodule.span K (Set.range b)（下面代码片段中的下划线对应无用的信息 v ∈ ⊤）。

noncomputable example (b : ι → V) (b_indep : LinearIndependent K b)
    (b_spans : ∀ v, v ∈ Submodule.span K (Set.range b)) : Basis ι K V :=
  Basis.mk b_indep (fun v _ ↦ b_spans v)

-- The family of vectors underlying the above basis is indeed ``b``.
example (b : ι → V) (b_indep : LinearIndependent K b)
    (b_spans : ∀ v, v ∈ Submodule.span K (Set.range b)) (i : ι) :
    Basis.mk b_indep (fun v _ ↦ b_spans v) i = b i :=
  Basis.mk_apply b_indep (fun v _ ↦ b_spans v) i

特别地，模型向量空间 ι →₀ K 有一个所谓的标准基；其 repr 函数作用于任意向量时就是恒等同构。它称为 Finsupp.basisSingleOne，其中 Finsupp 表示有限支撑函数，而 basisSingleOne 指基向量是在除单个输入值以外处都为零的函数。更精确地说，由 i : ι 作指标的基向量是 Finsupp.single i 1，即在 i 处取值 1、在其他地方取值 0 的有限支撑函数。

variable [DecidableEq ι]

example : Finsupp.basisSingleOne.repr = LinearEquiv.refl K (ι →₀ K) :=
  rfl

example (i : ι) : Finsupp.basisSingleOne i = Finsupp.single i 1 :=
  rfl

当指标类型有限时，不需要有限支撑函数的这些讨论。在这种情况下，可以使用更简单的 Pi.basisFun，它给出整个 ι → K 的一组基。

example [Finite ι] (x : ι → K) (i : ι) : (Pi.basisFun K ι).repr x i = x i := by
  simp

回到抽象向量空间基的一般情形，可以把任意向量表示为基向量的线性组合。先看有限基的简单情形。

example [Fintype ι] : ∑ i : ι, B.repr v i • (B i) = v :=
  B.sum_repr v

当 ι 不是有限类型时，上面的陈述先验上没有意义：不能在 ι 上取和。然而，被求和函数的支撑是有限的（它就是 B.repr v 的支撑）。因此需要使用能体现这一点的构造。这里 Mathlib 使用一个需要花时间熟悉的专门函数：Finsupp.linearCombination（它建立在更一般的 Finsupp.sum 之上）。给定一个从类型 ι 到基域 K 的有限支撑函数 c，以及任意函数 f 从 ι 到 V，Finsupp.linearCombination K f c 是在 c 的支撑上对标量乘法 c • f 求和。特别地，可以把它替换成在任意包含 c 支撑的有限集上求和。

example (c : ι →₀ K) (f : ι → V) (s : Finset ι) (h : c.support ⊆ s) :
    Finsupp.linearCombination K f c = ∑ i ∈ s, c i • f i :=
  Finsupp.linearCombination_apply_of_mem_supported K h

也可以假设 f 是有限支撑的，并仍然得到一个良定义的和。但 Finsupp.linearCombination 所作的选择才与这里关于基的讨论相关，因为它允许陈述 Basis.sum_repr 的一般化版本。

example : Finsupp.linearCombination K B (B.repr v) = v :=
  B.linearCombination_repr v

也许会疑惑，为什么这里 K 是显式参数，尽管它可以从 c 的类型推断出来。关键在于，部分应用后的 Finsupp.linearCombination K f 本身是有意义的对象。它不是一个从 ι →₀ K 到 V 的普通函数，而是一个 K-线性映射。

variable (f : ι → V) in
#check (Finsupp.linearCombination K f : (ι →₀ K) →ₗ[K] V)

回到数学讨论，需要理解的是，在形式化数学中，基下的向量表示并没有你可能想象的那样有用。事实上，直接使用基的更抽象性质往往高效得多。特别地，基与代数中其他自由对象相联系的泛性质允许通过指定基向量的像来构造线性映射。这就是 Basis.constr。对任意 K-向量空间 W，我们的基 B 给出从 ι → W 到 V →ₗ[K] W 的线性同构 Basis.constr B K。这个同构由如下性质刻画：它把任意函数 u : ι → W 送到一个线性映射；该线性映射对每个 i : ι 都把基向量 B i 送到 u i。

section

variable {W : Type*} [AddCommGroup W] [Module K W]
         (φ : V →ₗ[K] W) (u : ι → W)

#check (B.constr K : (ι → W) ≃ₗ[K] (V →ₗ[K] W))

#check (B.constr K u : V →ₗ[K] W)

example (i : ι) : B.constr K u (B i) = u i :=
  B.constr_basis K u i

这个性质确实是刻画性的，因为线性映射由其在基上的取值决定：

example (φ ψ : V →ₗ[K] W) (h : ∀ i, φ (B i) = ψ (B i)) : φ = ψ :=
  B.ext h

如果目标空间上也有一组基 B'，那么可以把线性映射与矩阵等同起来。这个等同是一个 K-线性同构。

variable {ι' : Type*} (B' : Basis ι' K W) [Fintype ι] [DecidableEq ι] [Fintype ι'] [DecidableEq ι']

open LinearMap

#check (toMatrix B B' : (V →ₗ[K] W) ≃ₗ[K] Matrix ι' ι K)

open Matrix -- get access to the ``*ᵥ`` notation for multiplication between matrices and vectors.

example (φ : V →ₗ[K] W) (v : V) : (toMatrix B B' φ) *ᵥ (B.repr v) = B'.repr (φ v) :=
  toMatrix_mulVec_repr B B' φ v


variable {ι'' : Type*} (B'' : Basis ι'' K W) [Fintype ι''] [DecidableEq ι'']

example (φ : V →ₗ[K] W) : (toMatrix B B'' φ) = (toMatrix B' B'' .id) * (toMatrix B B' φ) := by
  simp

end

作为关于这一主题的练习，我们将证明保证自同态具有良定义行列式的定理的一部分。具体而言，我们要证明：当两组基由同一类型作指标时，它们附加到任意自同态上的矩阵具有相同的行列式。为了得到完整结果，还需要补充使用所有基的指标类型彼此同构这一事实。

当然，Mathlib 已经知道这一点，并且 simp 可以立即关闭目标；因此不要过早使用它，而应使用给定的引理。

open Module LinearMap Matrix

-- Some lemmas coming from the fact that `LinearMap.toMatrix` is an algebra morphism.
#check toMatrix_comp
#check id_comp
#check comp_id
#check toMatrix_id

-- Some lemmas coming from the fact that ``Matrix.det`` is a multiplicative monoid morphism.
#check Matrix.det_mul
#check Matrix.det_one

example [Fintype ι] (B' : Basis ι K V) (φ : End K V) :
    (toMatrix B B φ).det = (toMatrix B' B' φ).det := by
  set M := toMatrix B B φ
  set M' := toMatrix B' B' φ
  set P := (toMatrix B B') LinearMap.id
  set P' := (toMatrix B' B) LinearMap.id
  sorry
end

10.4.3. 维数

回到单个向量空间的情形，基也可用于定义维数概念。这里同样有有限维向量空间的初等情形。对这类空间，我们期望维数是一个自然数。这就是 Module.finrank。它把基域作为显式参数，因为同一个交换群可以成为不同域上的向量空间。

section

#check (Module.finrank K V : ℕ)

-- `Fin n → K` is the archetypical space with dimension `n` over `K`.
example (n : ℕ) : Module.finrank K (Fin n → K) = n :=
  Module.finrank_fin_fun K

-- Seen as a vector space over itself, `ℂ` has dimension one.
example : Module.finrank ℂ ℂ = 1 :=
  Module.finrank_self ℂ

-- But as a real vector space it has dimension two.
example : Module.finrank ℝ ℂ = 2 :=
  Complex.finrank_real_complex

注意，Module.finrank 对任意向量空间都有定义。对无限维向量空间，它返回零，正如除以零返回零一样。

当然，许多引理需要有限维假设。这正是类型类 FiniteDimensional 的作用。例如，请思考如果没有这个假设，下一个例子会如何失败。

example [FiniteDimensional K V] : 0 < Module.finrank K V ↔ Nontrivial V  :=
  Module.finrank_pos_iff

在上面的陈述中，Nontrivial V 表示 V 至少有两个不同元素。注意，Module.finrank_pos_iff 没有显式参数。从左到右使用它时这没有问题；但从右到左使用时就不行，因为 Lean 无法从命题 Nontrivial V 猜出 K。此时，在检查该引理是关于一个名为 R 的环陈述之后，使用命名参数语法很有帮助。因此可以写作：

example [FiniteDimensional K V] (h : 0 < Module.finrank K V) : Nontrivial V := by
  apply (Module.finrank_pos_iff (R := K)).1
  exact h

上面的写法有些奇怪，因为我们已经有假设 h，所以完全可以直接给出完整证明 Module.finrank_pos_iff.1 h；不过在更复杂的情形中，了解这种写法很有用。

按定义，FiniteDimensional K V 可以从任意一组基读出。

variable {ι : Type*} (B : Module.Basis ι K V)

example [Finite ι] : FiniteDimensional K V := Module.Basis.finiteDimensional_of_finite B

example [FiniteDimensional K V] : Finite ι :=
  (FiniteDimensional.fintypeBasisIndex B).finite
end

利用线性子空间对应的子类型具有向量空间结构这一事实，可以讨论子空间的维数。

section
variable (E F : Submodule K V) [FiniteDimensional K V]

open Module

example : finrank K (E ⊔ F : Submodule K V) + finrank K (E ⊓ F : Submodule K V) =
    finrank K E + finrank K F :=
  Submodule.finrank_sup_add_finrank_inf_eq E F

example : finrank K E ≤ finrank K V := Submodule.finrank_le E

在上面第一个陈述中，类型标注的目的，是确保到 Type* 的强制转换不会过早触发。

现在可以做一个关于 finrank 与子空间的练习。

example (h : finrank K V < finrank K E + finrank K F) :
    Nontrivial (E ⊓ F : Submodule K V) := by
  sorry
end

现在转向维数理论的一般情形。在这种情况下，finrank 没有用；但我们仍然知道，同一向量空间的任意两组基，其指标类型之间存在一个双射。因此仍可希望把秩定义为一个基数，即“所有类型按存在双射这一等价关系取商”所得对象的一个元素。

讨论基数时，就更难像本书其他地方那样忽略与罗素悖论有关的基础问题。不存在所有类型组成的类型，因为那会导致逻辑不一致。这个问题由宇宙层级解决，而我们通常尽量不去显式处理它。

每个类型都有一个宇宙层级，这些层级的行为类似自然数。特别地，有第零层级，相应的宇宙 Type 0 简记为 Type。这个宇宙足以容纳几乎所有经典数学。例如，ℕ 和 ℝ 的类型都是 Type。每个层级 u 都有一个后继，记作 u + 1；并且 Type u 的类型是 Type (u+1)。

但宇宙层级并不是自然数，它们具有完全不同的性质，并且自身没有类型。特别地，不能在 Lean 中陈述诸如 u ≠ u + 1 这样的命题。根本不存在一个使这句话可安放其中的类型。即使陈述 Type u ≠ Type (u+1) 也没有意义，因为 Type u 与 Type (u+1) 具有不同的类型。

每当写 Type* 时，Lean 都会插入一个名为 u_n 的宇宙层级变量，其中 n 是一个数字。这使定义和陈述能够生活在所有宇宙中。

给定宇宙层级 u，可以在 Type u 上定义一个等价关系：若类型 α 与 β 之间存在双射，则称它们等价。商类型 Cardinal.{u} 生活在 Type (u+1) 中。花括号表示宇宙变量。α : Type u 在这个商中的像是 Cardinal.mk α : Cardinal.{u}。

但我们不能直接比较不同宇宙中的基数。因此，从技术上说，不能把向量空间 V 的秩定义为所有可作为 V 的基指标类型的基数。于是，秩改定义为 V 中所有线性无关集的基数的上确界 Module.rank K V。若 V 的宇宙层级为 u，则其秩的类型为 Cardinal.{u}。

#check V -- Type u_2
#check Module.rank K V -- Cardinal.{u_2}

仍然可以把这个定义与基联系起来。事实上，宇宙层级上还有一个交换的最大值运算 max；给定两个宇宙层级 u 和 v，有一个操作 Cardinal.lift.{u, v} : Cardinal.{v} → Cardinal.{max v u}，它允许把基数放入共同的宇宙中，从而陈述维数定理。

universe u v -- `u` and `v` will denote universe levels

variable {ι : Type u} (B : Module.Basis ι K V)
         {ι' : Type v} (B' : Module.Basis ι' K V)

example : Cardinal.lift.{v, u} (.mk ι) = Cardinal.lift.{u, v} (.mk ι') :=
  mk_eq_mk_of_basis B B'

可以通过从自然数到有限基数的强制转换，把有限维情形与上述讨论联系起来（更精确地说，是到生活在 Cardinal.{v} 中的有限基数，其中 v 是 V 的宇宙层级）。

example [FiniteDimensional K V] :
    (Module.finrank K V : Cardinal) = Module.rank K V :=
  Module.finrank_eq_rank K V
