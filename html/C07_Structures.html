7. 结构

现代数学本质上依赖代数结构；这些结构封装了可在多种情境中实例化的模式。数学提供了定义这类结构并构造其具体实例的多种方法。

因此，Lean 也提供了形式化定义结构并使用结构的相应方式。你已经在 Lean 中见过一些代数结构的例子，例如Chapter 2中讨论过的环和格。本章将解释那里出现过的神秘方括号标注 [Ring α] 和 [Lattice α]，并说明如何自己定义和使用代数结构。

若需要更多技术细节，可以参考 Theorem Proving in Lean，以及 Anne Baanen 的论文 Use and abuse of instance parameters in the Lean mathematical library。

7.1. 定义结构

在最广泛的意义上，structure是对一组数据的规约，其中可能还包含这些数据必须满足的约束。该结构的一个instance，就是满足这些约束的一份具体数据包。例如，可以规定一个点是三个实数构成的元组：

@[ext]
structure Point where
  x : ℝ
  y : ℝ
  z : ℝ

标注 @[ext] 告诉 Lean 自动生成可用于证明结构两个实例相等的定理：只要它们的各个分量相等，就能证明实例相等。这一性质称为extensionality。

#check Point.ext

example (a b : Point) (hx : a.x = b.x) (hy : a.y = b.y) (hz : a.z = b.z) : a = b := by
  ext
  repeat' assumption

随后可以定义 Point 结构的具体实例。Lean 提供了多种做法。

def myPoint1 : Point where
  x := 2
  y := -1
  z := 4

def myPoint2 : Point :=
  { x := 2, y := -1, z := 4 }

def myPoint3 : Point :=
  ⟨2, -1, 4⟩

def myPoint4 :=
  Point.mk 2 (-1) 4

在第一个例子中，结构的字段被显式命名。myPoint4 定义中提到的函数 Point.mk 称为 Point 结构的constructor，因为它用于构造元素。若愿意，也可以指定另一个名称，例如 build。

structure Point' where build ::
  x : ℝ
  y : ℝ
  z : ℝ

#check Point'.build 2 (-1) 4

接下来的两个例子展示如何在结构上定义函数。第二个例子显式写出了构造子 Point.mk，而第一个例子为了简洁使用匿名构造子。Lean 可以根据所标明的 add 的类型推断出相关构造子。通常约定，将与 Point 这类结构相关的定义和定理放入同名命名空间。在下面的例子中，由于已经打开 Point 命名空间，add 的全名是 Point.add。当命名空间未打开时，必须使用全名。不过请记住，匿名投影记号常常很方便，它允许我们写 a.add b 而不是 Point.add a b。Lean 会把前者解释为后者，因为 a 的类型是 Point。

namespace Point

def add (a b : Point) : Point :=
  ⟨a.x + b.x, a.y + b.y, a.z + b.z⟩

def add' (a b : Point) : Point where
  x := a.x + b.x
  y := a.y + b.y
  z := a.z + b.z

#check add myPoint1 myPoint2
#check myPoint1.add myPoint2

end Point

#check Point.add myPoint1 myPoint2
#check myPoint1.add myPoint2

下面我们仍会把定义放在相关命名空间中，但在引用的代码片段里省略命名空间命令。为了证明加法函数的性质，可以用 rw 展开定义，并用 ext 把结构两个元素之间的等式化归为各分量之间的等式。下面我们使用关键字 protected，使得即使命名空间已经打开，该定理名称仍为 Point.add_comm。当我们希望避免与 add_comm 这样的通用定理产生歧义时，这很有用。

protected theorem add_comm (a b : Point) : add a b = add b a := by
  rw [add, add]
  ext <;> dsimp
  repeat' apply add_comm

example (a b : Point) : add a b = add b a := by simp [add, add_comm]

由于 Lean 可以在内部展开定义并化简投影，有时我们想要的等式会按定义成立。

theorem add_x (a b : Point) : (a.add b).x = a.x + b.x :=
  rfl

也可以用模式匹配在结构上定义函数，方式类似于我们在Section 5.2中定义递归函数的方式。下面的定义 addAlt 与 addAlt' 本质相同；唯一区别是第二个使用匿名构造子记号。虽然有时这样定义函数很方便，而且结构 η-约化使这个替代定义按定义等价，但在后续证明中它可能不那么方便。特别地，rw [addAlt] 会留下一个含有 match 语句、更混乱的目标视图。

def addAlt : Point → Point → Point
  | Point.mk x₁ y₁ z₁, Point.mk x₂ y₂ z₂ => ⟨x₁ + x₂, y₁ + y₂, z₁ + z₂⟩

def addAlt' : Point → Point → Point
  | ⟨x₁, y₁, z₁⟩, ⟨x₂, y₂, z₂⟩ => ⟨x₁ + x₂, y₁ + y₂, z₁ + z₂⟩

theorem addAlt_x (a b : Point) : (a.addAlt b).x = a.x + b.x := by
  rfl

theorem addAlt_comm (a b : Point) : addAlt a b = addAlt b a := by
  rw [addAlt, addAlt]
  -- the same proof still works, but the goal view here is harder to read
  ext <;> dsimp
  repeat' apply add_comm

数学构造常常需要拆解已打包的信息，再以不同方式重新组合。因此 Lean 和 Mathlib 提供许多高效完成这件事的方法，是很自然的。作为练习，请尝试证明 Point.add 满足结合律。然后为点定义标量乘法，并证明它对加法满足分配律。

protected theorem add_assoc (a b c : Point) : (a.add b).add c = a.add (b.add c) := by
  sorry

def smul (r : ℝ) (a : Point) : Point :=
  sorry

theorem smul_distrib (r : ℝ) (a b : Point) :
    (smul r a).add (smul r b) = smul r (a.add b) := by
  sorry

使用结构只是通向代数抽象的第一步。我们还没有办法把 Point.add 与通用符号 + 关联起来，也没有办法把 Point.add_comm 和 Point.add_assoc 与通用定理 add_comm 和 add_assoc 关联起来。这些任务属于使用结构的algebraic方面，我们将在下一节解释如何完成。现在，只需把结构看作打包对象和信息的一种方式。

结构特别有用的一点是：它不仅能指定数据类型，也能指定这些数据必须满足的约束。在 Lean 中，后者表示为类型为 Prop 的字段。例如，standard 2-simplex定义为满足 \(x ≥ 0\)、\(y ≥ 0\)、\(z ≥ 0\) 和 \(x + y + z = 1\) 的点 \((x, y, z)\) 的集合。如果你不熟悉这个概念，应当画一幅图，并说服自己这个集合就是三维空间中以 \((1, 0, 0)\)、\((0, 1, 0)\) 和 \((0, 0, 1)\) 为顶点的等边三角形连同其内部。在 Lean 中可以如下表示它：

structure StandardTwoSimplex where
  x : ℝ
  y : ℝ
  z : ℝ
  x_nonneg : 0 ≤ x
  y_nonneg : 0 ≤ y
  z_nonneg : 0 ≤ z
  sum_eq : x + y + z = 1

注意，最后四个字段引用了 x、y 和 z，也就是前三个字段。可以定义一个从二单纯形到自身的映射，交换 x 与 y：

def swapXy (a : StandardTwoSimplex) : StandardTwoSimplex
    where
  x := a.y
  y := a.x
  z := a.z
  x_nonneg := a.y_nonneg
  y_nonneg := a.x_nonneg
  z_nonneg := a.z_nonneg
  sum_eq := by rw [add_comm a.y a.x, a.sum_eq]

更有趣的是，可以计算单纯形上两个点的中点。为了使用实数上的除法，我们在本文件开头加入了短语 noncomputable section。

noncomputable section

def midpoint (a b : StandardTwoSimplex) : StandardTwoSimplex
    where
  x := (a.x + b.x) / 2
  y := (a.y + b.y) / 2
  z := (a.z + b.z) / 2
  x_nonneg := div_nonneg (add_nonneg a.x_nonneg b.x_nonneg) (by norm_num)
  y_nonneg := div_nonneg (add_nonneg a.y_nonneg b.y_nonneg) (by norm_num)
  z_nonneg := div_nonneg (add_nonneg a.z_nonneg b.z_nonneg) (by norm_num)
  sum_eq := by field_simp; linarith [a.sum_eq, b.sum_eq]

这里我们用简短的证明项建立了 x_nonneg、y_nonneg 和 z_nonneg，但用策略模式通过 by 建立 sum_eq。

给定满足 \(0 \le \lambda \le 1\) 的参数 \(\lambda\)，可以取标准 2-单纯形中两点 \(a\) 和 \(b\) 的加权平均 \(\lambda a + (1 - \lambda) b\)。请仿照上面的 midpoint 函数定义这个函数。

结构可以依赖参数。例如，可以把标准 2-单纯形推广为任意 \(n\) 的标准 \(n\)-单纯形。在这个阶段，你不需要知道类型 Fin n 的任何内容，只需知道它有 \(n\) 个元素，并且 Lean 知道如何在其上求和。

open BigOperators

structure StandardSimplex (n : ℕ) where
  V : Fin n → ℝ
  NonNeg : ∀ i : Fin n, 0 ≤ V i
  sum_eq_one : (∑ i, V i) = 1

namespace StandardSimplex

def midpoint (n : ℕ) (a b : StandardSimplex n) : StandardSimplex n
    where
  V i := (a.V i + b.V i) / 2
  NonNeg := by
    intro i
    apply div_nonneg
    · linarith [a.NonNeg i, b.NonNeg i]
    norm_num
  sum_eq_one := by
    simp [div_eq_mul_inv, ← Finset.sum_mul, Finset.sum_add_distrib,
      a.sum_eq_one, b.sum_eq_one]
    norm_num

end StandardSimplex

作为练习，请看看能否定义标准 \(n\)-单纯形中两点的加权平均。可以使用 Finset.sum_add_distrib 和 Finset.mul_sum 操作相关求和。

我们已经看到，结构可以把数据和性质打包在一起。有趣的是，它们也可以只打包性质而不包含数据。例如，下一个结构 IsLinear 把线性的两个组成部分打包在一起。

structure IsLinear (f : ℝ → ℝ) where
  is_additive : ∀ x y, f (x + y) = f x + f y
  preserves_mul : ∀ x c, f (c * x) = c * f x

section
variable (f : ℝ → ℝ) (linf : IsLinear f)

#check linf.is_additive
#check linf.preserves_mul

end

值得指出的是，结构并不是打包数据的唯一方式。数据结构 Point 可以用通用的类型乘积定义，而 IsLinear 可以用简单的 and 定义。

def Point'' :=
  ℝ × ℝ × ℝ

def IsLinear' (f : ℝ → ℝ) :=
  (∀ x y, f (x + y) = f x + f y) ∧ ∀ x c, f (c * x) = c * f x

通用类型构造甚至可以替代分量之间存在依赖的结构。例如，subtype构造把一份数据与一个性质结合起来。可以把下一个例子中的类型 PReal 看作正实数类型。任意 x : PReal 都有两个分量：它的值，以及它为正的性质。可以通过 x.val 访问前者，其类型为 ℝ；通过 x.property 访问后者，它表示事实 0 < x.val。

def PReal :=
  { y : ℝ // 0 < y }

section
variable (x : PReal)

#check x.val
#check x.property
#check x.1
#check x.2

end

我们本来也可以用子类型定义标准 2-单纯形，以及任意 \(n\) 的标准 \(n\)-单纯形。

def StandardTwoSimplex' :=
  { p : ℝ × ℝ × ℝ // 0 ≤ p.1 ∧ 0 ≤ p.2.1 ∧ 0 ≤ p.2.2 ∧ p.1 + p.2.1 + p.2.2 = 1 }

def StandardSimplex' (n : ℕ) :=
  { v : Fin n → ℝ // (∀ i : Fin n, 0 ≤ v i) ∧ (∑ i, v i) = 1 }

类似地，Sigma types是有序对的推广，其中第二个分量的类型依赖于第一个分量的类型。

def StdSimplex := Σ n : ℕ, StandardSimplex n

section
variable (s : StdSimplex)

#check s.fst
#check s.snd

#check s.1
#check s.2

end

给定 s : StdSimplex，第一个分量 s.fst 是一个自然数，第二个分量是相应单纯形 StandardSimplex s.fst 的一个元素。Sigma 类型与子类型的区别在于，Sigma 类型的第二个分量是数据，而不是命题。

不过，尽管可以用乘积、子类型和 Sigma 类型替代结构，使用结构仍有许多优势。定义结构会抽象掉底层表示，并为访问各分量的函数提供自定义名称。这使证明更加稳健：只依赖结构接口的证明，通常在我们改变定义后仍能继续工作，只要把旧访问器用新定义重新定义出来。此外，正如我们马上将看到的，Lean 支持把结构编织成丰富且相互关联的层级体系，并管理它们之间的相互作用。

7.2. 代数结构

为了澄清“algebraic structure”这一短语的含义，考虑一些例子会很有帮助。

  1. partially ordered set由一个集合 \(P\) 以及 \(P\) 上的二元关系 \(\le\) 组成，该关系满足传递性、自反性和反对称性。
  2. group由一个集合 \(G\)、一个结合的二元运算、一个单位元 \(1\)，以及一个函数 \(g \mapsto g^{-1}\) 组成；该函数为 \(G\) 中每个 \(g\) 返回一个逆元。若运算可交换，则群称为abelian或commutative。
  3. lattice是带有交和并的偏序集。
  4. ring由一个（以加法方式书写的）阿贝尔群 \((R, +, 0, x \mapsto -x)\)，连同一个结合的乘法运算 \(\cdot\) 和单位元 \(1\) 组成，并且乘法对加法满足分配律。若乘法可交换，则环称为commutative环。
  5. ordered ring \((R, +, 0, -, \cdot, 1, \le)\) 由一个环连同其元素上的偏序组成，使得对 \(R\) 中任意 \(a\)、\(b\) 和 \(c\)，若 \(a \le b\)，则 \(a + c \le b + c\)；并且对 \(R\) 中任意 \(a\) 和 \(b\)，若 \(0 \le a\) 且 \(0 \le b\)，则 \(0 \le a b\)。
  6. metric space由一个集合 \(X\) 和函数 \(d : X \times X \to \mathbb{R}\) 组成，满足如下性质：对 \(X\) 中任意 \(x\) 和 \(y\)，有 \(d(x, y) \ge 0\)。\(d(x, y) = 0\) 当且仅当 \(x = y\)。对 \(X\) 中任意 \(x\) 和 \(y\)，有 \(d(x, y) = d(y, x)\)。对 \(X\) 中任意 \(x\)、\(y\) 和 \(z\)，有 \(d(x, z) \le d(x, y) + d(y, z)\)。
  7. 对 \(X\) 中任意 \(x\) 和 \(y\)，有 \(d(x, y) \ge 0\)。

在这些例子中，结构的元素属于一个集合，即carrier set；它有时代表整个结构。例如，当我们说“令 \(G\) 为一个群”，随后又说“令 \(g \in G\)”时，我们同时用 \(G\) 表示该结构及其载集。并非每个代数结构都以这种方式关联到单个载集。例如，bipartite graph涉及两个集合之间的关系，Galois connection也是如此。category也涉及两个重要集合，通常称为objects和morphisms。

这些例子显示，为了支持代数推理，证明助手必须完成一些任务。首先，它需要识别结构的具体实例。数系 \(\mathbb{Z}\)、\(\mathbb{Q}\) 和 \(\mathbb{R}\) 都是有序环，因此我们应能在这些任一实例中应用关于有序环的通用定理。有时，一个具体集合可以以不止一种方式成为某结构的实例。例如，除了作为实分析基础的 \(\mathbb{R}\) 上通常拓扑之外，我们还可以考虑 \(\mathbb{R}\) 上的discrete拓扑，其中每个集合都是开集。

其次，证明助手需要支持结构上的通用记号。在 Lean 中，记号 * 用于所有通常数系中的乘法，也用于通用群和环中的乘法。当我们使用类似 f x * y 的表达式时，Lean 必须利用关于 f、x 和 y 的类型信息来判定我们意指哪一种乘法。

第三，它需要处理这样一个事实：结构能够以多种方式从其他结构继承定义、定理和记号。有些结构通过增加更多公理来扩展其他结构。交换环仍然是环，所以任何在环中有意义的定义在交换环中也有意义，任何在环中成立的定理在交换环中也成立。有些结构通过增加更多数据来扩展其他结构。例如，任意环的加法部分是一个加法群。环结构增加了乘法和单位元，以及支配它们并把它们同加法部分联系起来的公理。有时我们可以用一个结构定义另一个结构。任意度量空间都有一个与之关联的典范拓扑，即metric space topology；任意线性序也可以关联多种拓扑。

最后，需要牢记，数学允许我们用函数和运算定义结构，正如我们用函数和运算定义数一样。群的乘积和幂仍是群。对每个 \(n\)，模 \(n\) 整数形成一个环；对每个 \(k > 0\)，以该环中元素为系数的多项式的 \(k \times k\) 矩阵又形成一个环。因此，我们可以像计算结构的元素一样方便地计算结构本身。这意味着代数结构在数学中具有双重身份：既是对象集合的容器，又是独立的对象。证明助手必须适应这种双重角色。

在处理一个具有相关代数结构的类型的元素时，证明助手需要识别该结构，并找到相关定义、定理和记号。这些听起来像是大量工作，而事实也确实如此。不过 Lean 使用一小组基本机制来完成这些任务。本节的目标就是解释这些机制，并展示如何使用它们。

第一个成分几乎显而易见：从形式上说，代数结构就是Section 7.1意义下的结构。代数结构是一组满足某些公理假设的数据的规约，而我们在Section 7.1中已经看到，structure 命令正是为容纳这种内容而设计的。这种配合可谓恰到好处。

给定一个数据类型 α，可以如下在 α 上定义群结构。

structure Group₁ (α : Type*) where
  mul : α → α → α
  one : α
  inv : α → α
  mul_assoc : ∀ x y z : α, mul (mul x y) z = mul x (mul y z)
  mul_one : ∀ x : α, mul x one = x
  one_mul : ∀ x : α, mul one x = x
  inv_mul_cancel : ∀ x : α, mul (inv x) x = one

注意，类型 α 是定义 Group₁ 中的一个parameter。因此，应把对象 struc : Group₁ α 理解为 α 上的一个群结构。我们在Section 2.2中看到，inv_mul_cancel 的对应形式 mul_inv_cancel 可由其他群公理推出，因此无需把它加入定义。

这个群定义类似于 Mathlib 中 Group 的定义；我们选择名称 Group₁ 来区分自己的版本。如果写 #check Group 并按住 Ctrl 点击定义，会看到 Mathlib 版本的 Group 被定义为扩展另一个结构；稍后会解释如何做到这一点。如果输入 #print Group，还会看到 Mathlib 版本的 Group 有若干额外字段。出于稍后要解释的原因，有时向结构中加入冗余信息是有用的，也就是为可由核心数据定义出的对象和函数增加额外字段。现在不必担心这一点。可以放心，我们的简化版本 Group₁ 在数学本质上与 Mathlib 使用的群定义相同。

有时把类型与结构打包在一起是有用的，Mathlib 也包含一个与下述定义等价的 Grp 结构定义：

structure Grp₁ where
  α : Type*
  str : Group₁ α

Mathlib 版本位于 Mathlib.Algebra.Category.Grp.Basic 中；若把它加入示例文件开头的导入中，就可以对它执行 #check。

出于下面会逐渐清楚的原因，更常用的做法是把类型 α 与结构 Group α 分开。我们把这两个对象合称为partially bundled structure，因为这种表示把大多数分量、但不是全部分量合并到一个结构中。在 Mathlib 中，当某个类型用作群的载体类型时，通常用像 G 这样的大写罗马字母表示它。

让我们通过定义 Group₁ 类型的一个元素来构造一个群。对于任意一对类型 α 和 β，Mathlib 定义了类型 Equiv α β，表示 α 与 β 之间的equivalences。Mathlib 还为这个类型定义了直观的记号 α ≃ β。元素 f : α ≃ β 是 α 与 β 之间的一个双射，由四个分量表示：从 α 到 β 的函数 f.toFun，从 β 到 α 的逆函数 f.invFun，以及说明这两个函数确实互为逆的两个性质。

variable (α β γ : Type*)
variable (f : α ≃ β) (g : β ≃ γ)

#check Equiv α β
#check (f.toFun : α → β)
#check (f.invFun : β → α)
#check (f.right_inv : ∀ x : β, f (f.invFun x) = x)
#check (f.left_inv : ∀ x : α, f.invFun (f x) = x)
#check (Equiv.refl α : α ≃ α)
#check (f.symm : β ≃ α)
#check (f.trans g : α ≃ γ)

注意最后三个构造名称颇具创造性。我们把恒等函数 Equiv.refl、取逆操作 Equiv.symm 和复合操作 Equiv.trans 看作“双射对应”这一性质为等价关系的显式证据。

还要注意，f.trans g 要求按相反顺序复合正向函数。Mathlib 已经声明了从 Equiv α β 到函数类型 α → β 的coercion，因此可以省略 .toFun，让 Lean 为我们插入它。

example (x : α) : (f.trans g).toFun x = g.toFun (f.toFun x) :=
  rfl

example (x : α) : (f.trans g) x = g (f x) :=
  rfl

example : (f.trans g : α → γ) = g ∘ f :=
  rfl

Mathlib 还定义了类型 Perm α，表示 α 到自身的等价。

example (α : Type*) : Equiv.Perm α = (α ≃ α) :=
  rfl

显然，Equiv.Perm α 在等价复合下形成一个群。我们约定方向，使 mul f g 等于 g.trans f，后者的正向函数是 f ∘ g。换言之，乘法就是通常意义下双射的复合。下面定义这个群：

def permGroup {α : Type*} : Group₁ (Equiv.Perm α)
    where
  mul f g := Equiv.trans g f
  one := Equiv.refl α
  inv := Equiv.symm
  mul_assoc f g h := (Equiv.trans_assoc h g f).symm
  one_mul := Equiv.trans_refl
  mul_one := Equiv.refl_trans
  inv_mul_cancel := Equiv.self_trans_symm

事实上，Mathlib 在文件 Algebra.Group.End 中为 Equiv.Perm α 定义的正是这个 Group 结构。照常，可以把鼠标悬停在 permGroup 定义中使用的定理上查看其陈述，也可以跳转到原文件中的定义，进一步了解它们如何实现。

在通常数学中，我们一般认为记号独立于结构。例如，可以考虑群 \((G_1, \cdot, 1, \cdot^{-1})\)、\((G_2, \circ, e, i(\cdot))\) 和 \((G_3, +, 0, -)\)。在第一种情况下，把二元运算写作 \(\cdot\)，把单位元写作 \(1\)，把取逆函数写作 \(x \mapsto x^{-1}\)。在第二和第三种情况下，我们使用所示的替代记号。然而，当在 Lean 中形式化群的概念时，记号与结构联系得更紧密。在 Lean 中，任意 Group 的分量命名为 mul、one 和 inv，稍后我们会看到乘法记号如何设置为引用这些分量。如果想使用加法记号，则改用同构的结构 AddGroup（即加法群底层的结构）。它的分量命名为 add、zero 和 neg，相应记号也正如预期。

我们正在取得进展。现在我们知道如何在 Lean 中定义代数结构，也知道如何定义这些结构的实例。但我们还希望把记号同结构关联起来，使得每个实例都能使用这些记号。此外，我们还希望安排成：可以在一个结构上定义操作，并在任何具体实例中使用它；也可以证明关于一个结构的定理，并在任何实例中使用它。

事实上，Mathlib 已经设置好，可以对 Equiv.Perm α 使用通用的群记号、定义和定理。

你可以检查，上面要求你为 Point 定义的加法群结构并非如此。现在我们的任务是理解底层发生了什么“魔法”，使得 Equiv.Perm α 的例子能够这样工作。

问题在于，Lean 需要能够利用我们输入的表达式中出现的信息，find相关记号和隐式的群结构。类似地，当我们用类型为 ℝ 的表达式 x 和 y 写 x + y 时，Lean 需要把符号 + 解释为实数上的相关加法函数。它还必须识别类型 ℝ 是交换环的一个实例，从而可以使用交换环的所有定义和定理。再举一个例子，Lean 中的连续性是相对于任意两个拓扑空间定义的。当有 f : ℝ → ℂ 且写 Continuous f 时，Lean 必须找到 ℝ 和 ℂ 上的相关拓扑。

这种机制由三件事结合实现。

  1. \(d(x, y) = 0\) 当且仅当 \(x = y\)。
  2. 对 \(X\) 中任意 \(x\) 和 \(y\)，有 \(d(x, y) = d(y, x)\)。
  3. 对 \(X\) 中任意 \(x\)、\(y\) 和 \(z\)，有 \(d(x, z) \le d(x, y) + d(y, z)\)。

标注 (grp : Group G) 告诉 Lean 应期待显式给出该参数；标注 {grp : Group G} 告诉 Lean 应尝试从表达式中的上下文线索推断它；而标注 [grp : Group G] 则告诉 Lean 应使用类型类推断来合成相应参数。由于使用这类参数的要点在于我们通常不需要显式引用它们，Lean 允许写 [Group G] 并让名称匿名。你大概已经注意到，Lean 会自动选择像 _inst_1 这样的名称。当把匿名方括号标注同 variable 命令一起使用时，只要变量仍在作用域内，Lean 就会自动向任何提到 G 的定义或定理添加参数 [Group G]。

如何注册 Lean 执行搜索所需的信息？回到群的例子，只需作两处改动。第一，不使用 structure 命令定义群结构，而使用关键字 class 表明它是类型类推断的候选。第二，不用 def 定义具体实例，而用关键字 instance 把该具体实例注册给 Lean。与类变量名一样，实例定义的名称也允许匿名，因为一般而言我们希望 Lean 找到它并使用它，而不必用细节打扰我们。

class Group₂ (α : Type*) where
  mul : α → α → α
  one : α
  inv : α → α
  mul_assoc : ∀ x y z : α, mul (mul x y) z = mul x (mul y z)
  mul_one : ∀ x : α, mul x one = x
  one_mul : ∀ x : α, mul one x = x
  inv_mul_cancel : ∀ x : α, mul (inv x) x = one

instance {α : Type*} : Group₂ (Equiv.Perm α) where
  mul f g := Equiv.trans g f
  one := Equiv.refl α
  inv := Equiv.symm
  mul_assoc f g h := (Equiv.trans_assoc h g f).symm
  one_mul := Equiv.trans_refl
  mul_one := Equiv.refl_trans
  inv_mul_cancel := Equiv.self_trans_symm

下面说明它们的用法。

  Group₂.mul x x

#check mySquare

section

命令 #check 显示，Group₂.mul 有一个隐式参数 [Group₂ α]，我们期待它由类型类推断找到，其中 α 是 Group₂.mul 的参数类型。换言之，{α : Type*} 是群元素类型的隐式参数，[Group₂ α] 是 α 上群结构的隐式参数。类似地，当为 Group₂ 定义通用平方函数 my_square 时，我们使用隐式参数 {α : Type*} 表示元素类型，并使用隐式参数 [Group₂ α] 表示 Group₂ 结构。

在第一个例子中，当写 Group₂.mul f g 时，f 和 g 的类型告诉 Lean，Group₂.mul 的参数 α 必须实例化为 Equiv.Perm β。这意味着 Lean 必须找到 Group₂ (Equiv.Perm β) 的一个元素。前面的 instance 声明恰好告诉 Lean 如何做到这一点。问题解决了。

这种注册信息、使 Lean 在需要时能够找到它的简单机制极其有用。下面是一种出现方式。在 Lean 的基础中，数据类型 α 可以为空。然而在许多应用中，知道某个类型至少有一个元素是有用的。例如，函数 List.headI 返回列表的第一个元素；当列表为空时，它可以返回默认值。为了使这件事可行，Lean 库定义了类 Inhabited α，它只是存储一个默认值。我们可以证明类型 Point 是一个实例：

instance : Inhabited Point where default := ⟨0, 0, 0⟩

#check (default : Point)

example : ([] : List Point).headI = default :=
  rfl

类型类推断机制也用于通用记号。表达式 x + y 是 Add.add x y 的缩写，其中，如你所料，Add α 是一个在 α 上存储二元函数的类。写 x + y 会告诉 Lean 寻找 [Add.add α] 的已注册实例并使用相应函数。下面，我们为 Point 注册加法函数。

instance : Add Point where add := Point.add

section
variable (x y : Point)

#check x + y

example : x + y = Point.add x y :=
  rfl

end

这样，也可以把记号 + 指派给其他类型上的二元运算。

但还可以做得更好。我们已经看到，* 可用于任意群，+ 可用于任意加法群，而二者都可用于任意环。当在 Lean 中定义一个新的环实例时，不必为该实例定义 + 和 *，因为 Lean 知道它们已为每个环定义。我们可以用这种方法为类 Group₂ 指定记号：

instance {α : Type*} [Group₂ α] : Mul α :=
  ⟨Group₂.mul⟩

instance {α : Type*} [Group₂ α] : One α :=
  ⟨Group₂.one⟩

instance {α : Type*} [Group₂ α] : Inv α :=
  ⟨Group₂.inv⟩

section
variable {α : Type*} (f g : Equiv.Perm α)

#check f * 1 * g⁻¹

def foo : f * 1 * g⁻¹ = g.symm.trans ((Equiv.refl α).trans f) :=
  rfl

end

这种方法之所以有效，是因为 Lean 会执行递归搜索。根据我们声明的实例，Lean 可以通过寻找 Group₂ (Equiv.Perm α) 的实例来找到 Mul (Equiv.Perm α) 的实例；而它又能找到 Group₂ (Equiv.Perm α) 的实例，因为我们已经提供了一个。Lean 能够发现这两个事实并把它们串联起来。

刚才给出的例子是危险的，因为 Lean 库中也有 Group (Equiv.Perm α) 的实例，而乘法在任意群上都已定义。因此，究竟找到哪个实例是有歧义的。事实上，除非显式指定不同优先级，Lean 会偏向较新的声明。此外，还有另一种方式告诉 Lean 一个结构是另一个结构的实例，即使用关键字 extends。Mathlib 正是以这种方式说明例如每个交换环都是环。更多信息可参见Section 8以及 Theorem Proving in Lean 中关于类型类推断的section on class inference。

一般来说，对于一个已经定义了记号的代数结构实例，再为 * 指定取值并不是好主意。在 Lean 中重新定义 Group 的概念只是一个人为例子。不过，在这种情况下，群记号的两种解释都会以同样方式展开为 Equiv.trans、Equiv.refl 和 Equiv.symm。

7.3. 构造高斯整数

现在我们通过构造一个重要数学对象，即Gaussian integers，并证明它是欧几里得整环，来展示 Lean 中代数层级的用法。换言之，按照一直使用的术语，我们将定义高斯整数，并证明它们是欧几里得整环结构的一个实例。

用通常数学语言说，高斯整数集 \(\Bbb{Z}[i]\) 是复数集合 \(\{ a + b i \mid a, b \in \Bbb{Z}\}\)。不过，这里我们的目标不是把它们定义为复数的子集，而是把它们定义为一个自身独立的数据类型。为此，我们把高斯整数表示为一对整数，分别看作real部和imaginary部。

@[ext]
structure GaussInt where
  re : ℤ
  im : ℤ

我们首先证明高斯整数具有环结构，其中 0 定义为 ⟨0, 0⟩，1 定义为 ⟨1, 0⟩，加法逐点定义。为了确定乘法定义，记住我们希望元素 \(i\)（表示为 ⟨0, 1⟩）是 \(-1\) 的平方根。因此我们希望

\[\begin{split}(a + bi) (c + di) & = ac + bci + adi + bd i^2 \\ & = (ac - bd) + (bc + ad)i.\end{split}\]

这就解释了下面 Mul 的定义。

instance : Zero GaussInt :=
  ⟨⟨0, 0⟩⟩

instance : One GaussInt :=
  ⟨⟨1, 0⟩⟩

instance : Add GaussInt :=
  ⟨fun x y ↦ ⟨x.re + y.re, x.im + y.im⟩⟩

instance : Neg GaussInt :=
  ⟨fun x ↦ ⟨-x.re, -x.im⟩⟩

instance : Mul GaussInt :=
  ⟨fun x y ↦ ⟨x.re * y.re - x.im * y.im, x.re * y.im + x.im * y.re⟩⟩

如Section 7.1所述，把与某个数据类型相关的所有定义放在同名命名空间中是一个好习惯。因此，在与本章相关的 Lean 文件中，这些定义都放在 GaussInt 命名空间中。

现在我们将证明高斯整数还具有一个重要附加性质。Euclidean domain是一个环 \(R\)，并配备一个norm函数 \(N : R \to \mathbb{N}\)，满足下面两个性质：

  • topological space由一个集合 \(X\) 和 \(X\) 的子集族 \(\mathcal T\) 组成，后者称为 \(X\) 的open subsets of，并满足如下性质：空集和 \(X\) 是开集。两个开集的交是开集。任意开集族的并是开集。
  • 空集和 \(X\) 是开集。

带有 \(N(a) = |a|\) 的整数环 \(\Bbb{Z}\) 是欧几里得整环的典型例子。在这种情况下，可以取 \(q\) 为 \(a\) 除以 \(b\) 的整数商，取 \(r\) 为余数。注意，当 a 和 b 是整数时，a / b 在 Lean 中表示整数除法，a % b 表示余数。它们被定义为满足如下性质：

example (a b : ℤ) : b ≠ 0 → 0 ≤ a % b :=
  Int.emod_nonneg a

example (a b : ℤ) : b ≠ 0 → a % b < |b| :=
  Int.emod_lt_abs a

在任意环中，若元素 \(a\) 整除 \(1\)，则称 \(a\) 为unit。若非零元素 \(a\) 不能写成 \(a = bc\) 的形式，其中 \(b\) 和 \(c\) 都不是单位，则称它为irreducible。在整数中，每个不可约元素 \(a\) 都是prime；也就是说，只要 \(a\) 整除乘积 \(bc\)，它就整除 \(b\) 或整除 \(c\)。但在其他环中，这个性质可能失败。在环 \(\Bbb{Z}[\sqrt{-5}]\) 中，我们有

\[6 = 2 \cdot 3 = (1 + \sqrt{-5})(1 - \sqrt{-5}),\]

并且元素 \(2\)、\(3\)、\(1 + \sqrt{-5}\) 和 \(1 - \sqrt{-5}\) 全都不可约，但它们不是素元。例如，\(2\) 整除乘积 \((1 + \sqrt{-5})(1 - \sqrt{-5})\)，却不整除任一因子。特别地，唯一分解不再成立：数 \(6\) 可以以不止一种方式分解为不可约元素。

相比之下，每个欧几里得整环都是唯一分解整环，这蕴含每个不可约元素都是素元。欧几里得整环的公理蕴含，任意非零元素都可以写成不可约元素的有限乘积。它们还蕴含，可以使用欧几里得算法求任意两个非零元素 a 和 b 的最大公因子，也就是可被任意其他公因子整除的元素。这进一步蕴含，不可约元素分解在乘以单位的意义下唯一。

现在证明高斯整数是欧几里得整环，其中范数定义为 \(N(a + bi) = (a + bi)(a - bi) = a^2 + b^2\)。高斯整数 \(a - bi\) 称为 \(a + bi\) 的conjugate。不难验证，对任意复数 \(x\) 和 \(y\)，都有 \(N(xy) = N(x)N(y)\)。

为了看出这个范数定义使高斯整数成为欧几里得整环，只有第一个性质具有挑战。设我们想把 \(a + bi = (c + di) q + r\) 写出来，其中 \(q\) 与 \(r\) 合适。把 \(a + bi\) 和 \(c + di\) 视为复数，进行除法

\[\frac{a + bi}{c + di} = \frac{(a + bi)(c - di)}{(c + di)(c-di)} = \frac{ac + bd}{c^2 + d^2} + \frac{bc -ad}{c^2+d^2} i.\]

所得实部和虚部未必是整数，但可以把它们舍入到最接近的整数 \(u\) 和 \(v\)。于是可将右端写成 \((u + vi) + (u' + v'i)\)，其中 \(u' + v'i\) 是剩余部分。注意有 \(|u'| \le 1/2\) 和 \(|v'| \le 1/2\)，从而

\[N(u' + v' i) = (u')^2 + (v')^2 \le 1/4 + 1/4 \le 1/2.\]

两边乘以 \(c + di\)，得到

\[a + bi = (c + di) (u + vi) + (c + di) (u' + v'i).\]

令 \(q = u + vi\) 且 \(r = (c + di) (u' + v'i)\)，则有 \(a + bi = (c + di) q + r\)，只需估计 \(N(r)\)：

\[N(r) = N(c + di)N(u' + v'i) \le N(c + di) \cdot 1/2 < N(c + di).\]

刚才的论证要求把高斯整数视为复数的子集。因此，在 Lean 中形式化它的一种选择，是把高斯整数嵌入复数，把整数嵌入高斯整数，定义从实数到整数的舍入函数，并非常谨慎地在这些数系之间来回切换。事实上，Mathlib 采用的正是这种方法；其中高斯整数本身被构造为quadratic integers环的特例。参见文件 GaussianInt.lean。

这里我们改为给出一个始终停留在整数中的论证。这展示了形式化数学时常遇到的一种选择。给定一个需要库中尚无概念或机制的论证，有两种选择：要么形式化所需概念和机制，要么调整论证以利用已有概念和机制。当前者的结果可在其他语境中使用时，通常值得投入时间。从实用角度看，有时寻找更初等的证明会更高效。

整数通常的带余除法定理说，对每个 \(a\) 和非零 \(b\)，存在 \(q\) 与 \(r\)，使得 \(a = b q + r\) 且 \(0 \le r < |b|\)。这里我们将使用下面的变体：存在 \(q'\) 与 \(r'\)，使得 \(a = b q' + r'\) 且 \(|r'| \le |b|/2\)。你可以检查，如果第一个陈述中的 \(r\) 满足 \(r \le |b|/2\)，则可取 \(q' = q\) 且 \(r' = r\)；否则可取 \(q' = q + 1\) 且 \(r' = r - b\)（当 \(b\) 为正时；其他情况相应调整）。我们感谢 Heather Macbeth 提出下面这个更优雅的方法，它避免了分类定义。只需在除法前把 b / 2 加到 a 上，然后从余数中减去它。

def div' (a b : ℤ) :=
  (a + b / 2) / b

def mod' (a b : ℤ) :=
  (a + b / 2) % b - b / 2

theorem div'_add_mod' (a b : ℤ) : b * div' a b + mod' a b = a := by
  rw [div', mod']
  linarith [Int.mul_ediv_add_emod (a + b / 2) b]

theorem abs_mod'_le (a b : ℤ) (h : 0 < b) : |mod' a b| ≤ b / 2 := by
  rw [mod', abs_le]
  constructor
  · linarith [Int.emod_nonneg (a + b / 2) h.ne']
  have := Int.emod_lt_of_pos (a + b / 2) h
  have := Int.mul_ediv_add_emod b 2
  have := Int.emod_lt_of_pos b zero_lt_two
  linarith

注意这里使用了我们的老朋友 linarith。还需要把 mod' 用 div' 表示出来。

theorem mod'_eq (a b : ℤ) : mod' a b = a - b * div' a b := by linarith [div'_add_mod' a b]

最后，用记号 x / y 为高斯整数定义除法，它把复数商舍入到最接近的高斯整数。为此我们使用自定义的 Int.div'。如上所算，若 x 为 \(a + bi\)、y 为 \(c + di\)，则 x / y 的实部和虚部分别是最接近以下数值的整数：

\[\frac{ac + bd}{c^2 + d^2} \quad \text{and} \quad \frac{bc -ad}{c^2+d^2},\]

这里分子分别是 \((a + bi) (c - di)\) 的实部和虚部，而两个分母都等于 \(c + di\) 的范数。

instance : Div GaussInt :=
  ⟨fun x y ↦ ⟨Int.div' (x * conj y).re (norm y), Int.div' (x * conj y).im (norm y)⟩⟩

theorem norm_mod_lt (x : GaussInt) {y : GaussInt} (hy : y ≠ 0) :
    (x % y).norm < y.norm := by
  have norm_y_pos : 0 < norm y := by rwa [norm_pos]
  have H1 : x % y * conj y = ⟨Int.mod' (x * conj y).re (norm y), Int.mod' (x * conj y).im (norm y)⟩
  · ext <;> simp [Int.mod'_eq, mod_def, div_def, norm] <;> ring
  have H2 : norm (x % y) * norm y ≤ norm y / 2 * norm y
  · calc
      norm (x % y) * norm y = norm (x % y * conj y) := by simp only [norm_mul, norm_conj]
      _ = |Int.mod' (x.re * y.re + x.im * y.im) (norm y)| ^ 2
          + |Int.mod' (-(x.re * y.im) + x.im * y.re) (norm y)| ^ 2 := by simp [H1, norm, sq_abs]
      _ ≤ (y.norm / 2) ^ 2 + (y.norm / 2) ^ 2 := by gcongr <;> apply Int.abs_mod'_le _ _ norm_y_pos
      _ = norm y / 2 * (norm y / 2 * 2) := by ring
      _ ≤ norm y / 2 * norm y := by gcongr; apply Int.ediv_mul_le; norm_num
  calc norm (x % y) ≤ norm y / 2 := le_of_mul_le_mul_right H2 norm_y_pos
    _ < norm y := by
        apply Int.ediv_lt_of_lt_mul
        · norm_num
        · linarith

我们已经接近终点。范数函数 norm 把高斯整数映到非负整数。我们需要一个把高斯整数映到自然数的函数，并通过把 norm 与函数 Int.natAbs 复合来得到它；后者把整数映到自然数。接下来两个引理中的第一个说明，将范数映到自然数再映回整数不会改变值。第二个重新表述范数递减这一事实。

theorem coe_natAbs_norm (x : GaussInt) : (x.norm.natAbs : ℤ) = x.norm :=
  Int.natAbs_of_nonneg (norm_nonneg _)

theorem natAbs_norm_mod_lt (x y : GaussInt) (hy : y ≠ 0) :
    (x % y).norm.natAbs < y.norm.natAbs := by
  apply Int.ofNat_lt.1
  simp only [Int.natCast_natAbs, abs_of_nonneg, norm_nonneg]
  exact norm_mod_lt x hy

还需要建立欧几里得整环上范数函数的第二个关键性质。

theorem not_norm_mul_left_lt_norm (x : GaussInt) {y : GaussInt} (hy : y ≠ 0) :
    ¬(norm (x * y)).natAbs < (norm x).natAbs := by
  apply not_lt_of_ge
  rw [norm_mul, Int.natAbs_mul]
  apply le_mul_of_one_le_right (Nat.zero_le _)
  apply Int.ofNat_le.1
  rw [coe_natAbs_norm]
  exact Int.add_one_le_of_lt ((norm_pos _).mpr hy)

现在可以把这些合在一起，证明高斯整数是欧几里得整环的一个实例。我们使用已经定义的商函数和余数函数。Mathlib 对欧几里得整环的定义比上面的定义更一般，因为它允许我们证明余数相对于任意良基度量递减。比较一个返回自然数的范数函数的取值，只是这类度量的一个实例；在这种情况下，所需性质就是定理 natAbs_norm_mod_lt 和 not_norm_mul_left_lt_norm。

    quotient := (· / ·)

    r := (measure (Int.natAbs ∘ norm)).1
